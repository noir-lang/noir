1//! Verifies the implicit invariant that Brillig SSA must satisfy around
2//! `array_set` and reference counts.
3//!
4//! # The invariant
5//!
6//! In Brillig, `array_set vX, i, x` may modify `vX`'s storage in place at runtime
7//! when `vX`'s reference count is 1. SSA-level semantics still says `vX` is unchanged
8//! and the `array_set` produces a fresh value; the in-place mutation is a runtime
9//! optimization that's only sound when no later use can observe `vX`'s pre-mutation
10//! contents through aliasing.
11//!
12//! Two mechanisms keep the optimization invisible to SSA semantics:
13//!
14//! 1. **`inc_rc`** before the `array_set` forces RC ≥ 2 at runtime so `array_set`
15//!    copies rather than mutating in place.
16//! 2. **Block-parameter threading** routes the post-mutation value forward as a new
17//!    SSA value (the `array_set`'s result), so no later instruction references
18//!    `vX` after the mutation.
19//!
20//! The frontend uses whichever mechanism the program needs. This pass verifies
21//! that one of them is in place for every `array_set` whose source has an
22//! aliased use reachable forward in the CFG.
23//!
24//! # Algorithm
25//!
26//! For each `array_set vX, …`:
27//!
28//! 1. **Backward alias-set.** Compute the set of values that may share `vX`'s
29//!    storage *at the `array_set`'s program point* by walking block-parameter →
30//!    predecessor-arg edges backward to a fixed point. Only aliasing introduced
31//!    by the values that flow *into* `vX`'s binding is included. Filtered to
32//!    drop values that can't represent pre-mutation storage: `array_set` /
33//!    `Call` results (always fresh), iteration-local fresh results (`MakeArray`
34//!    or `Call` on back-edge args), instruction results defined in the
35//!    `array_set`'s own block at an index after the `array_set` (they can land in
36//!    the backward set through a forward-then-back round-trip, but don't exist
37//!    as storage at the `array_set`'s program point yet), and **swap-excluded
38//!    siblings** — when `vX` is a loop-header parameter swapped onto a sibling
39//!    parameter that is freshly re-allocated each iteration, the sibling is a
40//!    distinct per-iteration storage and is dropped (see
41//!    [`Context::swap_excluded_aliases`]).
42//! 2. **Fast full-accept.** Accept the `array_set` outright (skipping the
43//!    forward walk) when some `inc_rc` on an alias-set member protects it on
44//!    every path by a cheap test: an `inc_rc` earlier in the `array_set`'s own
45//!    block, in a block that **dominates** it, or — for a non-source alias —
46//!    on a loop **back-edge** (the back-edge-participant relaxation, where the
47//!    frontend is deliberately managing iteration aliasing). See
48//!    [`Context::some_inc_rc_precedes`].
49//! 3. **Per-member coverage → use-set.** Otherwise narrow the alias-set to the
50//!    members that are *not* provably protected, and seed the forward walk
51//!    with only those ([`Context::unprotected_aliases`]). Coverage is per
52//!    member: thread the source's storage backward through the CFG
53//!    ([`Context::compute_uncovered_values`]) and drop a member when, on every
54//!    path where it can be the source's storage at the `array_set`, that storage
55//!    is RC-bumped by an `inc_rc` or is an iteration-local fresh allocation (a
56//!    `MakeArray`/`Call` re-executed each loop iteration). Dropping covered
57//!    members is what stops the forward walk from flagging a read of, say, an
58//!    `inc_rc`'d sibling on a path where the `array_set` never mutates that
59//!    storage. An empty use-set means every member is protected — accept.
60//! 4. **Protected-participant filter.** A second narrowing inside the forward
61//!    walk: drop every use-set member (other than the source) that both
62//!    carries its own `inc_rc` and is a loop back-edge *participant* (it
63//!    flows — directly or through forward edges — into a back-edge arg
64//!    position). Being in the alias-set means the value flows *into* the
65//!    source, so combined with its back-edge participation the `inc_rc` is
66//!    loop-carried: it runs before the value crosses the back-edge that
67//!    re-binds it onto the source, so by the time the value's storage
68//!    equals the source's it is RC ≥ 2 and the `array_set` copies. Reads of
69//!    it therefore can't observe an in-place mutation. The gate is **per
70//!    value** on that value's own `inc_rc`: a back-edge position fed by an
71//!    `inc_rc`'d value on one predecessor edge and an unprotected value on
72//!    another drops only the protected value, leaving the unprotected one
73//!    for the walk to flag. Restricting to alias-set members is the
74//!    soundness guard — a value that merely *receives* the source's storage
75//!    (a forward successor, not in the source's backward set) has its
76//!    `inc_rc` run *after* the `array_set` and so is not protected; it stays
77//!    in the use-set. This is what lets the verifier accept the latch-block
78//!    shape (an `inc_rc v` placed before `v` is threaded *forward* into the
79//!    latch that then closes the loop) without the unsoundness of crediting
80//!    an `inc_rc` to a value the `array_set` actually mutates first.
81//! 5. **Forward walk.** Walk the CFG forward from the `array_set` with the
82//!    use-set as the initial set of live aliases. At each block-parameter
83//!    crossing we both **kill** params that the predecessor rebinds to a
84//!    non-alias and **add** params whose arg is still an alias (so alias
85//!    propagation stays accurate as the walk crosses joins and loops). The
86//!    walk maintains two additional pieces of state:
87//!
88//!    - **`derived`**, the set of values that may share the source's storage
89//!      through transitive in-place chain mutations. Seeded with the
90//!      `array_set`'s own result; extended by every later `array_set` whose
91//!      `array` operand is already in `derived`.
92//!    - **`tainted_indices`**, the storage positions any chain link may
93//!      already have written. Seeded with the `array_set`'s own write index
94//!      (when constant) and widened by chain-link writes; collapsed to "all
95//!      positions" if any chain link uses a dynamic index.
96//!
97//!    An `array_get` on a use-set member is a hazard iff its read index is
98//!    covered by `tainted_indices`. An `array_set` on a use-set member is
99//!    index-aware too: it produces a copy of the source with one index
100//!    overwritten, so it observes (copies forward) only the indices it does
101//!    *not* write — a hazard iff some `tainted_indices` position differs
102//!    from its write index (a same-index write overwrites the mutation and
103//!    is not a hazard; it instead extends the chain). All other uses are
104//!    always hazards. An `inc_rc v` with `v ∈ use_set ∪ derived` lifts the
105//!    storage's RC,
106//!    so chain writes after it run on fresh storage — `derived` is cleared
107//!    at that point. `tainted_indices` is *not* cleared: prior chain writes
108//!    have already mutated the source's storage.
109//!
110//! # Precondition
111//!
112//! Must be run **after `mem2reg_brillig`**. The alias walk through
113//! block-parameter edges only reflects post-mem2reg aliasing; pre-mem2reg, an
114//! `Allocate`/`Store`/`Load` chain would route aliasing through references that
115//! this pass does not track.
116//!
117//! # Known limitations
118//!
119//! The verifier walks aliasing *only* through block-parameter edges. The
120//! following are not tracked and are documented gaps:
121//!
122//! - **Sibling-args-of-same-value.** A `jmp b1(v, v)` makes `b1`'s two
123//!   parameters runtime-equal, but the backward walk doesn't see this — each
124//!   parameter's backward set includes `v` but not the other parameter. The
125//!   `end_to_end_sibling_args_to_same_value_is_false_negative` and
126//!   `end_to_end_sibling_args_across_jmp_is_false_negative` tests pin this
127//!   down as documented false negatives.
128//! - **Nested-array `MakeArray`**, **`IfElse` on arrays**, **non-inlined
129//!   `Call` returns**, and **`Store`/`Load` on ineligible (nested-ref)
130//!   allocates** are likewise not tracked.
131//! - **Iteration-local fresh array read back within one iteration.** The
132//!   freshness wall (step 3) credits a `MakeArray`/`Call` that shares a loop
133//!   with the `array_set` as distinct per-iteration storage, which is sound
134//!   only because the result is normally threaded forward rather than the
135//!   freshly-allocated source being re-read after its in-place mutation. A
136//!   hand-written loop body that allocates an array, mutates it in place, and
137//!   then reads the *same* value back within the same iteration would not be
138//!   flagged. The frontend threads the `array_set` result instead of
139//!   re-reading the source, so it does not emit this shape; crediting
140//!   freshness only ever *accepts* more, so this can only be a false negative,
141//!   never a false positive.

165/// Pre-computed indices over a Brillig function. The verifier's per-array_set
166/// checks read from these structures rather than re-scanning the function.
167struct Context<'f> {
168    function: &'f Function,
169    /// Control-flow graph. Used by [`Context::some_inc_rc_precedes`] to walk
170    /// predecessors backward when checking whether the `inc_rc`-carrying
171    /// blocks collectively cover every path to an `array_set`.
172    cfg: ControlFlowGraph,
173    /// Dominator tree. Used by [`Context::some_inc_rc_precedes`] as a fast
174    /// path: a single `inc_rc` block that dominates the `array_set` covers
175    /// every path without the backward value walk.
176    dom_tree: DominatorTree,
177    /// Array values that allocate fresh storage at their definition
178    /// (`make_array` / `Call` results). Used as a freshness wall by
179    /// [`Context::compute_uncovered_values`].
180    fresh_array_values: HashSet<ValueId>,
181    /// The block set of each loop in the function. The freshness wall only
182    /// credits a fresh allocation when its defining block and the `array_set`'s
183    /// block lie in a common loop — i.e. the allocation re-executes every
184    /// iteration that reaches the `array_set`, so its storage is distinct per
185    /// iteration. A one-time allocation outside any shared loop is mutated in
186    /// place exactly once, so reading it back is a genuine hazard.
187    loop_blocks: Vec<BTreeSet<BasicBlockId>>,
188    /// For each array-typed value `V`, the set of values that may share
189    /// `V`'s storage **at `V`'s program point** — the source itself plus
190    /// anything that flows backward into it through block-parameter →
191    /// predecessor-arg chains. Computed to a fixed point.
192    ///
193    /// Values absent from this map have an implicit singleton class
194    /// `{v}` (typically instruction results that aren't passed as args
195    /// anywhere they'd matter — handled in [`Context::alias_set_for`]).
196    ///
197    /// The walk follows the directed `param ← arg` edges, so two values
198    /// that flow into different parameters of the same block are *not*
199    /// siblings of each other under backward aliasing — neither is
200    /// reachable from the other along param→arg edges. This is what
201    /// keeps path-merge over-approximation at join points out of the
202    /// alias-set.
203    backward_aliases: HashMap<ValueId, im::HashSet<ValueId>>,
204    /// For each array-typed value defined by an instruction, the
205    /// `(block, instruction-position-within-block)` of the defining
206    /// instruction. Used by the def-block-entry kill rule in
207    /// [`Context::succ_use_set`].
208    array_value_defs: HashMap<ValueId, (BasicBlockId, usize)>,
209    /// Every array-typed value that does **not** carry the pre-mutation
210    /// aliasing of an `array_set`'s source. Filtered out of every
211    /// alias-set (except when the value is the `array_set`'s own source)
212    /// — see [`Context::alias_set_for`]. Includes:
213    /// - `array_set` results: represent the *post*-mutation value of the
214    ///   source. Uses of them (or anything threaded from them through
215    ///   block params) are intentional reads, not hazards.
216    /// - `Call` results: typically fresh arrays allocated by the callee.
217    ///   Heuristic — a function that returns its input *is* a real
218    ///   alias, and filtering would miss those cases; that's a
219    ///   documented gap. In practice the frontend's array-returning
220    ///   functions allocate fresh storage.
221    non_aliasing_array_values: HashSet<ValueId>,
222    /// Back-edge args that re-allocate distinct storage every iteration:
223    /// `MakeArray` results (re-executes each iteration) and `Call` results
224    /// (the callee allocates fresh). `array_set` results are excluded —
225    /// they may mutate in place, so they aren't guaranteed-fresh. Two uses:
226    ///
227    /// - [`Context::alias_set_for`] drops such values from the alias-set:
228    ///   a back-edge that puts one in a loop-header parameter's class is
229    ///   conflating distinct runtime storages across iterations. (`Call`
230    ///   results are also dropped there by
231    ///   [`Context::non_aliasing_array_values`]; the overlap is harmless.)
232    /// - the swap freshening guard in [`Context::new`] requires the
233    ///   swapped-out sibling to be one of these — see
234    ///   [`Context::swap_excluded_aliases`].
235    ///
236    /// Values that are *not* back-edge args are kept in the alias-set:
237    /// they represent a one-time allocation whose storage the `array_set`
238    /// may mutate in place.
239    iteration_local_fresh: HashSet<ValueId>,
240    /// `inc_rc value` instructions indexed by their operand. Each entry is
241    /// the `(block, instruction-position-within-block)` of one `inc_rc`.
242    inc_rc_locations: HashMap<ValueId, Vec<(BasicBlockId, usize)>>,
243    /// Values that appear at least once as a jmp/jmpif arg on a loop
244    /// back-edge. Used by both:
245    ///
246    /// - [`Context::iteration_local_fresh`] (the back-edge args that are
247    ///   `make_array` or `Call` results, computed in [`Context::new`]); and
248    /// - the **back-edge-participant relaxation** in
249    ///   [`Context::some_inc_rc_precedes`]: an `inc_rc` on a non-source
250    ///   alias that's also a back-edge arg is taken as a codegen
251    ///   signal that the frontend is deliberately managing
252    ///   iteration aliasing, regardless of program-point order.
253    ///
254    /// Forward-edge cases (e.g. re-seeding a value with a global on a
255    /// non-loop branch) are handled precisely by [`Context::backward_aliases`]
256    /// — the forward-edge value simply isn't in the source's backward
257    /// set in the first place — so this signal only needs to cover
258    /// loop back-edges.
259    back_edge_args: HashSet<ValueId>,
260    /// The backward-alias closure of [`Context::back_edge_args`]: every
261    /// value that flows (directly or through forward block-parameter
262    /// edges) into a loop back-edge arg position. A value `v` is a
263    /// back-edge *participant* when its storage can become a loop-header
264    /// parameter's storage across a back-edge — even when `v` itself is
265    /// not the literal arg, but reaches one through a forward edge first
266    /// (e.g. `inc_rc v; jmp latch(v)` then `latch: jmp header(v)`, where
267    /// only the latch's param is the literal back-edge arg).
268    ///
269    /// Used by the **protected-participant filter** in
270    /// [`Context::find_reachable_aliased_use`]: a value that is in the
271    /// `array_set` source's alias-set (so it flows *into* the source),
272    /// is a back-edge participant, and carries its own `inc_rc` is
273    /// RC ≥ 2 whenever it equals the source's storage (the `inc_rc` is
274    /// loop-carried — it runs before the value crosses the back-edge onto
275    /// the source), so reads of it can't observe an in-place mutation and
276    /// it is dropped from the forward walk's use-set.
277    ///
278    /// The filter is gated **per value** on that value's own `inc_rc`:
279    /// membership in this set alone never exonerates anything. This is
280    /// what keeps it sound where widening the relaxation to "some
281    /// participant has an `inc_rc`" would not — a back-edge position fed
282    /// by an `inc_rc`'d value on one predecessor edge and an unprotected
283    /// value on another only drops the protected value, leaving the
284    /// unprotected one for the forward walk to flag.
285    back_edge_participants: HashSet<ValueId>,
286    /// **Swap exclusions.** For a loop-header parameter `P`, the set of
287    /// sibling parameters `Q` that `P` is rebound to on the loop
288    /// back-edge (`P ← Q`, the array-variable swap `c4 = c3`) while `Q`
289    /// is simultaneously rebound to a freshly-allocated, iteration-local
290    /// value — a `make_array` (`c3 = [..]`) or a `Call` result
291    /// (`c3 = f()`), but never an `array_set` result (which may be an
292    /// in-place mutation, not a fresh allocation). Such a `Q` is a *distinct
293    /// per-iteration storage* that `P` only *becomes* in a later
294    /// iteration — never the storage this iteration's `array_set` on `P`
295    /// mutates (the swap rotates a fresh allocation into `P` while the
296    /// mutated storage is left behind, dead). `Q` is dropped from `P`'s
297    /// alias-set in [`Context::alias_set_for`], which makes the forward
298    /// walk both **kill** `P` where the back-edge rebinds it to `Q` and
299    /// decline to **add** a fresh alias of `Q` past any later edge (the
300    /// inclusive-range *peel* re-runs the same swap on a *forward* exit
301    /// edge — excluding `Q` at the seed covers that edge too, which a
302    /// kill keyed on the back-edge alone could not).
303    ///
304    /// Guarded by a **back-edge-only** check: if `Q` also flows into `P`
305    /// on a forward edge, `P` and `Q` can already alias *within* an
306    /// iteration (the `array_set` would mutate `Q`'s live storage), so the
307    /// exclusion would be unsound and is not recorded. The freshening
308    /// requirement on `Q` is the second guard — a loop-invariant `Q`
309    /// swapped into `P` would make `P_k = Q_{k-1} = Q_k` and genuinely
310    /// alias.
311    ///
312    /// **Forward propagation.** After recording, exclusions are pushed
313    /// forward across non-back-edge parameter edges to a fixed point: on a
314    /// forward edge `R ← P`, `R` *is* `P` within the same iteration, so `R`
315    /// inherits `P`'s exclusions. This carries an exclusion from a swapped
316    /// loop-header parameter onto a successor parameter seeded from it — e.g.
317    /// an inner-loop header (the `array_set`'s source) fed from the swapped
318    /// outer-loop parameter — so `Q` is dropped for the inner source too.
319    /// Back-edges are excluded from propagation: across one, `R` is a *prior*
320    /// iteration's `P`, a distinct storage.
321    swap_excluded_aliases: HashMap<ValueId, HashSet<ValueId>>,
322}

324impl<'f> Context<'f> {
325    fn new(function: &'f Function) -> Self {
326        let cfg = ControlFlowGraph::with_function(function);
327        let post_order = PostOrder::with_cfg(&cfg);
328        let dom_tree = DominatorTree::with_cfg_and_post_order(&cfg, &post_order);
329
330        // Cache the reverse-post-order block list once. Used for the
331        // per-instruction setup pass below and (more importantly) by
332        // [`compute_backward_aliases`], whose fixed-point converges in
333        // far fewer passes when blocks are visited in RPO than in the
334        // ID-sorted order `function.reachable_blocks()` returns.
335        let rpo: Vec<BasicBlockId> = post_order.into_vec_reverse();
336
337        // Loop back-edges so we can recognize iteration-local
338        // `make_array` results (those passed as args on a back-edge),
339        // which represent fresh-per-iteration storage and so don't
340        // carry the array_set's pre-mutation aliasing forward through
341        // the loop header.
342        let loops = Loops::find_all(function, LoopOrder::InsideOut);
343        let back_edges: HashSet<(BasicBlockId, BasicBlockId)> =
344            loops.yet_to_unroll.iter().map(|l| (l.back_edge_start, l.header)).collect();
345        // The block set of each loop, used by the freshness wall in
346        // [`Context::compute_uncovered_values`] to decide whether a fresh
347        // allocation re-executes on every iteration that reaches an array_set.
348        let loop_blocks: Vec<BTreeSet<BasicBlockId>> =
349            loops.yet_to_unroll.iter().map(|l| l.blocks.clone()).collect();
350
351        let mut inc_rc_locations: HashMap<ValueId, Vec<(BasicBlockId, usize)>> = HashMap::default();
352        let mut array_value_defs: HashMap<ValueId, (BasicBlockId, usize)> = HashMap::default();
353        let mut non_aliasing_array_values: HashSet<ValueId> = HashSet::default();
354        let mut make_array_values: HashSet<ValueId> = HashSet::default();
355        let mut call_result_values: HashSet<ValueId> = HashSet::default();
356        let mut back_edge_args: HashSet<ValueId> = HashSet::default();
357        // For each destination block, the list of `(predecessor, args)`
358        // pairs collected from terminators. Used to drive the backward
359        // alias-set fixed-point: each block parameter's set is the union
360        // of its arg at that position from every incoming edge.
361        let mut incoming_edges: HashMap<BasicBlockId, Vec<(BasicBlockId, Vec<ValueId>)>> =
362            HashMap::default();
363
364        // Single pass over every reachable block to populate
365        // per-instruction indices and per-block incoming-edge tables.
366        for &block_id in &rpo {
367            for (idx, instruction_id) in function.dfg[block_id].instructions().iter().enumerate() {
368                let instruction = &function.dfg[*instruction_id];
369
370                if let Instruction::IncrementRc { value } = instruction {
371                    inc_rc_locations.entry(*value).or_default().push((block_id, idx));
372                }
373
374                let is_call = matches!(instruction, Instruction::Call { .. });
375                let is_non_aliasing =
376                    is_call || matches!(instruction, Instruction::ArraySet { .. });
377                let is_make_array = matches!(instruction, Instruction::MakeArray { .. });
378                for &result in function.dfg.instruction_results(*instruction_id) {
379                    if function.dfg.type_of_value(result).contains_an_array() {
380                        array_value_defs.insert(result, (block_id, idx));
381                        if is_non_aliasing {
382                            non_aliasing_array_values.insert(result);
383                        }
384                        if is_make_array {
385                            make_array_values.insert(result);
386                        }
387                        if is_call {
388                            call_result_values.insert(result);
389                        }
390                    }
391                }
392            }
393
394            if let Some(terminator) = function.dfg[block_id].terminator() {
395                match terminator {
396                    TerminatorInstruction::Jmp { destination, arguments, .. } => {
397                        incoming_edges
398                            .entry(*destination)
399                            .or_default()
400                            .push((block_id, arguments.clone()));
401                        if back_edges.contains(&(block_id, *destination)) {
402                            back_edge_args.extend(arguments.iter().copied());
403                        }
404                    }
405                    TerminatorInstruction::JmpIf {
406                        then_destination,
407                        then_arguments,
408                        else_destination,
409                        else_arguments,
410                        ..
411                    } => {
412                        incoming_edges
413                            .entry(*then_destination)
414                            .or_default()
415                            .push((block_id, then_arguments.clone()));
416                        incoming_edges
417                            .entry(*else_destination)
418                            .or_default()
419                            .push((block_id, else_arguments.clone()));
420                        if back_edges.contains(&(block_id, *then_destination)) {
421                            back_edge_args.extend(then_arguments.iter().copied());
422                        }
423                        if back_edges.contains(&(block_id, *else_destination)) {
424                            back_edge_args.extend(else_arguments.iter().copied());
425                        }
426                    }
427                    _ => (),
428                }
429            }
430        }
431
432        let backward_aliases = compute_backward_aliases(function, &rpo, &incoming_edges);
433
434        // Backward-alias closure of the literal back-edge args: every
435        // value that can become a loop-header parameter's storage across
436        // a back-edge, including those that only reach the back-edge arg
437        // position through a forward edge first. A literal back-edge arg
438        // that is a block parameter contributes its whole backward set;
439        // an instruction result or function parameter contributes only
440        // itself (its backward set is the singleton).
441        let mut back_edge_participants: HashSet<ValueId> = HashSet::default();
442        for &arg in &back_edge_args {
443            back_edge_participants.insert(arg);
444            if let Some(arg_set) = backward_aliases.get(&arg) {
445                back_edge_participants.extend(arg_set.iter().copied());
446            }
447        }
448
449        // A back-edge arg is iteration-local *fresh* if it re-allocates
450        // distinct storage every iteration: a `make_array` (re-executes)
451        // or a `Call` result (the callee allocates fresh — the same
452        // assumption [`Context::non_aliasing_array_values`] makes).
453        // `array_set` results are deliberately *excluded*: an `array_set`
454        // may mutate its source in place, so its result can be the *same*
455        // storage as a prior iteration's, breaking the distinct-generation
456        // guarantee the swap exclusion relies on.
457        let iteration_local_fresh: HashSet<ValueId> = back_edge_args
458            .iter()
459            .copied()
460            .filter(|v| make_array_values.contains(v) || call_result_values.contains(v))
461            .collect();
462
463        // Array values that allocate **fresh** storage at their definition:
464        // `make_array` (a new array) and `Call` results (the callee allocates;
465        // the same heuristic [`Context::non_aliasing_array_values`] makes).
466        // `array_set` results are excluded — they may mutate in place rather
467        // than allocate. Used by the freshness wall in
468        // [`Context::compute_uncovered_values`]: storage that is freshly
469        // allocated on a path has no other live alias, so an in-place mutation
470        // there cannot be observed through one.
471        let fresh_array_values: HashSet<ValueId> =
472            make_array_values.union(&call_result_values).copied().collect();
473
474        // Swap exclusions. For every loop back-edge `be_start → header`,
475        // inspect each array-typed header parameter at `source_pos`: if
476        // the back-edge rebinds it to a *sibling* header parameter
477        // (`source ← sibling`) whose own back-edge arg is an
478        // iteration-local fresh allocation, and the source and sibling
479        // receive distinct storage on every forward edge into the header,
480        // record the sibling as excluded from the source's alias-set. See
481        // [`Context::swap_excluded_aliases`].
482        let mut swap_excluded_aliases: HashMap<ValueId, HashSet<ValueId>> = HashMap::default();
483        for &(be_start, header) in &back_edges {
484            let Some(edges) = incoming_edges.get(&header) else { continue };
485            let Some(be_args) =
486                edges.iter().find(|(pred, _)| *pred == be_start).map(|(_, args)| args)
487            else {
488                continue;
489            };
490            let params = function.dfg.block_parameters(header);
491            for (source_pos, &source_param) in params.iter().enumerate() {
492                if !function.dfg.type_of_value(source_param).contains_an_array() {
493                    continue;
494                }
495                let Some(&sibling) = be_args.get(source_pos) else { continue };
496                // `source_param ← sibling` must be a genuine swap to a
497                // *different* sibling header parameter (not self-threading,
498                // not a result).
499                if sibling == source_param {
500                    continue;
501                }
502                let Some(sibling_pos) = params.iter().position(|&pp| pp == sibling) else {
503                    continue;
504                };
505                // The sibling's own back-edge arg must be an
506                // iteration-local fresh allocation (the `c3 = [..]` or
507                // `c3 = f()` half of the swap).
508                if !be_args.get(sibling_pos).is_some_and(|a| iteration_local_fresh.contains(a)) {
509                    continue;
510                }
511                // Loop-entry guard. On every *forward* edge into the
512                // header, the source param and the sibling must receive
513                // **distinct** storage, or they can already alias at the
514                // array_set in the entry iteration (`source_0 = sibling_0`)
515                // and the exclusion would mask a real hazard. Two ways
516                // that can happen:
517                //
518                // - the sibling flows into the source's forward arg
519                //   (`sibling ∈ backward(source_forward_arg)`) — the
520                //   source *is* the sibling from the start.
521                // - the source's and sibling's forward args share a
522                //   backward-set member — e.g. `jmp header(v, v)` feeds the
523                //   same `v` to both, so they're runtime-equal even though
524                //   the directed backward walk keeps them in separate sets.
525                let backward_set = |v: ValueId| -> im::HashSet<ValueId> {
526                    backward_aliases.get(&v).cloned().unwrap_or_else(|| im::HashSet::unit(v))
527                };
528                let entry_aliased = edges
529                    .iter()
530                    .filter(|(pred, _)| !back_edges.contains(&(*pred, header)))
531                    .any(|(_, args)| {
532                        let Some(&source_forward_arg) = args.get(source_pos) else {
533                            return false;
534                        };
535                        let source_forward_aliases = backward_set(source_forward_arg);
536                        if source_forward_aliases.contains(&sibling) {
537                            return true;
538                        }
539                        args.get(sibling_pos).is_some_and(|&sibling_forward_arg| {
540                            let sibling_forward_aliases = backward_set(sibling_forward_arg);
541                            source_forward_aliases
542                                .iter()
543                                .any(|x| sibling_forward_aliases.contains(x))
544                        })
545                    });
546                if entry_aliased {
547                    continue;
548                }
549                swap_excluded_aliases.entry(source_param).or_default().insert(sibling);
550            }
551        }
552
553        // Propagate swap exclusions forward across non-back-edge
554        // block-parameter edges. On a forward edge `P ← A`, `P` is `A` within
555        // the same iteration, so `P`'s storage is `A`'s storage; a sibling
556        // distinct from `A`'s per-iteration storage is therefore distinct from
557        // `P`'s too. This carries an exclusion recorded on a loop-header
558        // parameter to a successor parameter seeded from it — e.g. an
559        // inner-loop header (the array_set source) fed from the swapped
560        // outer-loop parameter — so `alias_set_for` drops the sibling for the
561        // inner source as well. Back-edges are excluded: across one, `P` is a
562        // *prior* iteration's `A`, a distinct storage.
563        let mut changed = true;
564        while changed {
565            changed = false;
566            for (dest, edges) in &incoming_edges {
567                let params = function.dfg.block_parameters(*dest);
568                for (i, &param) in params.iter().enumerate() {
569                    for (pred, args) in edges {
570                        if back_edges.contains(&(*pred, *dest)) {
571                            continue;
572                        }
573                        let Some(&arg) = args.get(i) else { continue };
574                        let Some(arg_excl) = swap_excluded_aliases.get(&arg).cloned() else {
575                            continue;
576                        };
577                        let entry = swap_excluded_aliases.entry(param).or_default();
578                        for x in arg_excl {
579                            if x != param && entry.insert(x) {
580                                changed = true;
581                            }
582                        }
583                    }
584                }
585            }
586        }
587
588        Self {
589            function,
590            cfg,
591            dom_tree,
592            fresh_array_values,
593            loop_blocks,
594            backward_aliases,
595            array_value_defs,
596            non_aliasing_array_values,
597            iteration_local_fresh,
598            inc_rc_locations,
599            back_edge_args,
600            back_edge_participants,
601            swap_excluded_aliases,
602        }
603    }

605    /// Look up `source`'s backward alias-set and filter out values that
606    /// can't represent the pre-mutation storage of the `array_set` at
607    /// `(array_set_block, array_set_idx)`. The source itself is always
608    /// kept.
609    ///
610    /// **Non-aliasing-result filter.** Drop values produced by an
611    /// `array_set` or `Call`:
612    /// - **`array_set` result** represents a *post*-mutation value of its
613    ///   source. Uses of it (or of any name it gets re-bound to through
614    ///   block-parameter threading) are intentional reads of the mutated
615    ///   storage, not hazards. Keeping them in the alias-set would defeat
616    ///   the per-arg kill rule in [`Context::succ_use_set`]: a back-edge
617    ///   `jmp b(v_arr_set)` would see `v_arr_set` in the use-set and
618    ///   refuse to kill the receiving param, letting the alias leak past
619    ///   the loop.
620    /// - **`Call` result** is typically a fresh array allocated by the
621    ///   callee, so it isn't a real alias of the `array_set`'s source.
622    ///   This is a heuristic — a function that returns its input *would*
623    ///   create a real alias, and we'd miss that. In practice the
624    ///   frontend's array-returning functions allocate fresh storage.
625    ///
626    /// Also drop **iteration-local fresh results** — `MakeArray` (or
627    /// `Call`) results that appear on at least one loop back-edge
628    /// ([`Context::iteration_local_fresh`]). Such a value re-allocates
629    /// fresh storage each iteration, so the loop-header parameter it
630    /// feeds on the back-edge holds a *different* allocation in the next
631    /// iteration than the one this iteration's `array_set` may have
632    /// mutated. **Non-back-edge `MakeArray` results stay in the
633    /// alias-set** — they represent a one-time allocation whose storage
634    /// the `array_set` can mutate in place.
635    ///
636    /// **Post-array_set-in-same-block filter.** Drop instruction
637    /// results whose defining position is in `array_set_block` at an
638    /// index *greater than* `array_set_idx`. Such a value doesn't exist
639    /// at the `array_set`'s program point yet — it's literally about to
640    /// be allocated on a later instruction — so it can't represent the
641    /// storage the `array_set` might mutate. It can land in the backward
642    /// set through a round-trip: `make_array → forward-arg → block
643    /// param → back-edge → source's param`. Dropping it at lookup time
644    /// makes the per-arg kill in [`Context::succ_use_set`] correctly
645    /// fire on the forward edge where the future-value is passed.
646    ///
647    /// The source itself is kept even when it happens to be filtered
648    /// (e.g. a chain `v_a = array_set _ ; v_b = array_set v_a`): `v_a`
649    /// is *this* check's source, so its forward uses are exactly what
650    /// we want to look for. The post-array_set filter can't fire on the
651    /// source because the source must be defined before the `array_set`
652    /// that uses it.
653    fn alias_set_for(
654        &self,
655        source: ValueId,
656        array_set_block: BasicBlockId,
657        array_set_idx: usize,
658    ) -> im::HashSet<ValueId> {
659        let class = self
660            .backward_aliases
661            .get(&source)
662            .cloned()
663            .unwrap_or_else(|| im::HashSet::unit(source));
664        class
665            .into_iter()
666            .filter(|&v| {
667                if v == source {
668                    return true;
669                }
670                if self.non_aliasing_array_values.contains(&v) {
671                    return false;
672                }
673                if self.iteration_local_fresh.contains(&v) {
674                    return false;
675                }
676                if self.swap_excluded_aliases.get(&source).is_some_and(|qs| qs.contains(&v)) {
677                    return false;
678                }
679                if let Some(&(def_block, def_idx)) = self.array_value_defs.get(&v)
680                    && def_block == array_set_block
681                    && def_idx > array_set_idx
682                {
683                    return false;
684                }
685                true
686            })
687            .collect()
688    }

690    /// The subset of `alias_set` whose members are **not** provably protected
691    /// on every path to the `array_set` — the use-set the forward walk must
692    /// check. An empty result means every member is protected, so the
693    /// `array_set` can be accepted without the forward walk.
694    ///
695    /// A member is dropped (provably protected) when, on every runtime path,
696    /// the storage it represents is either RC-bumped by an `inc_rc` (so the
697    /// `array_set` copies) or an iteration-local fresh allocation (so it has no
698    /// other live alias). Dropping covered members is what stops the forward
699    /// walk from flagging a read of an `inc_rc`'d sibling on a path where the
700    /// `array_set` never mutates that storage (the alias-set-merging
701    /// false-positive). Keeping the uncovered members preserves real hazards,
702    /// such as the issue SSA where the source itself is uncovered and read
703    /// after the `array_set`.
704    fn unprotected_aliases(
705        &self,
706        alias_set: &im::HashSet<ValueId>,
707        source: ValueId,
708        array_set_block: BasicBlockId,
709        array_set_idx: usize,
710    ) -> im::HashSet<ValueId> {
711        // Fast full-accept short-circuits cover the whole array_set at once.
712        if self.some_inc_rc_precedes(alias_set, source, array_set_block, array_set_idx) {
713            return im::HashSet::new();
714        }
715        // Otherwise compute per-member coverage and keep only uncovered members.
716        let uncovered = self.compute_uncovered_values(source, array_set_block, array_set_idx);
717        alias_set.iter().copied().filter(|m| uncovered.contains(m)).collect()
718    }

720    /// Fast, whole-array_set acceptance predicate: returns `true` when some
721    /// `inc_rc` on an alias-set member protects the `array_set` on **every**
722    /// path. This is the cheap common case; per-path coverage that this misses
723    /// (e.g. `inc_rc` on one arm and a fresh allocation on the other) is
724    /// handled by [`Context::compute_uncovered_values`].
725    ///
726    /// Three acceptances, each sound on every path:
727    ///
728    /// - **Same-block, before the `array_set`.** The block is on every path to
729    ///   its own `array_set`, so an earlier `inc_rc` always runs first.
730    /// - **Dominating block.** An `inc_rc` whose block dominates the `array_set`
731    ///   is on every path to it (e.g. an `inc_rc` in a loop pre-header).
732    /// - **Back-edge-participant relaxation.** An `inc_rc` on a *non-source*
733    ///   alias that appears as a jmp/jmpif arg on a loop **back-edge** is taken
734    ///   as a codegen signal regardless of program-point order: the frontend
735    ///   is deliberately managing iteration aliasing, and the bump's program
736    ///   point can come *after* the `array_set` — it is the back-edge thread-back
737    ///   that delivers the protection to the next iteration. Forward-edge cases
738    ///   don't need this because [`Context::backward_aliases`] keeps the
739    ///   forward-edge value out of the source's alias-set in the first place.
740    ///
741    /// Mere RPO precedence is *not* one of these: a sibling-branch `inc_rc`
742    /// precedes in RPO yet the other branch reaches the `array_set` without it
743    /// (`end_to_end_branch_local_inc_rc_does_not_protect_array_set_in_join_is_rejected`).
744    ///
745    /// Well-formed SSA contains no `DecrementRc`, so we don't need to worry
746    /// about a `dec_rc` intervening between the `inc_rc` and the `array_set`.
747    fn some_inc_rc_precedes(
748        &self,
749        alias_set: &im::HashSet<ValueId>,
750        source: ValueId,
751        array_set_block: BasicBlockId,
752        array_set_idx: usize,
753    ) -> bool {
754        for value in alias_set {
755            let Some(locations) = self.inc_rc_locations.get(value) else {
756                continue;
757            };
758            if *value != source && self.back_edge_args.contains(value) {
759                return true;
760            }
761            for &(inc_block, inc_idx) in locations {
762                if inc_block == array_set_block {
763                    if inc_idx < array_set_idx {
764                        return true;
765                    }
766                } else if self.dom_tree.dominates(inc_block, array_set_block) {
767                    return true;
768                }
769            }
770        }
771        false
772    }

774    /// Whether `value` is a fresh allocation whose storage is distinct on
775    /// every iteration that reaches the `array_set` in `array_set_block` — i.e.
776    /// a `make_array`/`Call` result whose defining block shares a loop with
777    /// `array_set_block`, so it re-executes each iteration. A fresh allocation
778    /// outside any shared loop is a one-time storage that the `array_set`
779    /// mutates exactly once; reading it back through the same name would
780    /// observe that mutation, so it does **not** count as protection.
781    fn is_iteration_local_fresh(&self, value: ValueId, array_set_block: BasicBlockId) -> bool {
782        if !self.fresh_array_values.contains(&value) {
783            return false;
784        }
785        let Some(&(def_block, _)) = self.array_value_defs.get(&value) else {
786            return false;
787        };
788        self.loop_blocks
789            .iter()
790            .any(|blocks| blocks.contains(&def_block) && blocks.contains(&array_set_block))
791    }

793    /// The set of values that lie on at least one **uncovered** backward path
794    /// from the `array_set` — a path on which the threaded source storage reaches
795    /// a function parameter / unthreadable root without crossing an `inc_rc` or
796    /// iteration-local-fresh wall. The caller intersects this with the
797    /// alias-set to obtain the forward walk's use-set; an empty intersection
798    /// means every member is protected, so the `array_set` is accepted.
799    ///
800    /// The walk threads the *exact* source storage backward: across a
801    /// block-parameter edge it follows the predecessor's argument, and at an
802    /// `array_set` result it continues with that `array_set`'s `array` operand
803    /// (the result shares the operand's storage). This recognizes path-specific
804    /// protection a block-level cut cannot — `inc_rc` on one arm of a diamond
805    /// and a fresh `make_array` on the other, threaded through a loop (the
806    /// `func_1`/`v20` fuzzer shape).
807    ///
808    /// It is computed over the `(block, value)` threading graph in two phases,
809    /// iteratively (the predecessor chain can be thousands of blocks deep on
810    /// large functions, which recursion would risk overflowing):
811    ///
812    /// 1. **Build.** From the seed `(array_set_block, source)`, expand each
813    ///    state to its backward successors. A wall (`inc_rc` on the value, or
814    ///    an iteration-local fresh allocation) is a covered sink. Reaching a
815    ///    block with no predecessors, or a value defined by an unthreadable
816    ///    instruction (a non-iteration-local allocation, or an `array_get` of a
817    ///    nested array), or an edge whose argument can't be resolved, is an
818    ///    *uncovered terminal*. `seed_idx` limits the `inc_rc` wall to bumps
819    ///    *before* the `array_set` in its own block; any other block on a path
820    ///    executes fully before the `array_set`, so an `inc_rc` anywhere in it
821    ///    counts.
822    /// 2. **Propagate.** A state is uncovered if it is an uncovered terminal or
823    ///    has an uncovered successor; iterate to a fixed point. A cycle with no
824    ///    uncovered terminal stays covered (a back-edge introduces no new
825    ///    entry-originating path).
826    ///
827    /// Crediting freshness only ever shrinks the use-set, so it can add
828    /// false-negatives (a freshly-allocated array read back through its own
829    /// name after an in-place mutation — a shape the frontend does not emit)
830    /// but never false-positives.
831    fn compute_uncovered_values(
832        &self,
833        source: ValueId,
834        array_set_block: BasicBlockId,
835        array_set_idx: usize,
836    ) -> HashSet<ValueId> {
837        struct Node {
838            successors: Vec<(BasicBlockId, ValueId)>,
839            uncovered_terminal: bool,
840        }
841
842        // Phase 1: build the backward threading graph.
843        let mut graph: HashMap<(BasicBlockId, ValueId), Node> = HashMap::default();
844        let mut worklist: Vec<(BasicBlockId, ValueId, Option<usize>)> =
845            vec![(array_set_block, source, Some(array_set_idx))];
846
847        while let Some((block, value, seed_idx)) = worklist.pop() {
848            if graph.contains_key(&(block, value)) {
849                continue;
850            }
851
852            // Wall: an iteration-local `make_array`/`Call` result is newly
853            // allocated storage on this path, distinct each iteration.
854            if self.is_iteration_local_fresh(value, array_set_block) {
855                graph
856                    .insert((block, value), Node { successors: vec![], uncovered_terminal: false });
857                continue;
858            }
859
860            // Wall: an `inc_rc` on the threaded value in this block.
861            if let Some(locations) = self.inc_rc_locations.get(&value)
862                && locations
863                    .iter()
864                    .any(|&(b, i)| b == block && seed_idx.is_none_or(|limit| i < limit))
865            {
866                graph
867                    .insert((block, value), Node { successors: vec![], uncovered_terminal: false });
868                continue;
869            }
870
871            // If `value` is defined in this block, the walk stops here.
872            if let Some(&(def_block, def_idx)) = self.array_value_defs.get(&value)
873                && def_block == block
874            {
875                let inst_id = self.function.dfg[block].instructions()[def_idx];
876                // An `array_set` result shares its operand's storage; continue
877                // threading with the operand.
878                if let Instruction::ArraySet { array, .. } = self.function.dfg[inst_id] {
879                    graph.insert(
880                        (block, value),
881                        Node { successors: vec![(block, array)], uncovered_terminal: false },
882                    );
883                    worklist.push((block, array, seed_idx));
884                    continue;
885                }
886                // A non-iteration-local allocation or other instruction we
887                // cannot thread: an uncovered terminal.
888                graph.insert((block, value), Node { successors: vec![], uncovered_terminal: true });
889                continue;
890            }
891
892            // Flows in from predecessors.
893            let preds: Vec<BasicBlockId> = self.cfg.predecessors(block).collect();
894            if preds.is_empty() {
895                // Reached the entry / a source-less block unwalled.
896                graph.insert((block, value), Node { successors: vec![], uncovered_terminal: true });
897                continue;
898            }
899            // If `value` is a parameter of this block, follow each predecessor's
900            // argument at that position; otherwise the same value flows in from
901            // every predecessor.
902            let params = self.function.dfg.block_parameters(block);
903            let param_pos = params.iter().position(|&p| p == value);
904            let mut successors = Vec::new();
905            let mut uncovered_terminal = false;
906            for &pred in &preds {
907                let next = match param_pos {
908                    Some(i) => match self.edge_arg(pred, block, i) {
909                        Some(arg) => arg,
910                        // An unresolvable edge argument: conservatively treat
911                        // as uncovered rather than silently dropping the path.
912                        None => {
913                            uncovered_terminal = true;
914                            continue;
915                        }
916                    },
917                    None => value,
918                };
919                successors.push((pred, next));
920                worklist.push((pred, next, None));
921            }
922            graph.insert((block, value), Node { successors, uncovered_terminal });
923        }
924
925        // Phase 2: propagate "uncovered" from terminals to a fixed point.
926        let mut uncovered: HashSet<(BasicBlockId, ValueId)> =
927            graph.iter().filter(|(_, n)| n.uncovered_terminal).map(|(k, _)| *k).collect();
928        let mut changed = true;
929        while changed {
930            changed = false;
931            for (state, node) in &graph {
932                if !uncovered.contains(state)
933                    && node.successors.iter().any(|s| uncovered.contains(s))
934                {
935                    uncovered.insert(*state);
936                    changed = true;
937                }
938            }
939        }
940
941        uncovered.into_iter().map(|(_, value)| value).collect()
942    }

944    /// The argument passed to parameter position `i` of `dest` on the edge
945    /// from `pred`. `None` if `pred`'s terminator does not target `dest` or has
946    /// no argument at that position.
947    fn edge_arg(&self, pred: BasicBlockId, dest: BasicBlockId, i: usize) -> Option<ValueId> {
948        match self.function.dfg[pred].terminator()? {
949            TerminatorInstruction::Jmp { destination, arguments, .. } if *destination == dest => {
950                arguments.get(i).copied()
951            }
952            TerminatorInstruction::JmpIf {
953                then_destination,
954                then_arguments,
955                else_destination,
956                else_arguments,
957                ..
958            } => {
959                if *then_destination == dest {
960                    then_arguments.get(i).copied()
961                } else if *else_destination == dest {
962                    else_arguments.get(i).copied()
963                } else {
964                    None
965                }
966            }
967            _ => None,
968        }
969    }

971    /// Forward CFG walk from after the `array_set` looking for a
972    /// non-terminator instruction that reads a value still in the alias
973    /// use-set.
974    ///
975    /// **Use-set evolution.** Starts as `alias_set` — which is **already
976    /// narrowed** to the uncovered members by [`Context::unprotected_aliases`]
977    /// before this is called, so provably-protected aliases never enter the
978    /// walk — and only shrinks from there. Kills are applied **during
979    /// propagation** to each successor — see [`Context::succ_use_set`] for the
980    /// kill rules.
981    ///
982    /// **What counts as a use.** Only non-terminator operands. Terminator
983    /// arguments are the legitimate threading mechanism the invariant
984    /// relies on; `jmp b(v_alias)` is how the post-mutation value reaches
985    /// the next block where it is re-bound to that block's parameter. The
986    /// kill logic already accounts for these args.
987    ///
988    /// The original `array_set` itself is also skipped, in case a cycle
989    /// re-enters its block — it is, by construction, a use of its own
990    /// source, not a hazard.
991    ///
992    /// **Chain-aware index filter.** Two auxiliary sets evolve alongside the
993    /// `use_set` to make the filter sound in the presence of `array_set`
994    /// chains:
995    ///
996    /// - **`derived`** tracks values that may share the `array_set`'s source
997    ///   storage at runtime through transitive in-place mutations. Seeded
998    ///   with the `array_set`'s own result; grown on every later `array_set`
999    ///   whose `array` operand is already in `derived`; propagated across
1000    ///   block-param edges the same way the alias use-set is.
1001    /// - **`tainted_indices`** tracks the set of storage positions any
1002    ///   chain link may have already written. `Some(set)` accumulates
1003    ///   constant write indices (seeded with `write_index_const`); set to
1004    ///   `None` (= "all positions") as soon as any chain write uses a
1005    ///   dynamic index.
1006    ///
1007    /// When we encounter `array_get v, idx` with `v ∈ use_set`, the read
1008    /// is a hazard iff `tainted_indices` covers `idx`. With both indices
1009    /// constant, that's a set-membership test; otherwise (either side
1010    /// dynamic, or `tainted_indices == None`) the verifier conservatively
1011    /// flags. An `array_set v, idx, _` with `v ∈ use_set` is index-aware in
1012    /// the same way: it copies forward every position except `idx`, so it
1013    /// is a hazard iff some `tainted_indices` position differs from `idx`
1014    /// (a same-index write overwrites the mutation and extends the chain
1015    /// instead). All other uses on a `use_set` member are always flagged —
1016    /// the SSA-vs-runtime divergence isn't index-local for them.
1017    ///
1018    /// **`IncrementRc` clears `derived`.** A program-point `inc_rc v` with
1019    /// `v ∈ use_set ∪ derived` lifts the storage's RC ≥ 2, so any
1020    /// subsequent `array_set` on a chain participant runs on fresh
1021    /// storage and can no longer clobber the source. We clear `derived`
1022    /// at that point. `tainted_indices` is *not* cleared — past damage
1023    /// to the shared storage is still observable through `use_set` reads.
1024    ///
1025    /// **Cycle detection.** Re-entering a block with a state that's a
1026    /// (`use_set`, derived, tainted_indices)-subset of the frontier we've
1027    /// already explored adds no new information. Each component is
1028    /// unioned into the frontier on entry; for `tainted_indices`, `None`
1029    /// is the absorbing element (a previously-`None` frontier covers any
1030    /// re-entry).
1031    fn find_reachable_aliased_use(
1032        &self,
1033        alias_set: &im::HashSet<ValueId>,
1034        source: ValueId,
1035        array_set_id: InstructionId,
1036        array_set_block: BasicBlockId,
1037        array_set_idx: usize,
1038        write_index_const: Option<FieldElement>,
1039    ) -> Option<AliasedUse> {
1040        let mut visited: HashMap<BasicBlockId, WalkState> = HashMap::default();
1041
1042        // Protected-participant set. Exclude a non-source alias `v` when
1043        // all three hold:
1044        //
1045        // 1. `v ∈ alias_set` — `v` is in `source`'s backward set, i.e. it
1046        //    actually *flows into* `source`. This is the soundness guard:
1047        //    a value that only *receives* `source`'s storage (a forward
1048        //    successor) is not in the backward set, so its `inc_rc` runs
1049        //    *after* the array_set and cannot protect it. Requiring
1050        //    membership keeps such values in the use-set to be flagged.
1051        // 2. `v` carries its own `inc_rc` — combined with (3), the bump
1052        //    runs before `v` crosses the back-edge that re-binds it onto
1053        //    `source`'s loop-header parameter, so by the time `v`'s
1054        //    storage equals `source`'s it is RC ≥ 2 and the array_set
1055        //    copies rather than mutating in place.
1056        // 3. `v` is a loop back-edge participant
1057        //    ([`Context::back_edge_participants`]) — its storage reaches
1058        //    `source` through a back-edge (possibly via a forward edge
1059        //    into the back-edge arg first), which is what makes the
1060        //    `inc_rc` loop-carried.
1061        //
1062        // The gate is **per value** on that value's own `inc_rc` — a
1063        // sibling's `inc_rc` never exonerates an unprotected value. A
1064        // back-edge position fed by an `inc_rc`'d value on one predecessor
1065        // edge and an unprotected value on another protects only the
1066        // former; the latter stays in the use-set for the walk to flag.
1067        //
1068        // The removal is **sticky**: protected members are kept out of the
1069        // use-set for the whole walk, not just the initial seed. A
1070        // protected participant that is itself a loop-header parameter
1071        // would otherwise be re-introduced the moment another in-use alias
1072        // flows into its position (the add-rule in
1073        // [`Context::succ_use_set`]); excluding it there too keeps it out.
1074        // Dropping it from the seed also lets the per-arg kill rule drop
1075        // the loop-header parameter on the back-edge, since the value
1076        // threaded back is this protected participant rather than a
1077        // still-live alias.
1078        let protected: im::HashSet<ValueId> = alias_set
1079            .iter()
1080            .copied()
1081            .filter(|&v| {
1082                v != source
1083                    && self.inc_rc_locations.contains_key(&v)
1084                    && self.back_edge_participants.contains(&v)
1085            })
1086            .collect();
1087        let use_set: im::HashSet<ValueId> =
1088            alias_set.iter().copied().filter(|v| !protected.contains(v)).collect();
1089
1090        let array_set_result = self.function.dfg.instruction_results(array_set_id)[0];
1091        let initial_state = WalkState {
1092            use_set,
1093            derived: im::HashSet::unit(array_set_result),
1094            tainted: write_index_const.map(im::HashSet::unit),
1095        };
1096        let mut worklist: Vec<WalkFrame> = vec![WalkFrame {
1097            block: array_set_block,
1098            start_idx: array_set_idx + 1,
1099            state: initial_state,
1100        }];
1101
1102        while let Some(WalkFrame { block, start_idx, mut state }) = worklist.pop() {
1103            // No alias-set members in use means no read of this array_set's
1104            // source's storage can surface a hazard from here on.
1105            if state.use_set.is_empty() {
1106                continue;
1107            }
1108
1109            // Cycle/redundancy check + bookkeeping only applies to *full*
1110            // block entries (start_idx == 0). The very first frame of the
1111            // walk starts mid-block and only covers a suffix of the block —
1112            // recording it here would incorrectly let a later back-edge
1113            // entry to the same block skip the unexplored prefix.
1114            if start_idx == 0 {
1115                if let Some(prev) = visited.get(&block)
1116                    && state.is_covered_by(prev)
1117                {
1118                    continue;
1119                }
1120                let new_frontier = match visited.get(&block) {
1121                    Some(prev) => state.merge(prev),
1122                    None => state.clone(),
1123                };
1124                visited.insert(block, new_frontier);
1125            }
1126
1127            let instructions = self.function.dfg[block].instructions();
1128            for &inst_id in instructions.iter().skip(start_idx) {
1129                if inst_id == array_set_id {
1130                    continue;
1131                }
1132                let inst = &self.function.dfg[inst_id];
1133
1134                match inst {
1135                    // `inc_rc` on an alias or chain-derived value lifts the
1136                    // shared storage's RC; subsequent chain writes run on
1137                    // fresh storage and stop tainting the source. Past
1138                    // tainted indices survive — the damage is already done.
1139                    Instruction::IncrementRc { value } => {
1140                        if state.derived.contains(value) || state.use_set.contains(value) {
1141                            state.derived.clear();
1142                        }
1143                    }
1144                    // Well-formed Brillig SSA shouldn't contain dec_rc;
1145                    // skip if encountered.
1146                    Instruction::DecrementRc { .. } => {}
1147                    Instruction::ArraySet { array, index, value, .. } => {
1148                        let array_in_use = state.use_set.contains(array);
1149                        if array_in_use {
1150                            // Index-aware, mirroring the `array_get` rule below.
1151                            // `array_set v, i, x` produces a *copy* of `v` with
1152                            // index `i` overwritten, so it only observes (copies
1153                            // forward) the indices it does **not** write. It can
1154                            // therefore surface the source's in-place mutation
1155                            // only if it copies a tainted index — i.e. some
1156                            // tainted index differs from this write index. A
1157                            // write to the (sole) tainted index overwrites the
1158                            // mutation and observes nothing. A dynamic write or
1159                            // read index, or fully-tainted (`None`) storage, is
1160                            // flagged conservatively.
1161                            let write_idx = self.function.dfg.get_numeric_constant(*index);
1162                            let observes_tainted = match (&state.tainted, write_idx) {
1163                                (None, _) | (_, None) => true,
1164                                (Some(t), Some(c)) => t.iter().any(|i| *i != c),
1165                            };
1166                            if observes_tainted {
1167                                return Some(AliasedUse { instruction: inst_id, value: *array });
1168                            }
1169                        }
1170                        if self.function.dfg.type_of_value(*value).contains_an_array()
1171                            && state.use_set.contains(value)
1172                        {
1173                            return Some(AliasedUse { instruction: inst_id, value: *value });
1174                        }
1175                        // Chain extension: a write through a value that may
1176                        // share the source's storage — a `derived` member, or a
1177                        // `use_set` member whose write we just cleared as
1178                        // non-observing — keeps mutating that storage, so record
1179                        // its write index and track the result.
1180                        if state.derived.contains(array) || array_in_use {
1181                            match (
1182                                state.tainted.as_mut(),
1183                                self.function.dfg.get_numeric_constant(*index),
1184                            ) {
1185                                (Some(t), Some(c)) => {
1186                                    t.insert(c);
1187                                }
1188                                _ => {
1189                                    state.tainted = None;
1190                                }
1191                            }
1192                            let [result] = self.function.dfg.instruction_result(inst_id);
1193                            state.derived.insert(result);
1194                        }
1195                    }
1196                    Instruction::ArrayGet { array, index, .. } => {
1197                        if state.use_set.contains(array) {
1198                            let read_idx = self.function.dfg.get_numeric_constant(*index);
1199                            let hazard = match (&state.tainted, read_idx) {
1200                                (None, _) | (_, None) => true,
1201                                (Some(set), Some(c)) => set.contains(&c),
1202                            };
1203                            if hazard {
1204                                return Some(AliasedUse { instruction: inst_id, value: *array });
1205                            }
1206                        }
1207                    }
1208                    other => {
1209                        // Any other instruction reading an alias-set member
1210                        // through an array-typed operand is a hazard. The
1211                        // SSA-vs-runtime divergence isn't index-local for
1212                        // these instruction kinds.
1213                        let mut hit: Option<ValueId> = None;
1214                        other.for_each_value(|v| {
1215                            if hit.is_some() {
1216                                return;
1217                            }
1218                            if self.function.dfg.type_of_value(v).contains_an_array()
1219                                && state.use_set.contains(&v)
1220                            {
1221                                hit = Some(v);
1222                            }
1223                        });
1224                        if let Some(v) = hit {
1225                            return Some(AliasedUse { instruction: inst_id, value: v });
1226                        }
1227                    }
1228                }
1229            }
1230
1231            let Some(terminator) = self.function.dfg[block].terminator() else { continue };
1232            match terminator {
1233                TerminatorInstruction::Jmp { destination, arguments, .. } => {
1234                    let next = self.succ_walk_state(*destination, arguments, &state, &protected);
1235                    worklist.push(WalkFrame { block: *destination, start_idx: 0, state: next });
1236                }
1237                TerminatorInstruction::JmpIf {
1238                    then_destination,
1239                    then_arguments,
1240                    else_destination,
1241                    else_arguments,
1242                    ..
1243                } => {
1244                    let then_state =
1245                        self.succ_walk_state(*then_destination, then_arguments, &state, &protected);
1246                    worklist.push(WalkFrame {
1247                        block: *then_destination,
1248                        start_idx: 0,
1249                        state: then_state,
1250                    });
1251                    let else_state =
1252                        self.succ_walk_state(*else_destination, else_arguments, &state, &protected);
1253                    worklist.push(WalkFrame {
1254                        block: *else_destination,
1255                        start_idx: 0,
1256                        state: else_state,
1257                    });
1258                }
1259                TerminatorInstruction::Return { .. }
1260                | TerminatorInstruction::Unreachable { .. } => (),
1261            }
1262        }
1263
1264        None
1265    }
1266
1267    /// Propagate the walk state across a block-parameter edge. `use_set`
1268    /// and `derived` follow the same kill/add rules ([`Context::succ_use_set`]);
1269    /// `tainted` is carried unchanged, since it tracks storage positions
1270    /// (not SSA values).
1271    fn succ_walk_state(
1272        &self,
1273        dest: BasicBlockId,
1274        arguments: &[ValueId],
1275        state: &WalkState,
1276        protected: &im::HashSet<ValueId>,
1277    ) -> WalkState {
1278        WalkState {
1279            use_set: self.succ_use_set(dest, arguments, &state.use_set, protected),
1280            derived: self.succ_use_set(dest, arguments, &state.derived, protected),
1281            tainted: state.tainted.clone(),
1282        }
1283    }
1284
1285    /// Compute the use-set carried into `dest` when its predecessor jumps
1286    /// with `arguments`.
1287    ///
1288    /// Three rules apply, in order:
1289    ///
1290    /// 1. **Per-arg propagation — block parameters of `dest`.** For each
1291    ///    `dest.params[i]`, look at the corresponding `arguments[i]`:
1292    ///    - **Kill.** If the param is in `use_set` and the arg is not, the
1293    ///      param is rebound to a value that no longer aliases the
1294    ///      `array_set`'s source (most commonly the `array_set`'s own
1295    ///      result, excluded at lookup time): drop it.
1296    ///    - **Add.** If the param is not in `use_set` but the arg is,
1297    ///      this edge introduces a fresh alias: the param at `dest`'s
1298    ///      entry shares storage with an alias-set member, so add it —
1299    ///      *unless* the param is a protected back-edge participant
1300    ///      ([`Context::find_reachable_aliased_use`]), which must stay out
1301    ///      of the use-set for the whole walk (its own `inc_rc` keeps it
1302    ///      RC ≥ 2). Without this exclusion a protected participant that
1303    ///      is itself a loop-header parameter would be re-introduced the
1304    ///      moment any in-use alias flowed into its position.
1305    ///    - Otherwise (both in or both out), no change.
1306    ///
1307    ///    Only array-typed params participate.
1308    ///
1309    /// 2. **Unconditional kill — instructions defined in `dest`.** For
1310    ///    each alias-set member whose defining block is `dest`: drop it.
1311    ///    Re-entering `dest` re-executes the defining instruction (e.g.
1312    ///    `load`, `make_array`), producing a fresh runtime value; the
1313    ///    previous iteration's storage is no longer reachable through that
1314    ///    name.
1315    fn succ_use_set(
1316        &self,
1317        dest: BasicBlockId,
1318        arguments: &[ValueId],
1319        use_set: &im::HashSet<ValueId>,
1320        protected: &im::HashSet<ValueId>,
1321    ) -> im::HashSet<ValueId> {
1322        let mut result = use_set.clone();
1323
1324        // (1) Per-arg propagation for params of `dest`.
1325        let params = self.function.dfg.block_parameters(dest);
1326        for (i, &param) in params.iter().enumerate() {
1327            if !self.function.dfg.type_of_value(param).contains_an_array() {
1328                continue;
1329            }
1330            let arg_in_use_set = arguments.get(i).is_some_and(|arg| use_set.contains(arg));
1331            let param_in_use_set = use_set.contains(&param);
1332            match (param_in_use_set, arg_in_use_set) {
1333                (true, false) => {
1334                    result.remove(&param);
1335                }
1336                (false, true) if !protected.contains(&param) => {
1337                    result.insert(param);
1338                }
1339                _ => {}
1340            }
1341        }
1342
1343        // (2) Unconditional kill for instruction-defined values whose
1344        // def-block is `dest` (re-execution on cycle re-entry produces a
1345        // fresh value).
1346        for &v in use_set {
1347            if self.array_value_defs.get(&v).map(|(b, _)| *b) == Some(dest) {
1348                result.remove(&v);
1349            }
1350        }
1351
1352        result
1353    }
1354}

1356/// Compute, for each array-typed block parameter, the set of values that
1357/// may share its storage at the parameter's binding — itself plus every
1358/// value that flows into it through some chain of predecessor →
1359/// block-parameter edges.
1360///
1361/// Runs a fixed-point iteration in reverse-post-order: each block
1362/// parameter starts with `{p}`, and each pass unions in `args[p_index]`
1363/// (and its own backward set, if it's also a block parameter) for
1364/// every incoming edge. The iteration stops when no set grows.
1365///
1366/// Visiting blocks in RPO means each parameter sees its forward-edge
1367/// predecessors' sets *already at their fixed point* (RPO orders
1368/// predecessor before successor along forward edges), so the only
1369/// source of additional passes is loop back-edges. In practice this
1370/// converges in 1 pass for loop-free code and 2-3 for typical
1371/// loops — versus O(chain-depth) passes under arbitrary block order.
1372///
1373/// # Why the walk is directed (param ← arg), not symmetric
1374///
1375/// A symmetric closure over `(param, arg)` pairs would put two sibling
1376/// args passed to the same block in the same class via the shared
1377/// parameter, even though at runtime they never refer to the same
1378/// storage (each runtime path takes only one of them). Following the
1379/// directed `param ← arg` edges instead means sibling parameters don't
1380/// co-mingle in each other's sets — only the predecessors that flow
1381/// *into* a given parameter do.
1382///
1383/// # Why we deliberately do **not** chase past `array_set` results
1384///
1385/// At runtime the result may or may not alias the source depending on
1386/// RC. The verifier's job is to flag the cases where it *might*. We
1387/// model this by *excluding* the result from each alias-set at lookup
1388/// time (in [`Context::alias_set_for`]) rather than refusing to chase
1389/// through it here. Otherwise a chain
1390/// `v1 = array_set v0 ; v2 = array_set v1 ; v3 = array_set v2` would
1391/// pull `v0` into the source of every later `array_set`, and an
1392/// `inc_rc v0` that legitimately protects only `v1`'s `array_set` would
1393/// falsely appear to protect `v3` as well. See
1394/// `alias_set_does_not_walk_array_set_chains` for a worked example —
1395/// the chain isn't visible to the backward walk anyway because
1396/// `v1`, `v2`, `v3` are instruction results (not block parameters),
1397/// so there's nothing to chase from `v3`.
1398fn compute_backward_aliases(
1399    function: &Function,
1400    rpo: &[BasicBlockId],
1401    incoming_edges: &HashMap<BasicBlockId, Vec<(BasicBlockId, Vec<ValueId>)>>,
1402) -> HashMap<ValueId, im::HashSet<ValueId>> {
1403    let mut result: HashMap<ValueId, im::HashSet<ValueId>> = HashMap::default();
1404
1405    // Seed each array-typed block parameter with `{p}`.
1406    for &block_id in rpo {
1407        for &param in function.dfg.block_parameters(block_id) {
1408            if function.dfg.type_of_value(param).contains_an_array() {
1409                result.insert(param, im::HashSet::unit(param));
1410            }
1411        }
1412    }
1413
1414    // Fixed-point in RPO: keep iterating until no parameter's set grows.
1415    // Each pass folds in args from every incoming edge, transitively
1416    // through any predecessor arg that's itself a block parameter.
1417    let mut changed = true;
1418    while changed {
1419        changed = false;
1420        for &block_id in rpo {
1421            let params = function.dfg.block_parameters(block_id);
1422            let Some(incoming) = incoming_edges.get(&block_id) else {
1423                continue;
1424            };
1425            for (i, &param) in params.iter().enumerate() {
1426                if !function.dfg.type_of_value(param).contains_an_array() {
1427                    continue;
1428                }
1429                let mut new_set = result.get(&param).cloned().unwrap_or_else(im::HashSet::new);
1430                let prev_len = new_set.len();
1431                for (_pred, args) in incoming {
1432                    if let Some(&arg) = args.get(i) {
1433                        // Pull in `arg`'s backward set if it's a block
1434                        // parameter; otherwise the arg is an instruction
1435                        // result whose backward set is the singleton.
1436                        if let Some(arg_set) = result.get(&arg) {
1437                            new_set.extend(arg_set.iter().copied());
1438                        } else {
1439                            new_set.insert(arg);
1440                        }
1441                    }
1442                }
1443                if new_set.len() > prev_len {
1444                    result.insert(param, new_set);
1445                    changed = true;
1446                }
1447            }
1448        }
1449    }
1450
1451    result
1452}

1454/// Per-frame state of the forward reachable-use walk: which alias-set
1455/// members are live, which chain-derived values share storage with the
1456/// source, and which storage positions any chain link may already have
1457/// clobbered. Also serves as the per-block visited frontier for cycle
1458/// detection — see [`WalkState::is_covered_by`] / [`WalkState::merge`].
1459#[derive(Clone)]
1460struct WalkState {
1461    /// Alias-set members live at this point: values that may share the
1462    /// `array_set`'s source storage *at the `array_set`'s program point*. A
1463    /// non-terminator read of one of these is the hazard the walk is
1464    /// looking for.
1465    use_set: im::HashSet<ValueId>,
1466    /// Values that may share the source's storage through transitive
1467    /// in-place chain mutations. Seeded with the `array_set`'s own result;
1468    /// extended by every later `array_set` whose `array` operand is
1469    /// already in `derived`.
1470    derived: im::HashSet<ValueId>,
1471    /// Storage positions any chain link may already have written. `None`
1472    /// (= "all positions") absorbs a dynamic chain write — once we lose
1473    /// precise tracking we can't recover it.
1474    tainted: Option<im::HashSet<FieldElement>>,
1475}
1476
1477impl WalkState {
1478    /// `self` is covered by `prev` when every potential hazard `self`
1479    /// could surface is also reachable from `prev`. For `use_set` and
1480    /// `derived` that's the subset relation; for `tainted`, `None`
1481    /// (= "all positions") absorbs as the upper bound — a previously
1482    /// `None` frontier covers any re-entry, but a current `None`
1483    /// requires `prev` also to be `None`.
1484    fn is_covered_by(&self, prev: &Self) -> bool {
1485        if !self.use_set.is_subset(&prev.use_set) || !self.derived.is_subset(&prev.derived) {
1486            return false;
1487        }
1488        match (&prev.tainted, &self.tainted) {
1489            (None, _) => true,
1490            (_, None) => false,
1491            (Some(p), Some(c)) => c.is_subset(p),
1492        }
1493    }
1494
1495    /// Component-wise union — the frontier we record after visiting a
1496    /// block, so future re-entries can be cycle-checked against
1497    /// everything we've already explored from that block.
1498    fn merge(&self, other: &Self) -> Self {
1499        Self {
1500            use_set: self.use_set.clone().union(other.use_set.clone()),
1501            derived: self.derived.clone().union(other.derived.clone()),
1502            tainted: match (&self.tainted, &other.tainted) {
1503                (None, _) | (_, None) => None,
1504                (Some(a), Some(b)) => Some(a.clone().union(b.clone())),
1505            },
1506        }
1507    }
1508}

1510/// One entry on the forward reachable-use walk's worklist: a block to
1511/// enter, the instruction index to start at within that block, and the
1512/// walk's evolving state. `start_idx > 0` denotes the very first frame,
1513/// which continues inside the `array_set`'s own block past the `array_set`
1514/// instruction itself; all later frames enter at block start.
1515struct WalkFrame {
1516    block: BasicBlockId,
1517    start_idx: usize,
1518    state: WalkState,
1519}

1521/// A non-terminator instruction reachable forward from an `array_set` that
1522/// reads a value still in the alias-set — the *aliased use* that the
1523/// reachable-use walk surfaced. Carries both pieces so callers can build a
1524/// diagnostic that names the offending alias value and the instruction
1525/// that observed it.
1526#[derive(Debug)]
1527struct AliasedUse {
1528    /// The instruction that uses the aliased value as a (non-terminator)
1529    /// operand.
1530    instruction: InstructionId,
1531    /// The alias-set member that was used. Names *which* alias triggered
1532    /// the flag — useful when the alias-set has more than one member.
1533    value: ValueId,
1534}

1536/// Test support shared by both submodules' `tests`. The reject helpers stay
1537/// per-module (each asserts its own error variant), but acceptance is the same
1538/// property for both — *neither* verifier rejects — so it lives here.
1539#[cfg(test)]
1540pub(crate) mod tests {
1541    use super::{array_set, call};
1542    use crate::{errors::RtResult, ssa::ssa_gen::Ssa};
1543
1544    /// Run the full `rc_invariant` check — every submodule verifier — over
1545    /// `ssa`, returning the first violation.
1546    pub(crate) fn verify_all(ssa: &Ssa) -> RtResult<()> {
1547        array_set::verify(ssa)?;
1548        call::verify(ssa)?;
1549        Ok(())
1550    }
1551
1552    /// Assert the full `rc_invariant` check ([`verify_all`]) accepts `src`.
1553    pub(crate) fn assert_verifier_accepts(src: &str) {
1554        assert_verifier_accepts_because(src, "");
1555    }
1556
1557    /// Same as [`assert_verifier_accepts`] but includes `reason` in the panic
1558    /// message — useful for documenting why the SSA is *expected* to be
1559    /// accepted (e.g. "loop exit reads a rebound block-param").
1560    pub(crate) fn assert_verifier_accepts_because(src: &str, reason: &str) {
1561        let ssa = Ssa::from_str(src).expect("SSA parses");
1562        if let Err(err) = verify_all(&ssa) {
1563            if reason.is_empty() {
1564                panic!("expected the verifier to accept, but it rejected: {err:?}");
1565            } else {
1566                panic!("expected the verifier to accept ({reason}), but it rejected: {err:?}");
1567            }
1568        }
1569    }
