1//! Verifies the implicit invariant that Brillig SSA must satisfy around
2//! `array_set` and reference counts.
3//!
4//! # The invariant
5//!
6//! In Brillig, `array_set vX, i, x` may modify `vX`'s storage in place at runtime
7//! when `vX`'s reference count is 1. SSA-level semantics still says `vX` is unchanged
8//! and the array_set produces a fresh value; the in-place mutation is a runtime
9//! optimization that's only sound when no later use can observe `vX`'s pre-mutation
10//! contents through aliasing.
11//!
12//! Two mechanisms keep the optimization invisible to SSA semantics:
13//!
14//! 1. **`inc_rc`** before the `array_set` forces RC ≥ 2 at runtime so `array_set`
15//!    copies rather than mutating in place.
16//! 2. **Block-parameter threading** routes the post-mutation value forward as a new
17//!    SSA value (the `array_set`'s result), so no later instruction references
18//!    `vX` after the mutation.
19//!
20//! The frontend uses whichever mechanism the program needs. This pass verifies
21//! that one of them is in place for every `array_set` whose source has an
22//! aliased use reachable forward in the CFG.
23//!
24//! # Algorithm
25//!
26//! For each `array_set vX, …`:
27//!
28//! 1. **Backward alias-set.** Compute the set of values that may share `vX`'s
29//!    storage *at the array_set's program point* by walking block-parameter →
30//!    predecessor-arg edges backward to a fixed point. Only aliasing introduced
31//!    by the values that flow *into* `vX`'s binding is included. Filtered to
32//!    drop values that can't represent pre-mutation storage: `array_set` /
33//!    `Call` results (always fresh), iteration-local fresh results (`MakeArray`
34//!    or `Call` on back-edge args), instruction results defined in the
35//!    array_set's own block at an index after the array_set (they can land in
36//!    the backward set through a forward-then-back round-trip, but don't exist
37//!    as storage at the array_set's program point yet), and **swap-excluded
38//!    siblings** — when `vX` is a loop-header parameter swapped onto a sibling
39//!    parameter that is freshly re-allocated each iteration, the sibling is a
40//!    distinct per-iteration storage and is dropped (see
41//!    [`Context::swap_excluded_aliases`]).
42//! 2. **inc_rc precedence / back-edge-participant.** If some `inc_rc` on an
43//!    alias-set member either RPO-precedes the array_set or sits on a
44//!    non-source alias that's also a loop back-edge arg, accept — the
45//!    frontend is deliberately managing iteration aliasing.
46//! 3. **Protected-participant filter.** Before the forward walk, drop from
47//!    the use-set every alias-set member (other than the source) that both
48//!    carries its own `inc_rc` and is a loop back-edge *participant* (it
49//!    flows — directly or through forward edges — into a back-edge arg
50//!    position). Being in the alias-set means the value flows *into* the
51//!    source, so combined with its back-edge participation the `inc_rc` is
52//!    loop-carried: it runs before the value crosses the back-edge that
53//!    re-binds it onto the source, so by the time the value's storage
54//!    equals the source's it is RC ≥ 2 and the array_set copies. Reads of
55//!    it therefore can't observe an in-place mutation. The gate is **per
56//!    value** on that value's own `inc_rc`: a back-edge position fed by an
57//!    `inc_rc`'d value on one predecessor edge and an unprotected value on
58//!    another drops only the protected value, leaving the unprotected one
59//!    for the walk to flag. Restricting to alias-set members is the
60//!    soundness guard — a value that merely *receives* the source's storage
61//!    (a forward successor, not in the source's backward set) has its
62//!    `inc_rc` run *after* the array_set and so is not protected; it stays
63//!    in the use-set. This is what lets the verifier accept the latch-block
64//!    shape (an `inc_rc v` placed before `v` is threaded *forward* into the
65//!    latch that then closes the loop) without the unsoundness of crediting
66//!    an `inc_rc` to a value the array_set actually mutates first.
67//! 4. **Forward walk.** Otherwise, walk the CFG forward from the array_set with
68//!    the (filtered) alias-set as the initial use-set. At each block-parameter
69//!    crossing we both **kill** params that the predecessor rebinds to a
70//!    non-alias and **add** params whose arg is still an alias (so alias
71//!    propagation stays accurate as the walk crosses joins and loops). The
72//!    walk maintains two additional pieces of state:
73//!
74//!    - **`derived`**, the set of values that may share the source's storage
75//!      through transitive in-place chain mutations. Seeded with the
76//!      array_set's own result; extended by every later `array_set` whose
77//!      `array` operand is already in `derived`.
78//!    - **`tainted_indices`**, the storage positions any chain link may
79//!      already have written. Seeded with the array_set's own write index
80//!      (when constant) and widened by chain-link writes; collapsed to "all
81//!      positions" if any chain link uses a dynamic index.
82//!
83//!    An `array_get` on a use-set member is a hazard iff its read index is
84//!    covered by `tainted_indices`. An `array_set` on a use-set member is
85//!    index-aware too: it produces a copy of the source with one index
86//!    overwritten, so it observes (copies forward) only the indices it does
87//!    *not* write — a hazard iff some `tainted_indices` position differs
88//!    from its write index (a same-index write overwrites the mutation and
89//!    is not a hazard; it instead extends the chain). All other uses are
90//!    always hazards. An `inc_rc v` with `v ∈ use_set ∪ derived` lifts the
91//!    storage's RC,
92//!    so chain writes after it run on fresh storage — `derived` is cleared
93//!    at that point. `tainted_indices` is *not* cleared: prior chain writes
94//!    have already mutated the source's storage.
95//!
96//! # Precondition
97//!
98//! Must be run **after `mem2reg_brillig`**. The alias walk through
99//! block-parameter edges only reflects post-mem2reg aliasing; pre-mem2reg, an
100//! `Allocate`/`Store`/`Load` chain would route aliasing through references that
101//! this pass does not track.
102//!
103//! # Known limitations
104//!
105//! The verifier walks aliasing *only* through block-parameter edges. The
106//! following are not tracked and are documented gaps:
107//!
108//! - **Sibling-args-of-same-value.** A `jmp b1(v, v)` makes `b1`'s two
109//!   parameters runtime-equal, but the backward walk doesn't see this — each
110//!   parameter's backward set includes `v` but not the other parameter. The
111//!   `end_to_end_sibling_args_to_same_value_is_false_negative` and
112//!   `end_to_end_sibling_args_across_jmp_is_false_negative` tests pin this
113//!   down as documented false negatives.
114//! - **Nested-array `MakeArray`**, **`IfElse` on arrays**, **non-inlined
115//!   `Call` returns**, and **`Store`/`Load` on ineligible (nested-ref)
116//!   allocates** are likewise not tracked.

141/// Verifies the `array_set` / `inc_rc` aliasing invariant on every Brillig
142/// function in `ssa`. See the module-level docs for details.
143///
144/// The entire module containing this function is gated behind
145/// `#[cfg(debug_assertions)]`, so it is a no-op (and absent at the linker
146/// level) in release builds — see the pipeline wiring in
147/// [`crate::ssa::primary_passes`].
148pub(crate) fn verify_array_set_rc_invariant(ssa: &Ssa) -> RtResult<()> {
149    for function in ssa.functions.values() {
150        verify_function(function)?;
151    }
152    Ok(())
153}
154
155/// Per-function entry point. Skips ACIR functions (the invariant only
156/// applies to Brillig, where `array_set` may mutate in place) and runs the
157/// alias / dominance / reachable-use checks for every `array_set` in the
158/// function.
159fn verify_function(function: &Function) -> RtResult<()> {
160    if !function.runtime().is_brillig() {
161        return Ok(());
162    }
163
164    let ctx = Context::new(function);
165
166    for block_id in function.reachable_blocks() {
167        for (idx, instruction_id) in function.dfg[block_id].instructions().iter().enumerate() {
168            let Instruction::ArraySet { array, index: write_index, .. } =
169                function.dfg[*instruction_id]
170            else {
171                continue;
172            };
173
174            let alias_set = ctx.alias_set_for(array, block_id, idx);
175
176            // Cheap check first: if any `inc_rc` on an alias-set member
177            // precedes this `array_set` in flow order, treat the aliasing
178            // as already protected. See `some_inc_rc_precedes` for the
179            // rationale (relaxed from dominance to RPO precedence).
180            if ctx.some_inc_rc_precedes(&alias_set, array, block_id, idx) {
181                continue;
182            }
183
184            // The array_set's index, if constant — lets the walk skip
185            // `array_get`s on disjoint constant indices (the pedersen-style
186            // pattern). A dynamic write index means we must conservatively
187            // flag every aliased read.
188            let write_index_const = function.dfg.get_numeric_constant(write_index);
189
190            // Expensive: forward CFG walk looking for an aliased read.
191            // A hit means the `array_set` may mutate storage in place
192            // (RC=1) and a downstream instruction will observe the
193            // pre-mutation contents through an aliased name.
194            if let Some(hit) = ctx.find_reachable_aliased_use(
195                &alias_set,
196                array,
197                *instruction_id,
198                block_id,
199                idx,
200                write_index_const,
201            ) {
202                let call_stack = function.dfg.get_instruction_call_stack(*instruction_id);
203                let aliased_use_call_stack =
204                    function.dfg.get_instruction_call_stack(hit.instruction);
205                let message = format!(
206                    "array_set in function {} on array {array} has an aliased read of {} on a \
207                     forward path with no preceding `inc_rc`; the in-place mutation would be \
208                     observable through that alias",
209                    function.name(),
210                    hit.value,
211                );
212                return Err(RuntimeError::ArraySetAliasViolation {
213                    message,
214                    call_stack,
215                    aliased_use_call_stack,
216                });
217            }
218        }
219    }
220    Ok(())
221}

223/// Pre-computed indices over a Brillig function. The verifier's per-array_set
224/// checks read from these structures rather than re-scanning the function.
225struct Context<'f> {
226    function: &'f Function,
227    /// Dominator tree. Mutable because `dominates` populates an internal cache.
228    dom_tree: DominatorTree,
229    /// For each array-typed value `V`, the set of values that may share
230    /// `V`'s storage **at `V`'s program point** — the source itself plus
231    /// anything that flows backward into it through block-parameter →
232    /// predecessor-arg chains. Computed to a fixed point.
233    ///
234    /// Values absent from this map have an implicit singleton class
235    /// `{v}` (typically instruction results that aren't passed as args
236    /// anywhere they'd matter — handled in [`Context::alias_set_for`]).
237    ///
238    /// The walk follows the directed `param ← arg` edges, so two values
239    /// that flow into different parameters of the same block are *not*
240    /// siblings of each other under backward aliasing — neither is
241    /// reachable from the other along param→arg edges. This is what
242    /// keeps path-merge over-approximation at join points out of the
243    /// alias-set.
244    backward_aliases: HashMap<ValueId, im::HashSet<ValueId>>,
245    /// For each array-typed value defined by an instruction, the
246    /// `(block, instruction-position-within-block)` of the defining
247    /// instruction. Used by the def-block-entry kill rule in
248    /// [`Context::succ_use_set`].
249    array_value_defs: HashMap<ValueId, (BasicBlockId, usize)>,
250    /// Every array-typed value that does **not** carry the pre-mutation
251    /// aliasing of an `array_set`'s source. Filtered out of every
252    /// alias-set (except when the value is the `array_set`'s own source)
253    /// — see [`Context::alias_set_for`]. Includes:
254    /// - `array_set` results: represent the *post*-mutation value of the
255    ///   source. Uses of them (or anything threaded from them through
256    ///   block params) are intentional reads, not hazards.
257    /// - `Call` results: typically fresh arrays allocated by the callee.
258    ///   Heuristic — a function that returns its input *is* a real
259    ///   alias, and filtering would miss those cases; that's a
260    ///   documented gap. In practice the frontend's array-returning
261    ///   functions allocate fresh storage.
262    non_aliasing_array_values: HashSet<ValueId>,
263    /// Back-edge args that re-allocate distinct storage every iteration:
264    /// `MakeArray` results (re-executes each iteration) and `Call` results
265    /// (the callee allocates fresh). `array_set` results are excluded —
266    /// they may mutate in place, so they aren't guaranteed-fresh. Two uses:
267    ///
268    /// - [`Context::alias_set_for`] drops such values from the alias-set:
269    ///   a back-edge that puts one in a loop-header parameter's class is
270    ///   conflating distinct runtime storages across iterations. (`Call`
271    ///   results are also dropped there by
272    ///   [`Context::non_aliasing_array_values`]; the overlap is harmless.)
273    /// - the swap freshening guard in [`Context::new`] requires the
274    ///   swapped-out sibling to be one of these — see
275    ///   [`Context::swap_excluded_aliases`].
276    ///
277    /// Values that are *not* back-edge args are kept in the alias-set:
278    /// they represent a one-time allocation whose storage the array_set
279    /// may mutate in place.
280    iteration_local_fresh: HashSet<ValueId>,
281    /// `inc_rc value` instructions indexed by their operand. Each entry is
282    /// the `(block, instruction-position-within-block)` of one `inc_rc`.
283    inc_rc_locations: HashMap<ValueId, Vec<(BasicBlockId, usize)>>,
284    /// Values that appear at least once as a jmp/jmpif arg on a loop
285    /// back-edge. Used by both:
286    ///
287    /// - [`Context::iteration_local_fresh`] (the back-edge args that are
288    ///   `make_array` or `Call` results, computed in [`Context::new`]); and
289    /// - the **back-edge-participant relaxation** in
290    ///   [`Context::some_inc_rc_precedes`]: an `inc_rc` on a non-source
291    ///   alias that's also a back-edge arg is taken as a codegen
292    ///   signal that the frontend is deliberately managing
293    ///   iteration aliasing, regardless of program-point order.
294    ///
295    /// Forward-edge cases (e.g. re-seeding a value with a global on a
296    /// non-loop branch) are handled precisely by [`Context::backward_aliases`]
297    /// — the forward-edge value simply isn't in the source's backward
298    /// set in the first place — so this signal only needs to cover
299    /// loop back-edges.
300    back_edge_args: HashSet<ValueId>,
301    /// The backward-alias closure of [`Context::back_edge_args`]: every
302    /// value that flows (directly or through forward block-parameter
303    /// edges) into a loop back-edge arg position. A value `v` is a
304    /// back-edge *participant* when its storage can become a loop-header
305    /// parameter's storage across a back-edge — even when `v` itself is
306    /// not the literal arg, but reaches one through a forward edge first
307    /// (e.g. `inc_rc v; jmp latch(v)` then `latch: jmp header(v)`, where
308    /// only the latch's param is the literal back-edge arg).
309    ///
310    /// Used by the **protected-participant filter** in
311    /// [`Context::find_reachable_aliased_use`]: a value that is in the
312    /// array_set source's alias-set (so it flows *into* the source),
313    /// is a back-edge participant, and carries its own `inc_rc` is
314    /// RC ≥ 2 whenever it equals the source's storage (the `inc_rc` is
315    /// loop-carried — it runs before the value crosses the back-edge onto
316    /// the source), so reads of it can't observe an in-place mutation and
317    /// it is dropped from the forward walk's use-set.
318    ///
319    /// The filter is gated **per value** on that value's own `inc_rc`:
320    /// membership in this set alone never exonerates anything. This is
321    /// what keeps it sound where widening the relaxation to "some
322    /// participant has an `inc_rc`" would not — a back-edge position fed
323    /// by an `inc_rc`'d value on one predecessor edge and an unprotected
324    /// value on another only drops the protected value, leaving the
325    /// unprotected one for the forward walk to flag.
326    back_edge_participants: HashSet<ValueId>,
327    /// **Swap exclusions.** For a loop-header parameter `P`, the set of
328    /// sibling parameters `Q` that `P` is rebound to on the loop
329    /// back-edge (`P ← Q`, the array-variable swap `c4 = c3`) while `Q`
330    /// is simultaneously rebound to a freshly-allocated, iteration-local
331    /// value — a `make_array` (`c3 = [..]`) or a `Call` result
332    /// (`c3 = f()`), but never an `array_set` result (which may be an
333    /// in-place mutation, not a fresh allocation). Such a `Q` is a *distinct
334    /// per-iteration storage* that `P` only *becomes* in a later
335    /// iteration — never the storage this iteration's `array_set` on `P`
336    /// mutates (the swap rotates a fresh allocation into `P` while the
337    /// mutated storage is left behind, dead). `Q` is dropped from `P`'s
338    /// alias-set in [`Context::alias_set_for`], which makes the forward
339    /// walk both **kill** `P` where the back-edge rebinds it to `Q` and
340    /// decline to **add** a fresh alias of `Q` past any later edge (the
341    /// inclusive-range *peel* re-runs the same swap on a *forward* exit
342    /// edge — excluding `Q` at the seed covers that edge too, which a
343    /// kill keyed on the back-edge alone could not).
344    ///
345    /// Guarded by a **back-edge-only** check: if `Q` also flows into `P`
346    /// on a forward edge, `P` and `Q` can already alias *within* an
347    /// iteration (the array_set would mutate `Q`'s live storage), so the
348    /// exclusion would be unsound and is not recorded. The freshening
349    /// requirement on `Q` is the second guard — a loop-invariant `Q`
350    /// swapped into `P` would make `P_k = Q_{k-1} = Q_k` and genuinely
351    /// alias.
352    swap_excluded_aliases: HashMap<ValueId, HashSet<ValueId>>,
353}

355impl<'f> Context<'f> {
356    fn new(function: &'f Function) -> Self {
357        let cfg = ControlFlowGraph::with_function(function);
358        let post_order = PostOrder::with_cfg(&cfg);
359        let dom_tree = DominatorTree::with_cfg_and_post_order(&cfg, &post_order);
360
361        // Cache the reverse-post-order block list once. Used for the
362        // per-instruction setup pass below and (more importantly) by
363        // [`compute_backward_aliases`], whose fixed-point converges in
364        // far fewer passes when blocks are visited in RPO than in the
365        // ID-sorted order `function.reachable_blocks()` returns.
366        let rpo: Vec<BasicBlockId> = post_order.into_vec_reverse();
367
368        // Loop back-edges so we can recognize iteration-local
369        // `make_array` results (those passed as args on a back-edge),
370        // which represent fresh-per-iteration storage and so don't
371        // carry the array_set's pre-mutation aliasing forward through
372        // the loop header.
373        let back_edges: HashSet<(BasicBlockId, BasicBlockId)> =
374            Loops::find_all(function, LoopOrder::InsideOut)
375                .yet_to_unroll
376                .iter()
377                .map(|l| (l.back_edge_start, l.header))
378                .collect();
379
380        let mut inc_rc_locations: HashMap<ValueId, Vec<(BasicBlockId, usize)>> = HashMap::default();
381        let mut array_value_defs: HashMap<ValueId, (BasicBlockId, usize)> = HashMap::default();
382        let mut non_aliasing_array_values: HashSet<ValueId> = HashSet::default();
383        let mut make_array_values: HashSet<ValueId> = HashSet::default();
384        let mut call_result_values: HashSet<ValueId> = HashSet::default();
385        let mut back_edge_args: HashSet<ValueId> = HashSet::default();
386        // For each destination block, the list of `(predecessor, args)`
387        // pairs collected from terminators. Used to drive the backward
388        // alias-set fixed-point: each block parameter's set is the union
389        // of its arg at that position from every incoming edge.
390        let mut incoming_edges: HashMap<BasicBlockId, Vec<(BasicBlockId, Vec<ValueId>)>> =
391            HashMap::default();
392
393        // Single pass over every reachable block to populate
394        // per-instruction indices and per-block incoming-edge tables.
395        for &block_id in &rpo {
396            for (idx, instruction_id) in function.dfg[block_id].instructions().iter().enumerate() {
397                let instruction = &function.dfg[*instruction_id];
398
399                if let Instruction::IncrementRc { value } = instruction {
400                    inc_rc_locations.entry(*value).or_default().push((block_id, idx));
401                }
402
403                let is_call = matches!(instruction, Instruction::Call { .. });
404                let is_non_aliasing =
405                    is_call || matches!(instruction, Instruction::ArraySet { .. });
406                let is_make_array = matches!(instruction, Instruction::MakeArray { .. });
407                for &result in function.dfg.instruction_results(*instruction_id) {
408                    if function.dfg.type_of_value(result).contains_an_array() {
409                        array_value_defs.insert(result, (block_id, idx));
410                        if is_non_aliasing {
411                            non_aliasing_array_values.insert(result);
412                        }
413                        if is_make_array {
414                            make_array_values.insert(result);
415                        }
416                        if is_call {
417                            call_result_values.insert(result);
418                        }
419                    }
420                }
421            }
422
423            if let Some(terminator) = function.dfg[block_id].terminator() {
424                match terminator {
425                    TerminatorInstruction::Jmp { destination, arguments, .. } => {
426                        incoming_edges
427                            .entry(*destination)
428                            .or_default()
429                            .push((block_id, arguments.to_vec()));
430                        if back_edges.contains(&(block_id, *destination)) {
431                            back_edge_args.extend(arguments.iter().copied());
432                        }
433                    }
434                    TerminatorInstruction::JmpIf {
435                        then_destination,
436                        then_arguments,
437                        else_destination,
438                        else_arguments,
439                        ..
440                    } => {
441                        incoming_edges
442                            .entry(*then_destination)
443                            .or_default()
444                            .push((block_id, then_arguments.to_vec()));
445                        incoming_edges
446                            .entry(*else_destination)
447                            .or_default()
448                            .push((block_id, else_arguments.to_vec()));
449                        if back_edges.contains(&(block_id, *then_destination)) {
450                            back_edge_args.extend(then_arguments.iter().copied());
451                        }
452                        if back_edges.contains(&(block_id, *else_destination)) {
453                            back_edge_args.extend(else_arguments.iter().copied());
454                        }
455                    }
456                    _ => (),
457                }
458            }
459        }
460
461        let backward_aliases = compute_backward_aliases(function, &rpo, &incoming_edges);
462
463        // Backward-alias closure of the literal back-edge args: every
464        // value that can become a loop-header parameter's storage across
465        // a back-edge, including those that only reach the back-edge arg
466        // position through a forward edge first. A literal back-edge arg
467        // that is a block parameter contributes its whole backward set;
468        // an instruction result or function parameter contributes only
469        // itself (its backward set is the singleton).
470        let mut back_edge_participants: HashSet<ValueId> = HashSet::default();
471        for &arg in &back_edge_args {
472            back_edge_participants.insert(arg);
473            if let Some(arg_set) = backward_aliases.get(&arg) {
474                back_edge_participants.extend(arg_set.iter().copied());
475            }
476        }
477
478        // A back-edge arg is iteration-local *fresh* if it re-allocates
479        // distinct storage every iteration: a `make_array` (re-executes)
480        // or a `Call` result (the callee allocates fresh — the same
481        // assumption [`Context::non_aliasing_array_values`] makes).
482        // `array_set` results are deliberately *excluded*: an `array_set`
483        // may mutate its source in place, so its result can be the *same*
484        // storage as a prior iteration's, breaking the distinct-generation
485        // guarantee the swap exclusion relies on.
486        let iteration_local_fresh: HashSet<ValueId> = back_edge_args
487            .iter()
488            .copied()
489            .filter(|v| make_array_values.contains(v) || call_result_values.contains(v))
490            .collect();
491
492        // Swap exclusions. For every loop back-edge `be_start → header`,
493        // inspect each array-typed header parameter at `source_pos`: if
494        // the back-edge rebinds it to a *sibling* header parameter
495        // (`source ← sibling`) whose own back-edge arg is an
496        // iteration-local fresh allocation, and the source and sibling
497        // receive distinct storage on every forward edge into the header,
498        // record the sibling as excluded from the source's alias-set. See
499        // [`Context::swap_excluded_aliases`].
500        let mut swap_excluded_aliases: HashMap<ValueId, HashSet<ValueId>> = HashMap::default();
501        for &(be_start, header) in &back_edges {
502            let Some(edges) = incoming_edges.get(&header) else { continue };
503            let Some(be_args) =
504                edges.iter().find(|(pred, _)| *pred == be_start).map(|(_, args)| args)
505            else {
506                continue;
507            };
508            let params = function.dfg.block_parameters(header);
509            for (source_pos, &source_param) in params.iter().enumerate() {
510                if !function.dfg.type_of_value(source_param).contains_an_array() {
511                    continue;
512                }
513                let Some(&sibling) = be_args.get(source_pos) else { continue };
514                // `source_param ← sibling` must be a genuine swap to a
515                // *different* sibling header parameter (not self-threading,
516                // not a result).
517                if sibling == source_param {
518                    continue;
519                }
520                let Some(sibling_pos) = params.iter().position(|&pp| pp == sibling) else {
521                    continue;
522                };
523                // The sibling's own back-edge arg must be an
524                // iteration-local fresh allocation (the `c3 = [..]` or
525                // `c3 = f()` half of the swap).
526                if !be_args.get(sibling_pos).is_some_and(|a| iteration_local_fresh.contains(a)) {
527                    continue;
528                }
529                // Loop-entry guard. On every *forward* edge into the
530                // header, the source param and the sibling must receive
531                // **distinct** storage, or they can already alias at the
532                // array_set in the entry iteration (`source_0 = sibling_0`)
533                // and the exclusion would mask a real hazard. Two ways
534                // that can happen:
535                //
536                // - the sibling flows into the source's forward arg
537                //   (`sibling ∈ backward(source_forward_arg)`) — the
538                //   source *is* the sibling from the start.
539                // - the source's and sibling's forward args share a
540                //   backward-set member — e.g. `jmp header(v, v)` feeds the
541                //   same `v` to both, so they're runtime-equal even though
542                //   the directed backward walk keeps them in separate sets.
543                let backward_set = |v: ValueId| -> im::HashSet<ValueId> {
544                    backward_aliases.get(&v).cloned().unwrap_or_else(|| im::HashSet::unit(v))
545                };
546                let entry_aliased = edges
547                    .iter()
548                    .filter(|(pred, _)| !back_edges.contains(&(*pred, header)))
549                    .any(|(_, args)| {
550                        let Some(&source_forward_arg) = args.get(source_pos) else {
551                            return false;
552                        };
553                        let source_forward_aliases = backward_set(source_forward_arg);
554                        if source_forward_aliases.contains(&sibling) {
555                            return true;
556                        }
557                        args.get(sibling_pos).is_some_and(|&sibling_forward_arg| {
558                            let sibling_forward_aliases = backward_set(sibling_forward_arg);
559                            source_forward_aliases
560                                .iter()
561                                .any(|x| sibling_forward_aliases.contains(x))
562                        })
563                    });
564                if entry_aliased {
565                    continue;
566                }
567                swap_excluded_aliases.entry(source_param).or_default().insert(sibling);
568            }
569        }
570
571        Self {
572            function,
573            dom_tree,
574            backward_aliases,
575            array_value_defs,
576            non_aliasing_array_values,
577            iteration_local_fresh,
578            inc_rc_locations,
579            back_edge_args,
580            back_edge_participants,
581            swap_excluded_aliases,
582        }
583    }

585    /// Look up `source`'s backward alias-set and filter out values that
586    /// can't represent the pre-mutation storage of the `array_set` at
587    /// `(array_set_block, array_set_idx)`. The source itself is always
588    /// kept.
589    ///
590    /// **Non-aliasing-result filter.** Drop values produced by an
591    /// `array_set` or `Call`:
592    /// - **`array_set` result** represents a *post*-mutation value of its
593    ///   source. Uses of it (or of any name it gets re-bound to through
594    ///   block-parameter threading) are intentional reads of the mutated
595    ///   storage, not hazards. Keeping them in the alias-set would defeat
596    ///   the per-arg kill rule in [`Context::succ_use_set`]: a back-edge
597    ///   `jmp b(v_arr_set)` would see `v_arr_set` in the use-set and
598    ///   refuse to kill the receiving param, letting the alias leak past
599    ///   the loop.
600    /// - **`Call` result** is typically a fresh array allocated by the
601    ///   callee, so it isn't a real alias of the array_set's source.
602    ///   This is a heuristic — a function that returns its input *would*
603    ///   create a real alias, and we'd miss that. In practice the
604    ///   frontend's array-returning functions allocate fresh storage.
605    ///
606    /// Also drop **iteration-local fresh results** — `MakeArray` (or
607    /// `Call`) results that appear on at least one loop back-edge
608    /// ([`Context::iteration_local_fresh`]). Such a value re-allocates
609    /// fresh storage each iteration, so the loop-header parameter it
610    /// feeds on the back-edge holds a *different* allocation in the next
611    /// iteration than the one this iteration's `array_set` may have
612    /// mutated. **Non-back-edge `MakeArray` results stay in the
613    /// alias-set** — they represent a one-time allocation whose storage
614    /// the array_set can mutate in place.
615    ///
616    /// **Post-array_set-in-same-block filter.** Drop instruction
617    /// results whose defining position is in `array_set_block` at an
618    /// index *greater than* `array_set_idx`. Such a value doesn't exist
619    /// at the array_set's program point yet — it's literally about to
620    /// be allocated on a later instruction — so it can't represent the
621    /// storage the array_set might mutate. It can land in the backward
622    /// set through a round-trip: `make_array → forward-arg → block
623    /// param → back-edge → source's param`. Dropping it at lookup time
624    /// makes the per-arg kill in [`Context::succ_use_set`] correctly
625    /// fire on the forward edge where the future-value is passed.
626    ///
627    /// The source itself is kept even when it happens to be filtered
628    /// (e.g. a chain `v_a = array_set _ ; v_b = array_set v_a`): `v_a`
629    /// is *this* check's source, so its forward uses are exactly what
630    /// we want to look for. The post-array_set filter can't fire on the
631    /// source because the source must be defined before the array_set
632    /// that uses it.
633    fn alias_set_for(
634        &self,
635        source: ValueId,
636        array_set_block: BasicBlockId,
637        array_set_idx: usize,
638    ) -> im::HashSet<ValueId> {
639        let class = self
640            .backward_aliases
641            .get(&source)
642            .cloned()
643            .unwrap_or_else(|| im::HashSet::unit(source));
644        class
645            .into_iter()
646            .filter(|&v| {
647                if v == source {
648                    return true;
649                }
650                if self.non_aliasing_array_values.contains(&v) {
651                    return false;
652                }
653                if self.iteration_local_fresh.contains(&v) {
654                    return false;
655                }
656                if self.swap_excluded_aliases.get(&source).is_some_and(|qs| qs.contains(&v)) {
657                    return false;
658                }
659                if let Some(&(def_block, def_idx)) = self.array_value_defs.get(&v)
660                    && def_block == array_set_block
661                    && def_idx > array_set_idx
662                {
663                    return false;
664                }
665                true
666            })
667            .collect()
668    }

670    /// Returns `true` if some `inc_rc r` for an `r ∈ alias_set` exists at a
671    /// program point that precedes the `array_set` in flow order — either
672    /// in a strictly-earlier position within the same block, or in a
673    /// different block whose reverse-post-order index is smaller.
674    ///
675    /// # Why RPO precedence, not dominance
676    ///
677    /// The strict invariant would be "the `inc_rc` dominates the
678    /// `array_set`" — i.e., is on *every* path. But the frontend in
679    /// practice emits **path-specific** `inc_rc`s: each path that creates
680    /// an alias gets its own `inc_rc`, with no single dominating one. See
681    /// e.g. `brillig_array_ifelse` where `b1` has `inc_rc v8` and `b4` has
682    /// `inc_rc v2`, neither dominating the `array_set` in `b6`. On every
683    /// runtime path the alias is either covered by an `inc_rc` or the
684    /// values are freshly allocated, but the verifier can't observe
685    /// path-specific freshness with the current alias-set model.
686    ///
687    /// We weaken the check accordingly: presence of *any* `inc_rc` on an
688    /// alias-set member, anywhere earlier in flow, is taken as the
689    /// frontend signalling "I'm aware of this aliasing and have handled
690    /// it." Absence of any such `inc_rc` *combined* with a forward aliased
691    /// read (checked separately) still flags as a hazard, which is the
692    /// shape PR-12671 had. The back-edge-participant relaxation below
693    /// widens this further to cover `inc_rc`s placed after the
694    /// array_set, where iteration aliasing makes the bump's program
695    /// point misleading.
696    ///
697    /// # Back-edge-participant relaxation
698    ///
699    /// In addition to RPO precedence, an `inc_rc` is also accepted when
700    /// it lives on an alias-set member that:
701    ///
702    /// - is **not** the array_set's own source (an `inc_rc` on the
703    ///   source itself, *after* the array_set, doesn't protect that
704    ///   array_set — it would be suspicious frontend output rather
705    ///   than a signal); and
706    /// - appears as a jmp/jmpif arg on a loop **back-edge** somewhere
707    ///   in the function ([`Context::back_edge_args`]).
708    ///
709    /// The intuition: when the frontend emits an `inc_rc` on a
710    /// non-source alias that's threaded back through a loop, it's
711    /// deliberately managing iteration aliasing. The bump's program
712    /// point can come *after* the array_set in source order — it's
713    /// the back-edge thread-back that delivers the protection to the
714    /// next iteration. Forward-edge cases (e.g. re-seeding a value
715    /// with a global on a non-loop branch) don't need this relaxation
716    /// because [`Context::backward_aliases`] is precise enough to keep
717    /// the forward-edge value out of the source's alias-set in the
718    /// first place.
719    ///
720    /// Well-formed SSA contains no `DecrementRc`, so we don't need to
721    /// worry about a `dec_rc` intervening between the `inc_rc` and the
722    /// `array_set`.
723    fn some_inc_rc_precedes(
724        &self,
725        alias_set: &im::HashSet<ValueId>,
726        source: ValueId,
727        array_set_block: BasicBlockId,
728        array_set_idx: usize,
729    ) -> bool {
730        for value in alias_set {
731            let Some(locations) = self.inc_rc_locations.get(value) else {
732                continue;
733            };
734            // Back-edge-participant relaxation: an `inc_rc` on a
735            // non-source alias that's also a loop back-edge arg is a
736            // codegen signal regardless of program-point order. The
737            // backward-alias walk already handles forward-edge
738            // threading precisely (the forward-edge value simply isn't
739            // in the source's backward set), so this relaxation only
740            // needs to cover back-edge iteration aliasing.
741            if *value != source && self.back_edge_args.contains(value) {
742                return true;
743            }
744            for &(inc_block, inc_idx) in locations {
745                if inc_block == array_set_block {
746                    if inc_idx < array_set_idx {
747                        return true;
748                    }
749                } else if self.dom_tree.reverse_post_order_cmp(inc_block, array_set_block)
750                    == Ordering::Less
751                {
752                    return true;
753                }
754            }
755        }
756        false
757    }

759    /// Forward CFG walk from after the `array_set` looking for a
760    /// non-terminator instruction that reads a value still in the alias
761    /// use-set.
762    ///
763    /// **Use-set evolution.** Starts as `alias_set` and only shrinks. Kills
764    /// are applied **during propagation** to each successor — see
765    /// [`Context::succ_use_set`] for the kill rules.
766    ///
767    /// **What counts as a use.** Only non-terminator operands. Terminator
768    /// arguments are the legitimate threading mechanism the invariant
769    /// relies on; `jmp b(v_alias)` is how the post-mutation value reaches
770    /// the next block where it is re-bound to that block's parameter. The
771    /// kill logic already accounts for these args.
772    ///
773    /// The original `array_set` itself is also skipped, in case a cycle
774    /// re-enters its block — it is, by construction, a use of its own
775    /// source, not a hazard.
776    ///
777    /// **Chain-aware index filter.** Two auxiliary sets evolve alongside the
778    /// `use_set` to make the filter sound in the presence of `array_set`
779    /// chains:
780    ///
781    /// - **`derived`** tracks values that may share the array_set's source
782    ///   storage at runtime through transitive in-place mutations. Seeded
783    ///   with the array_set's own result; grown on every later `array_set`
784    ///   whose `array` operand is already in `derived`; propagated across
785    ///   block-param edges the same way the alias use-set is.
786    /// - **`tainted_indices`** tracks the set of storage positions any
787    ///   chain link may have already written. `Some(set)` accumulates
788    ///   constant write indices (seeded with `write_index_const`); set to
789    ///   `None` (= "all positions") as soon as any chain write uses a
790    ///   dynamic index.
791    ///
792    /// When we encounter `array_get v, idx` with `v ∈ use_set`, the read
793    /// is a hazard iff `tainted_indices` covers `idx`. With both indices
794    /// constant, that's a set-membership test; otherwise (either side
795    /// dynamic, or `tainted_indices == None`) the verifier conservatively
796    /// flags. An `array_set v, idx, _` with `v ∈ use_set` is index-aware in
797    /// the same way: it copies forward every position except `idx`, so it
798    /// is a hazard iff some `tainted_indices` position differs from `idx`
799    /// (a same-index write overwrites the mutation and extends the chain
800    /// instead). All other uses on a `use_set` member are always flagged —
801    /// the SSA-vs-runtime divergence isn't index-local for them.
802    ///
803    /// **IncrementRc clears `derived`.** A program-point `inc_rc v` with
804    /// `v ∈ use_set ∪ derived` lifts the storage's RC ≥ 2, so any
805    /// subsequent `array_set` on a chain participant runs on fresh
806    /// storage and can no longer clobber the source. We clear `derived`
807    /// at that point. `tainted_indices` is *not* cleared — past damage
808    /// to the shared storage is still observable through `use_set` reads.
809    ///
810    /// **Cycle detection.** Re-entering a block with a state that's a
811    /// (use_set, derived, tainted_indices)-subset of the frontier we've
812    /// already explored adds no new information. Each component is
813    /// unioned into the frontier on entry; for `tainted_indices`, `None`
814    /// is the absorbing element (a previously-`None` frontier covers any
815    /// re-entry).
816    fn find_reachable_aliased_use(
817        &self,
818        alias_set: &im::HashSet<ValueId>,
819        source: ValueId,
820        array_set_id: InstructionId,
821        array_set_block: BasicBlockId,
822        array_set_idx: usize,
823        write_index_const: Option<FieldElement>,
824    ) -> Option<AliasedUse> {
825        let mut visited: HashMap<BasicBlockId, WalkState> = HashMap::default();
826
827        // Protected-participant set. Exclude a non-source alias `v` when
828        // all three hold:
829        //
830        // 1. `v ∈ alias_set` — `v` is in `source`'s backward set, i.e. it
831        //    actually *flows into* `source`. This is the soundness guard:
832        //    a value that only *receives* `source`'s storage (a forward
833        //    successor) is not in the backward set, so its `inc_rc` runs
834        //    *after* the array_set and cannot protect it. Requiring
835        //    membership keeps such values in the use-set to be flagged.
836        // 2. `v` carries its own `inc_rc` — combined with (3), the bump
837        //    runs before `v` crosses the back-edge that re-binds it onto
838        //    `source`'s loop-header parameter, so by the time `v`'s
839        //    storage equals `source`'s it is RC ≥ 2 and the array_set
840        //    copies rather than mutating in place.
841        // 3. `v` is a loop back-edge participant
842        //    ([`Context::back_edge_participants`]) — its storage reaches
843        //    `source` through a back-edge (possibly via a forward edge
844        //    into the back-edge arg first), which is what makes the
845        //    `inc_rc` loop-carried.
846        //
847        // The gate is **per value** on that value's own `inc_rc` — a
848        // sibling's `inc_rc` never exonerates an unprotected value. A
849        // back-edge position fed by an `inc_rc`'d value on one predecessor
850        // edge and an unprotected value on another protects only the
851        // former; the latter stays in the use-set for the walk to flag.
852        //
853        // The removal is **sticky**: protected members are kept out of the
854        // use-set for the whole walk, not just the initial seed. A
855        // protected participant that is itself a loop-header parameter
856        // would otherwise be re-introduced the moment another in-use alias
857        // flows into its position (the add-rule in
858        // [`Context::succ_use_set`]); excluding it there too keeps it out.
859        // Dropping it from the seed also lets the per-arg kill rule drop
860        // the loop-header parameter on the back-edge, since the value
861        // threaded back is this protected participant rather than a
862        // still-live alias.
863        let protected: im::HashSet<ValueId> = alias_set
864            .iter()
865            .copied()
866            .filter(|&v| {
867                v != source
868                    && self.inc_rc_locations.contains_key(&v)
869                    && self.back_edge_participants.contains(&v)
870            })
871            .collect();
872        let use_set: im::HashSet<ValueId> =
873            alias_set.iter().copied().filter(|v| !protected.contains(v)).collect();
874
875        let array_set_result = self.function.dfg.instruction_results(array_set_id)[0];
876        let initial_state = WalkState {
877            use_set,
878            derived: im::HashSet::unit(array_set_result),
879            tainted: write_index_const.map(im::HashSet::unit),
880        };
881        let mut worklist: Vec<WalkFrame> = vec![WalkFrame {
882            block: array_set_block,
883            start_idx: array_set_idx + 1,
884            state: initial_state,
885        }];
886
887        while let Some(WalkFrame { block, start_idx, mut state }) = worklist.pop() {
888            // No alias-set members in use means no read of this array_set's
889            // source's storage can surface a hazard from here on.
890            if state.use_set.is_empty() {
891                continue;
892            }
893
894            // Cycle/redundancy check + bookkeeping only applies to *full*
895            // block entries (start_idx == 0). The very first frame of the
896            // walk starts mid-block and only covers a suffix of the block —
897            // recording it here would incorrectly let a later back-edge
898            // entry to the same block skip the unexplored prefix.
899            if start_idx == 0 {
900                if let Some(prev) = visited.get(&block)
901                    && state.is_covered_by(prev)
902                {
903                    continue;
904                }
905                let new_frontier = match visited.get(&block) {
906                    Some(prev) => state.merge(prev),
907                    None => state.clone(),
908                };
909                visited.insert(block, new_frontier);
910            }
911
912            let instructions = self.function.dfg[block].instructions();
913            for inst_idx in start_idx..instructions.len() {
914                let inst_id = instructions[inst_idx];
915                if inst_id == array_set_id {
916                    continue;
917                }
918                let inst = &self.function.dfg[inst_id];
919
920                match inst {
921                    // `inc_rc` on an alias or chain-derived value lifts the
922                    // shared storage's RC; subsequent chain writes run on
923                    // fresh storage and stop tainting the source. Past
924                    // tainted indices survive — the damage is already done.
925                    Instruction::IncrementRc { value } => {
926                        if state.derived.contains(value) || state.use_set.contains(value) {
927                            state.derived.clear();
928                        }
929                    }
930                    // Well-formed Brillig SSA shouldn't contain dec_rc;
931                    // skip if encountered.
932                    Instruction::DecrementRc { .. } => {}
933                    Instruction::ArraySet { array, index, value, .. } => {
934                        let array_in_use = state.use_set.contains(array);
935                        if array_in_use {
936                            // Index-aware, mirroring the `array_get` rule below.
937                            // `array_set v, i, x` produces a *copy* of `v` with
938                            // index `i` overwritten, so it only observes (copies
939                            // forward) the indices it does **not** write. It can
940                            // therefore surface the source's in-place mutation
941                            // only if it copies a tainted index — i.e. some
942                            // tainted index differs from this write index. A
943                            // write to the (sole) tainted index overwrites the
944                            // mutation and observes nothing. A dynamic write or
945                            // read index, or fully-tainted (`None`) storage, is
946                            // flagged conservatively.
947                            let write_idx = self.function.dfg.get_numeric_constant(*index);
948                            let observes_tainted = match (&state.tainted, write_idx) {
949                                (None, _) | (_, None) => true,
950                                (Some(t), Some(c)) => t.iter().any(|i| *i != c),
951                            };
952                            if observes_tainted {
953                                return Some(AliasedUse { instruction: inst_id, value: *array });
954                            }
955                        }
956                        if self.function.dfg.type_of_value(*value).contains_an_array()
957                            && state.use_set.contains(value)
958                        {
959                            return Some(AliasedUse { instruction: inst_id, value: *value });
960                        }
961                        // Chain extension: a write through a value that may
962                        // share the source's storage — a `derived` member, or a
963                        // `use_set` member whose write we just cleared as
964                        // non-observing — keeps mutating that storage, so record
965                        // its write index and track the result.
966                        if state.derived.contains(array) || array_in_use {
967                            match (
968                                state.tainted.as_mut(),
969                                self.function.dfg.get_numeric_constant(*index),
970                            ) {
971                                (Some(t), Some(c)) => {
972                                    t.insert(c);
973                                }
974                                _ => {
975                                    state.tainted = None;
976                                }
977                            }
978                            let [result] = self.function.dfg.instruction_result(inst_id);
979                            state.derived.insert(result);
980                        }
981                    }
982                    Instruction::ArrayGet { array, index, .. } => {
983                        if state.use_set.contains(array) {
984                            let read_idx = self.function.dfg.get_numeric_constant(*index);
985                            let hazard = match (&state.tainted, read_idx) {
986                                (None, _) | (_, None) => true,
987                                (Some(set), Some(c)) => set.contains(&c),
988                            };
989                            if hazard {
990                                return Some(AliasedUse { instruction: inst_id, value: *array });
991                            }
992                        }
993                    }
994                    other => {
995                        // Any other instruction reading an alias-set member
996                        // through an array-typed operand is a hazard. The
997                        // SSA-vs-runtime divergence isn't index-local for
998                        // these instruction kinds.
999                        let mut hit: Option<ValueId> = None;
1000                        other.for_each_value(|v| {
1001                            if hit.is_some() {
1002                                return;
1003                            }
1004                            if self.function.dfg.type_of_value(v).contains_an_array()
1005                                && state.use_set.contains(&v)
1006                            {
1007                                hit = Some(v);
1008                            }
1009                        });
1010                        if let Some(v) = hit {
1011                            return Some(AliasedUse { instruction: inst_id, value: v });
1012                        }
1013                    }
1014                }
1015            }
1016
1017            let Some(terminator) = self.function.dfg[block].terminator() else { continue };
1018            match terminator {
1019                TerminatorInstruction::Jmp { destination, arguments, .. } => {
1020                    let next = self.succ_walk_state(*destination, arguments, &state, &protected);
1021                    worklist.push(WalkFrame { block: *destination, start_idx: 0, state: next });
1022                }
1023                TerminatorInstruction::JmpIf {
1024                    then_destination,
1025                    then_arguments,
1026                    else_destination,
1027                    else_arguments,
1028                    ..
1029                } => {
1030                    let then_state =
1031                        self.succ_walk_state(*then_destination, then_arguments, &state, &protected);
1032                    worklist.push(WalkFrame {
1033                        block: *then_destination,
1034                        start_idx: 0,
1035                        state: then_state,
1036                    });
1037                    let else_state =
1038                        self.succ_walk_state(*else_destination, else_arguments, &state, &protected);
1039                    worklist.push(WalkFrame {
1040                        block: *else_destination,
1041                        start_idx: 0,
1042                        state: else_state,
1043                    });
1044                }
1045                TerminatorInstruction::Return { .. }
1046                | TerminatorInstruction::Unreachable { .. } => (),
1047            }
1048        }
1049
1050        None
1051    }

1053    /// Propagate the walk state across a block-parameter edge. `use_set`
1054    /// and `derived` follow the same kill/add rules ([`Context::succ_use_set`]);
1055    /// `tainted` is carried unchanged, since it tracks storage positions
1056    /// (not SSA values).
1057    fn succ_walk_state(
1058        &self,
1059        dest: BasicBlockId,
1060        arguments: &[ValueId],
1061        state: &WalkState,
1062        protected: &im::HashSet<ValueId>,
1063    ) -> WalkState {
1064        WalkState {
1065            use_set: self.succ_use_set(dest, arguments, &state.use_set, protected),
1066            derived: self.succ_use_set(dest, arguments, &state.derived, protected),
1067            tainted: state.tainted.clone(),
1068        }
1069    }

1071    /// Compute the use-set carried into `dest` when its predecessor jumps
1072    /// with `arguments`.
1073    ///
1074    /// Three rules apply, in order:
1075    ///
1076    /// 1. **Per-arg propagation — block parameters of `dest`.** For each
1077    ///    `dest.params[i]`, look at the corresponding `arguments[i]`:
1078    ///    - **Kill.** If the param is in `use_set` and the arg is not, the
1079    ///      param is rebound to a value that no longer aliases the
1080    ///      array_set's source (most commonly the array_set's own
1081    ///      result, excluded at lookup time): drop it.
1082    ///    - **Add.** If the param is not in `use_set` but the arg is,
1083    ///      this edge introduces a fresh alias: the param at `dest`'s
1084    ///      entry shares storage with an alias-set member, so add it —
1085    ///      *unless* the param is a protected back-edge participant
1086    ///      ([`Context::find_reachable_aliased_use`]), which must stay out
1087    ///      of the use-set for the whole walk (its own `inc_rc` keeps it
1088    ///      RC ≥ 2). Without this exclusion a protected participant that
1089    ///      is itself a loop-header parameter would be re-introduced the
1090    ///      moment any in-use alias flowed into its position.
1091    ///    - Otherwise (both in or both out), no change.
1092    ///    Only array-typed params participate.
1093    ///
1094    /// 2. **Unconditional kill — instructions defined in `dest`.** For
1095    ///    each alias-set member whose defining block is `dest`: drop it.
1096    ///    Re-entering `dest` re-executes the defining instruction (e.g.
1097    ///    `load`, `make_array`), producing a fresh runtime value; the
1098    ///    previous iteration's storage is no longer reachable through that
1099    ///    name.
1100    fn succ_use_set(
1101        &self,
1102        dest: BasicBlockId,
1103        arguments: &[ValueId],
1104        use_set: &im::HashSet<ValueId>,
1105        protected: &im::HashSet<ValueId>,
1106    ) -> im::HashSet<ValueId> {
1107        let mut result = use_set.clone();
1108
1109        // (1) Per-arg propagation for params of `dest`.
1110        let params = self.function.dfg.block_parameters(dest);
1111        for (i, &param) in params.iter().enumerate() {
1112            if !self.function.dfg.type_of_value(param).contains_an_array() {
1113                continue;
1114            }
1115            let arg_in_use_set = arguments.get(i).is_some_and(|arg| use_set.contains(arg));
1116            let param_in_use_set = use_set.contains(&param);
1117            match (param_in_use_set, arg_in_use_set) {
1118                (true, false) => {
1119                    result.remove(&param);
1120                }
1121                (false, true) => {
1122                    if !protected.contains(&param) {
1123                        result.insert(param);
1124                    }
1125                }
1126                _ => {}
1127            }
1128        }
1129
1130        // (2) Unconditional kill for instruction-defined values whose
1131        // def-block is `dest` (re-execution on cycle re-entry produces a
1132        // fresh value).
1133        for &v in use_set {
1134            if self.array_value_defs.get(&v).map(|(b, _)| *b) == Some(dest) {
1135                result.remove(&v);
1136            }
1137        }
1138
1139        result
1140    }
1141}

1143/// Compute, for each array-typed block parameter, the set of values that
1144/// may share its storage at the parameter's binding — itself plus every
1145/// value that flows into it through some chain of predecessor →
1146/// block-parameter edges.
1147///
1148/// Runs a fixed-point iteration in reverse-post-order: each block
1149/// parameter starts with `{p}`, and each pass unions in `args[p_index]`
1150/// (and its own backward set, if it's also a block parameter) for
1151/// every incoming edge. The iteration stops when no set grows.
1152///
1153/// Visiting blocks in RPO means each parameter sees its forward-edge
1154/// predecessors' sets *already at their fixed point* (RPO orders
1155/// predecessor before successor along forward edges), so the only
1156/// source of additional passes is loop back-edges. In practice this
1157/// converges in 1 pass for loop-free code and 2-3 for typical
1158/// loops — versus O(chain-depth) passes under arbitrary block order.
1159///
1160/// # Why the walk is directed (param ← arg), not symmetric
1161///
1162/// A symmetric closure over `(param, arg)` pairs would put two sibling
1163/// args passed to the same block in the same class via the shared
1164/// parameter, even though at runtime they never refer to the same
1165/// storage (each runtime path takes only one of them). Following the
1166/// directed `param ← arg` edges instead means sibling parameters don't
1167/// co-mingle in each other's sets — only the predecessors that flow
1168/// *into* a given parameter do.
1169///
1170/// # Why we deliberately do **not** chase past `array_set` results
1171///
1172/// At runtime the result may or may not alias the source depending on
1173/// RC. The verifier's job is to flag the cases where it *might*. We
1174/// model this by *excluding* the result from each alias-set at lookup
1175/// time (in [`Context::alias_set_for`]) rather than refusing to chase
1176/// through it here. Otherwise a chain
1177/// `v1 = array_set v0 ; v2 = array_set v1 ; v3 = array_set v2` would
1178/// pull `v0` into the source of every later array_set, and an
1179/// `inc_rc v0` that legitimately protects only `v1`'s array_set would
1180/// falsely appear to protect `v3` as well. See
1181/// `alias_set_does_not_walk_array_set_chains` for a worked example —
1182/// the chain isn't visible to the backward walk anyway because
1183/// `v1`, `v2`, `v3` are instruction results (not block parameters),
1184/// so there's nothing to chase from `v3`.
1185fn compute_backward_aliases(
1186    function: &Function,
1187    rpo: &[BasicBlockId],
1188    incoming_edges: &HashMap<BasicBlockId, Vec<(BasicBlockId, Vec<ValueId>)>>,
1189) -> HashMap<ValueId, im::HashSet<ValueId>> {
1190    let mut result: HashMap<ValueId, im::HashSet<ValueId>> = HashMap::default();
1191
1192    // Seed each array-typed block parameter with `{p}`.
1193    for &block_id in rpo {
1194        for &param in function.dfg.block_parameters(block_id) {
1195            if function.dfg.type_of_value(param).contains_an_array() {
1196                result.insert(param, im::HashSet::unit(param));
1197            }
1198        }
1199    }
1200
1201    // Fixed-point in RPO: keep iterating until no parameter's set grows.
1202    // Each pass folds in args from every incoming edge, transitively
1203    // through any predecessor arg that's itself a block parameter.
1204    let mut changed = true;
1205    while changed {
1206        changed = false;
1207        for &block_id in rpo {
1208            let params = function.dfg.block_parameters(block_id);
1209            let Some(incoming) = incoming_edges.get(&block_id) else {
1210                continue;
1211            };
1212            for (i, &param) in params.iter().enumerate() {
1213                if !function.dfg.type_of_value(param).contains_an_array() {
1214                    continue;
1215                }
1216                let mut new_set = result.get(&param).cloned().unwrap_or_else(im::HashSet::new);
1217                let prev_len = new_set.len();
1218                for (_pred, args) in incoming {
1219                    if let Some(&arg) = args.get(i) {
1220                        // Pull in `arg`'s backward set if it's a block
1221                        // parameter; otherwise the arg is an instruction
1222                        // result whose backward set is the singleton.
1223                        if let Some(arg_set) = result.get(&arg) {
1224                            new_set.extend(arg_set.iter().copied());
1225                        } else {
1226                            new_set.insert(arg);
1227                        }
1228                    }
1229                }
1230                if new_set.len() > prev_len {
1231                    result.insert(param, new_set);
1232                    changed = true;
1233                }
1234            }
1235        }
1236    }
1237
1238    result
1239}

1241/// Per-frame state of the forward reachable-use walk: which alias-set
1242/// members are live, which chain-derived values share storage with the
1243/// source, and which storage positions any chain link may already have
1244/// clobbered. Also serves as the per-block visited frontier for cycle
1245/// detection — see [`WalkState::is_covered_by`] / [`WalkState::merge`].
1246#[derive(Clone)]
1247struct WalkState {
1248    /// Alias-set members live at this point: values that may share the
1249    /// array_set's source storage *at the array_set's program point*. A
1250    /// non-terminator read of one of these is the hazard the walk is
1251    /// looking for.
1252    use_set: im::HashSet<ValueId>,
1253    /// Values that may share the source's storage through transitive
1254    /// in-place chain mutations. Seeded with the array_set's own result;
1255    /// extended by every later `array_set` whose `array` operand is
1256    /// already in `derived`.
1257    derived: im::HashSet<ValueId>,
1258    /// Storage positions any chain link may already have written. `None`
1259    /// (= "all positions") absorbs a dynamic chain write — once we lose
1260    /// precise tracking we can't recover it.
1261    tainted: Option<im::HashSet<FieldElement>>,
1262}
1263
1264impl WalkState {
1265    /// `self` is covered by `prev` when every potential hazard `self`
1266    /// could surface is also reachable from `prev`. For `use_set` and
1267    /// `derived` that's the subset relation; for `tainted`, `None`
1268    /// (= "all positions") absorbs as the upper bound — a previously
1269    /// `None` frontier covers any re-entry, but a current `None`
1270    /// requires `prev` also to be `None`.
1271    fn is_covered_by(&self, prev: &Self) -> bool {
1272        if !self.use_set.is_subset(&prev.use_set) || !self.derived.is_subset(&prev.derived) {
1273            return false;
1274        }
1275        match (&prev.tainted, &self.tainted) {
1276            (None, _) => true,
1277            (_, None) => false,
1278            (Some(p), Some(c)) => c.is_subset(p),
1279        }
1280    }
1281
1282    /// Component-wise union — the frontier we record after visiting a
1283    /// block, so future re-entries can be cycle-checked against
1284    /// everything we've already explored from that block.
1285    fn merge(&self, other: &Self) -> Self {
1286        Self {
1287            use_set: self.use_set.clone().union(other.use_set.clone()),
1288            derived: self.derived.clone().union(other.derived.clone()),
1289            tainted: match (&self.tainted, &other.tainted) {
1290                (None, _) | (_, None) => None,
1291                (Some(a), Some(b)) => Some(a.clone().union(b.clone())),
1292            },
1293        }
1294    }
1295}

3473    /// A located `array_set` plus everything the per-array_set verifier
3474    /// checks would need. Returned by [`find_array_sets`] / [`first_array_set`]
3475    /// / [`last_array_set`] so each test reads one struct rather than a
3476    /// six-element tuple.
3477    struct ArraySetSite {
3478        block: super::BasicBlockId,
3479        idx: usize,
3480        instruction_id: super::InstructionId,
3481        source: ValueId,
3482        alias_set: im::HashSet<ValueId>,
3483        /// The `array_set`'s index when it is a numeric constant. `None`
3484        /// indicates a dynamic index, in which case the verifier
3485        /// conservatively flags every aliased use.
3486        write_index_const: Option<super::FieldElement>,
3487    }
3488
3489    fn find_array_sets(function: &Function, ctx: &Context<'_>) -> Vec<ArraySetSite> {
3490        let mut out = Vec::new();
3491        for block_id in function.reachable_blocks() {
3492            for (idx, instruction_id) in function.dfg[block_id].instructions().iter().enumerate() {
3493                if let Instruction::ArraySet { array, index, .. } = function.dfg[*instruction_id] {
3494                    out.push(ArraySetSite {
3495                        block: block_id,
3496                        idx,
3497                        instruction_id: *instruction_id,
3498                        source: array,
3499                        alias_set: ctx.alias_set_for(array, block_id, idx),
3500                        write_index_const: function.dfg.get_numeric_constant(index),
3501                    });
3502                }
3503            }
3504        }
3505        out
3506    }
3507
3508    fn first_array_set(function: &Function, ctx: &Context<'_>) -> Option<ArraySetSite> {
3509        find_array_sets(function, ctx).into_iter().next()
3510    }
3511
3512    fn last_array_set(function: &Function, ctx: &Context<'_>) -> Option<ArraySetSite> {
3513        find_array_sets(function, ctx).into_iter().last()
