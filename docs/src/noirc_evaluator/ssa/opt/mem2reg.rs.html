1//! The goal of the mem2reg SSA optimization pass is to replace any `Load` instructions to known
2//! addresses with the value stored at that address, if it is also known. This pass will also remove
3//! any `Store` instructions within a block that are no longer needed because no more loads occur in
4//! between the Store in question and the next Store.
5//!
6//! ## How the pass works:
7//! - Each block in each function is iterated in forward-order.
8//! - The starting value of each reference in the block is the unification of the same references
9//!   at the end of each direct predecessor block to the current block.
10//! - At each step, the value of each reference is either known or unknown, tracked with a map.
11//! - Two reference values unify to each other if they are exactly equal.
12//! - If a block has no predecessors, the starting value of each reference is unknown (not present in the map).
13//! - Throughout this pass, aliases of each reference are also tracked.
14//!   - References typically have 1 alias - themselves.
15//!   - A reference with multiple aliases means we will not be able to optimize out loads if the
16//!     reference is stored to. Note that this means we can still optimize out loads if these
17//!     aliased references are never stored to, or the store occurs after a load.
18//!   - A reference with 0 aliases means we were unable to find which reference this reference
19//!     refers to. If such a reference is stored to, we must conservatively invalidate every
20//!     reference in the current block.
21//! - We also track the last load instruction to each address per block.
22//!
23//! From there, to figure out the value of each reference at the end of block, iterate each instruction:
24//! - On `Instruction::Allocate`:
25//!   - Register a new reference was made with itself as its only alias
26//! - On `Instruction::Load { address }`:
27//!   - If `address` is known to only have a single alias (including itself) and if the value of
28//!     that alias is known, replace the value of the load with the known value.
29//!   - Furthermore, if the result of the load is a reference, mark the result as an alias
30//!     of the reference it dereferences to (if known).
31//!     - If which reference it dereferences to is not known, this load result has no aliases.
32//!   - We also track the last instance of a load instruction to each address in a block.
33//!     If we see that the last load instruction was from the same address as the current load instruction,
34//!     we move to replace the result of the current load with the result of the previous load.
35//!
36//!     This removal requires a couple conditions:
37//!       - No store occurs to that address before the next load,
38//!       - The address is not used as an argument to a call
39//!
40//!     This optimization helps us remove repeated loads for which there are not known values.
41//! - On `Instruction::Store { address, value }`:
42//!   - If the address of the store is known:
43//!     - If the address has exactly 1 alias:
44//!       - Set the value of the address to the known `value`.
45//!     - If the address has more than 1 alias:
46//!       - Clear out the known value of of every possible alias.
47//!     - If the address has 0 aliases:
48//!       - Conservatively mark every alias in the block as unknown.
49//!   - If the address of the store is not known:
50//!     - Conservatively mark every alias in the block as unknown.
51//!   - Additionally, if there were no Loads to any alias of the address between this Store and
52//!     the previous Store to the same address, the previous store can be removed.
53//!   - Remove the instance of the last load instruction to the address and its aliases
54//! - On `Instruction::Call { arguments }`:
55//!   - If any argument of the call is a reference, remove the known value of each alias of that
56//!     reference
57//!   - Any builtin functions that may return aliases if their input also contains a
58//!     reference should be tracked. Examples: `vector_push_back`, `vector_insert`, `vector_remove`, etc.
59//!   - Remove the instance of the last load instruction for any reference arguments and their aliases
60//!
61//! On a terminator instruction:
62//! - If the terminator is a `Jmp`:
63//!   - For each reference argument of the jmp, mark the corresponding block parameter it is passed
64//!     to as an alias for the jmp argument.
65//!
66//! Finally, if this is the only block in the function, we can remove any Stores that were not
67//! referenced by the terminator instruction.
68//!
69//! Repeating this algorithm for each block in the function in program order should result in
70//! optimizing out most known loads. However, identifying all aliases correctly has been proven
71//! undecidable in general (Landi, 1992). So this pass will not always optimize out all loads
72//! that could theoretically be optimized out. This pass can be performed at any time in the
73//! SSA optimization pipeline, although it will be more successful the simpler the program's CFG is.
74//! This pass is currently performed several times to enable other passes - most notably being
75//! performed before loop unrolling to try to allow for mutable variables used for loop indices.

104    /// Attempts to remove any load instructions that recover values that are already available in
105    /// scope, and attempts to remove stores that are subsequently redundant.
106    #[tracing::instrument(level = "trace", skip(self))]
107    pub(crate) fn mem2reg(mut self) -> Ssa {
108        for function in self.functions.values_mut() {
109            function.mem2reg();
110        }
111        self
112    }

115impl Function {
116    pub(crate) fn mem2reg(&mut self) {
117        // Analyze loops to find potential loop carried aliases
118        let loop_aliases = Self::analyze_loop_aliases(self);
119        // Perform mem2reg optimization with loop carried alias information
120        // Non-lop alias information will be analyzed as part of mem2reg
121        let mut context = PerFunctionContext::new(self, loop_aliases);
122        context.mem2reg();
123        context.remove_instructions();
124        context.update_data_bus();
125    }
126
127    /// Analyzes all loops in the function to find references with potential loop carried aliases.
128    ///
129    /// A loop carried alias occurs when a reference is stored into another reference
130    /// within a loop. This creates an aliasing relationship that persists across loop iterations,
131    /// making it unsafe to remove certain stores to these references.
132    ///
133    /// Returns a set of values that may have loop carried aliases.
134    fn analyze_loop_aliases(function: &Function) -> HashSet<ValueId> {
135        let loops = Loops::find_all(function, LoopOrder::OutsideIn);
136        let mut aliases: HashSet<ValueId> = HashSet::default();
137
138        // For each loop, find all `store ref_value at ref_address` patterns
139        for loop_info in &loops.yet_to_unroll {
140            for block_id in &loop_info.blocks {
141                let block = &function.dfg[*block_id];
142
143                for instruction_id in block.instructions() {
144                    if let Instruction::Store { address, value } = &function.dfg[*instruction_id] {
145                        // Check if the value is a reference.
146                        // This indicates we're storing a reference into another reference
147                        // An address is always a reference, as this is an invariant of the Store instruction.
148                        if function.dfg.value_is_reference(*value) {
149                            // Mark both the address and value as potentially aliased
150                            aliases.insert(*address);
151                            aliases.insert(*value);
152                        }
153                    }
154                }
155            }
156        }
157
158        aliases
159    }
160}

162struct PerFunctionContext<'f> {
163    cfg: ControlFlowGraph,
164    post_order: PostOrder,
165
166    blocks: BTreeMap<BasicBlockId, Block>,
167
168    inserter: FunctionInserter<'f>,
169
170    /// Store instructions which we should consider moving later.
171    /// Maps an address to the set of store instructions to that address that are eligible for
172    /// removal and the block they're in.
173    ///
174    /// We wait to remove store instructions until the end of mem2reg in case
175    /// any addresses are later discovered to have aliases. In the case of loops we may think an
176    /// instruction eligible for removal at first, only to have the address aliased only on the
177    /// next iteration of the loop.
178    store_instructions_to_remove: HashMap<ValueId, Vec<(BasicBlockId, InstructionId)>>,
179
180    /// Load and Store instructions that should be removed at the end of the pass.
181    ///
182    /// We avoid removing individual instructions as we go since removing elements
183    /// from the middle of Vecs many times will be slower than a single call to `retain`.
184    instructions_to_remove: HashSet<InstructionId>,
185
186    /// All instructions analyzed so far in the function.
187    /// Anything new can be reinserted,  while things that appear repeatedly have to be cloned.
188    instructions_analyzed: HashSet<InstructionId>,
189
190    /// Track a value's last load across all blocks.
191    /// If a value is not used in anymore loads we can remove the last store to that value.
192    last_loads: HashSet<ValueId>,
193
194    /// Track whether a reference was passed into another instruction (e.g. Call)
195    /// This is needed to determine whether we can remove a store.
196    instruction_input_references: HashSet<ValueId>,
197
198    /// Track whether a reference has been aliased, and store the respective
199    /// instruction that aliased that reference.
200    /// If that store has been set for removal, we can also remove this instruction.
201    aliased_references: HashMap<ValueId, HashSet<InstructionId>>,
202
203    /// Loop carried aliases: references that may have aliases due to stores within loops.
204    /// Contains values of references that should not have their stores removed.
205    loop_aliases: HashSet<ValueId>,
206}

209    fn new(function: &'f mut Function, loop_aliases: HashSet<ValueId>) -> Self {
210        let cfg = ControlFlowGraph::with_function(function);
211        let post_order = PostOrder::with_cfg(&cfg);
212        PerFunctionContext {
213            cfg,
214            post_order,
215            inserter: FunctionInserter::new(function),
216            blocks: BTreeMap::new(),
217            instructions_to_remove: HashSet::default(),
218            store_instructions_to_remove: Default::default(),
219            instructions_analyzed: HashSet::default(),
220            last_loads: HashSet::default(),
221            aliased_references: HashMap::default(),
222            instruction_input_references: HashSet::default(),
223            loop_aliases,
224        }

227    /// Check if an address has loop carried aliases.
228    ///
229    /// This is important for preventing incorrect store removal. If `store v3 at v2`
230    /// occurs in a loop, then v2 may point to v3 in future iterations. A subsequent
231    /// `store X at v3` should not have its previous store removed, as that store may
232    /// be read through v2 (which now aliases v3) in a future loop iteration.
233    ///
234    /// We only check the address, not the value being stored. The decision of
235    /// whether to remove a previous store to `address` depends solely on whether
236    /// `address` might be accessed through an alias, not on whether the value being
237    /// stored has aliases.
238    fn has_loop_carried_aliases(&self, address: ValueId) -> bool {
239        self.loop_aliases.contains(&address)
240    }

242    /// Apply the mem2reg pass to the given function.
243    ///
244    /// This function is expected to be the same one that the internal cfg, post_order, and
245    /// dom_tree were created from.
246    fn mem2reg(&mut self) {
247        // Iterate each block in reverse post order = forward order
248        let block_order = self.post_order.clone().into_vec_reverse();
249
250        for block in block_order {
251            let references = self.find_starting_references(block);
252            self.analyze_block(block, references);
253        }
254
255        let mut all_terminator_values = HashSet::default();
256        let mut per_func_block_params: HashSet<ValueId> = HashSet::default();
257        for (block_id, references) in self.blocks.iter_mut() {
258            let block_params = self.inserter.function.dfg.block_parameters(*block_id);
259            per_func_block_params.extend(block_params.iter());
260            let terminator = self.inserter.function.dfg[*block_id].unwrap_terminator();
261            terminator.for_each_value(|value| {
262                Self::collect_terminator_value_aliases(
263                    value,
264                    references,
265                    &self.inserter.function.dfg,
266                    &mut all_terminator_values,
267                );
268            });
269        }
270
271        self.add_instructions_to_remove(&all_terminator_values, &per_func_block_params);
272    }

274    fn add_instructions_to_remove(
275        &mut self,
276        all_terminator_values: &HashSet<ValueId>,
277        per_func_block_params: &HashSet<ValueId>,
278    ) {
279        // If we never load from an address within a function we can remove all stores to that address.
280        // This rule does not apply to reference parameters, which we must also check for before removing these stores.
281        for (_, block) in self.blocks.iter() {
282            for (store_address, store_instruction) in block.last_stores.iter() {
283                let store_alias_used = self.is_store_alias_used(
284                    store_address,
285                    block,
286                    all_terminator_values,
287                    per_func_block_params,
288                );
289
290                let is_dereference = block
291                    .expressions
292                    .get(store_address)
293                    .is_some_and(|expression| matches!(expression, Expression::Dereference(_)));
294
295                if !self.last_loads.contains(store_address) && !store_alias_used && !is_dereference
296                {
297                    self.instructions_to_remove.insert(*store_instruction);
298                }
299            }
300        }
301
302        for (address, instructions) in self.store_instructions_to_remove.drain() {
303            for (block_id, instruction) in instructions {
304                let block = &self.blocks[&block_id];
305                if block.get_aliases_for_value(address).single_alias().is_some() {
306                    self.instructions_to_remove.insert(instruction);
307                }
308            }
309        }
310    }

312    // Extra checks on where a reference can be used aside a load instruction.
313    // Even if all loads to a reference have been removed we need to make sure that
314    // an allocation did not come from an entry point or was passed to an entry point.
315    fn is_store_alias_used(
316        &self,
317        store_address: &ValueId,
318        block: &Block,
319        all_terminator_values: &HashSet<ValueId>,
320        per_func_block_params: &HashSet<ValueId>,
321    ) -> bool {
322        let reference_parameters = self.reference_parameters();
323
324        let aliases = block.get_aliases_for_value(*store_address);
325        if aliases.is_unknown() {
326            return true;
327        }
328        // Check whether the store address has an alias that crosses an entry point boundary (e.g. a Call or Return)
329        for alias in aliases.iter() {
330            if reference_parameters.contains(&alias) {
331                return true;
332            }
333
334            if per_func_block_params.contains(&alias) {
335                return true;
336            }
337
338            // Is any alias of this address an input to some function call, or a return value?
339            if self.instruction_input_references.contains(&alias) {
340                return true;
341            }
342
343            // Is any alias of this address used in a block terminator?
344            if all_terminator_values.contains(&alias) {
345                return true;
346            }
347
348            // Check whether there are any aliases whose instructions are not all marked for removal.
349            // If there is any alias marked to survive, we should not remove its last store.
350            if let Some(alias_instructions) = self.aliased_references.get(&alias) {
351                if !alias_instructions.is_subset(&self.instructions_to_remove) {
352                    return true;
353                }
354            }
355        }
356
357        false
358    }

360    /// Collects a terminator value and all its aliases into the output set.
361    ///
362    /// When a value has unknown aliases (e.g., because it contains references from
363    /// arrays with unknown contents), we fall back to directly collecting the array
364    /// elements if it's an array constant. This ensures that stores to references
365    /// within the array are not incorrectly removed.
366    fn collect_terminator_value_aliases(
367        value: ValueId,
368        references: &Block,
369        dfg: &DataFlowGraph,
370        output: &mut HashSet<ValueId>,
371    ) {
372        output.insert(value);
373
374        let typ = dfg.type_of_value(value);
375        if !typ.contains_reference() {
376            return;
377        }
378
379        let aliases = references.get_aliases_for_value(value);
380        if aliases.is_unknown() {
381            // If aliases are unknown but this is an array constant, directly collect
382            // the elements. This handles the case where an array contains references
383            // that have unknown aliases (e.g., from array parameters), but we still
384            // need to track the elements that DO have known aliases.
385            if let Some((elements, _)) = dfg.get_array_constant(value) {
386                for element in elements.iter() {
387                    Self::collect_terminator_value_aliases(*element, references, dfg, output);
388                }
389            }
390        } else {
391            output.extend(aliases.iter());
392        }
393    }

395    /// Collect the input parameters of the function which are of reference type.
396    /// All references are mutable, so these inputs are shared with the function caller
397    /// and thus stores should not be eliminated, even if the blocks in this function
398    /// don't use them anywhere.
399    fn reference_parameters(&self) -> BTreeSet<ValueId> {
400        let parameters = self.inserter.function.parameters().iter();
401        parameters
402            .filter(|param| self.inserter.function.dfg.value_is_reference(**param))
403            .copied()
404            .collect()
405    }

407    /// The value of each reference at the start of the given block is the unification
408    /// of the value of the same reference at the end of its predecessor blocks.
409    fn find_starting_references(&mut self, block: BasicBlockId) -> Block {
410        let mut predecessors = self.cfg.predecessors(block);
411
412        if let Some(first_predecessor) = predecessors.next() {
413            let mut first = self.blocks.get(&first_predecessor).cloned().unwrap_or_default();
414
415            first.last_stores.clear();
416
417            // Note that we have to start folding with the first block as the accumulator.
418            // If we started with an empty block, an empty block union'd with any other block
419            // is always also empty so we'd never be able to track any references across blocks.
420            predecessors.fold(first, |block, predecessor| {
421                let predecessor = self.blocks.entry(predecessor).or_default();
422                block.unify(predecessor)
423            })
424        } else {
425            Block::default()
426        }
427    }

429    /// Analyze a block with the given starting reference values.
430    ///
431    /// This will remove any known loads in the block and track the value of references
432    /// as they are stored to. When this function is finished, the value of each reference
433    /// at the end of this block will be remembered in `self.blocks`.
434    fn analyze_block(&mut self, block: BasicBlockId, mut references: Block) {
435        let instructions = self.inserter.function.dfg[block].take_instructions();
436
437        // If this is the entry block, take all the block parameters and assume they may
438        // be aliased to each other
439        if block == self.inserter.function.entry_block() {
440            self.add_aliases_for_reference_parameters(block, &mut references);
441        }
442
443        for instruction in instructions {
444            self.analyze_instruction(block, &mut references, instruction);
445        }
446
447        self.handle_terminator(block, &mut references);
448
449        // If there's only 1 block in the function total, we can remove any remaining last stores
450        // as well. We can't do this if there are multiple blocks since subsequent blocks may
451        // reference these stores.
452        if self.post_order.as_slice().len() == 1 {
453            self.remove_stores_that_do_not_alias_parameters_or_returns(&references);
454        }
455
456        self.blocks.insert(block, references);
457    }

459    /// Go through each parameter and register that all reference parameters of the same type are
460    /// possibly aliased to each other. If there are parameters with nested references (arrays of
461    /// references or references containing other references) we give up and assume all parameter
462    /// references are `AliasSet::unknown()`.
463    fn add_aliases_for_reference_parameters(&self, block: BasicBlockId, references: &mut Block) {
464        let dfg = &self.inserter.function.dfg;
465        let params = dfg.block_parameters(block);
466
467        let mut aliases: HashMap<Type, AliasSet> = HashMap::default();
468
469        for param in params {
470            match dfg.type_of_value(*param) {
471                // If the type indirectly contains a reference we have to assume all references
472                // are unknown since we don't have any ValueIds to use.
473                Type::Reference(element) if element.contains_reference() => {
474                    self.mark_all_unknown(params, references);
475                    return;
476                }
477                Type::Reference(element) => {
478                    let empty_aliases = AliasSet::known_empty();
479                    let alias_set =
480                        aliases.entry(element.as_ref().clone()).or_insert(empty_aliases);
481                    alias_set.insert(*param);
482                }
483                typ if typ.contains_reference() => {
484                    self.mark_all_unknown(params, references);
485                    return;
486                }
487                _ => continue,
488            }
489        }
490
491        for aliases in aliases.into_values() {
492            let first = aliases.first();
493            let first = first.expect("All parameters alias at least themselves or we early return");
494
495            let expression = Expression::Other(first);
496            let previous = references.aliases.insert(expression, aliases.clone());
497            assert!(previous.is_none());
498
499            for alias in aliases.iter() {
500                let previous = references.expressions.insert(alias, expression);
501                assert!(previous.is_none());
502            }
503        }
504    }

506    /// Add all instructions in `last_stores` to `self.instructions_to_remove` which do not
507    /// possibly alias any parameters of the given function.
508    fn remove_stores_that_do_not_alias_parameters_or_returns(&mut self, references: &Block) {
509        let reference_parameters = self.reference_parameters();
510
511        for (allocation, instruction) in &references.last_stores {
512            let aliases = references.get_aliases_for_value(*allocation);
513
514            let allocation_aliases_parameter =
515                aliases.any(|alias| reference_parameters.contains(&alias));
516            // If `allocation_aliases_parameter` is known to be false
517            if allocation_aliases_parameter != Some(false) {
518                continue;
519            }
520
521            let allocation_aliases_input_reference =
522                aliases.any(|alias| self.instruction_input_references.contains(&alias));
523            if allocation_aliases_input_reference != Some(false) {
524                continue;
525            }
526
527            self.instructions_to_remove.insert(*instruction);
528        }
529    }

531    fn analyze_instruction(
532        &mut self,
533        block_id: BasicBlockId,
534        references: &mut Block,
535        instruction_id: InstructionId,
536    ) {
537        // If the instruction was simplified and optimized out of the program we shouldn't analyze it.
538        // Analyzing it could make tracking aliases less accurate if it is e.g. an ArrayGet
539        // call that used to hold references but has since been optimized out to a known result.
540        // However, if we don't analyze it, then it may be a MakeArray replacing an ArraySet containing references,
541        // and we need to mark those references as used to keep their stores alive.
542        let (instruction, loc) = self.inserter.map_instruction(instruction_id);
543
544        // We track which instructions can be removed by ID; if we allowed the same ID to appear multiple times
545        // in a block then we could not tell them apart. When we see something the first time we can reuse it.
546        let allow_reinsert = self.instructions_analyzed.insert(instruction_id);
547
548        match self.inserter.push_instruction_value(
549            instruction,
550            instruction_id,
551            block_id,
552            loc,
553            allow_reinsert,
554        ) {
555            InsertInstructionResult::Results(id, _) => {
556                self.analyze_possibly_simplified_instruction(references, id, false, block_id);
557            }
558            InsertInstructionResult::SimplifiedTo(value) => {
559                // Globals cannot contain references thus we do not need to analyze insertion which simplified to them.
560                if self.inserter.function.dfg.is_global(value) {
561                    return;
562                }
563                let value = &self.inserter.function.dfg[value];
564                if let Value::Instruction { instruction, .. } = value {
565                    self.analyze_possibly_simplified_instruction(
566                        references,
567                        *instruction,
568                        true,
569                        block_id,
570                    );
571                }
572            }
573            InsertInstructionResult::SimplifiedToMultiple(values) => {
574                for value in values {
575                    // Globals cannot contain references thus we do not need to analyze insertion which simplified to them.
576                    if self.inserter.function.dfg.is_global(value) {
577                        continue;
578                    }
579                    let value = &self.inserter.function.dfg[value];
580                    if let Value::Instruction { instruction, .. } = value {
581                        self.analyze_possibly_simplified_instruction(
582                            references,
583                            *instruction,
584                            true,
585                            block_id,
586                        );
587                    }
588                }
589            }
590            InsertInstructionResult::InstructionRemoved => (),
591        }
592    }

594    fn analyze_possibly_simplified_instruction(
595        &mut self,
596        references: &mut Block,
597        instruction: InstructionId,
598        simplified: bool,
599        block_id: BasicBlockId,
600    ) {
601        let ins = &self.inserter.function.dfg[instruction];
602
603        // Some instructions, when simplified, cause problems if processed again.
604        // We do need it for MakeArray in some cases, and at the moment that is not problematic.
605        if simplified && !matches!(ins, Instruction::MakeArray { .. }) {
606            return;
607        }
608
609        match ins {
610            Instruction::Load { address } => {
611                let address = *address;
612
613                let [result] = self.inserter.function.dfg.instruction_result(instruction);
614                references.remember_dereference(self.inserter.function, address, result);
615
616                // If the load is known, replace it with the known value and remove the load.
617                if let Some(value) = references.get_known_value(address) {
618                    let [result] = self.inserter.function.dfg.instruction_result(instruction);
619                    self.inserter.map_value(result, value);
620                    self.instructions_to_remove.insert(instruction);
621                }
622                // Check whether the block has a repeat load from the same address (w/ no calls or stores in between the loads).
623                // If we do have a repeat load, we can remove the current load and map its result to the previous load's result.
624                else if let Some(last_load) = references.last_loads.get(&address) {
625                    let Instruction::Load { address: previous_address } =
626                        &self.inserter.function.dfg[*last_load]
627                    else {
628                        panic!("Expected a Load instruction here");
629                    };
630                    let [result] = self.inserter.function.dfg.instruction_result(instruction);
631                    let [previous_result] =
632                        self.inserter.function.dfg.instruction_result(*last_load);
633                    if *previous_address == address {
634                        self.inserter.map_value(result, previous_result);
635                        self.instructions_to_remove.insert(instruction);
636                    }
637                } else {
638                    // Remember that this address has been loaded, so stores to it should not be removed.
639                    self.last_loads.insert(address);
640                    // Stores to any of its aliases should also be considered loaded.
641                    self.last_loads.extend(references.get_aliases_for_value(address).iter());
642                }
643
644                // If the address is potentially aliased we must keep the stores to it
645                if references.get_aliases_for_value(address).single_alias().is_none() {
646                    references.mark_value_used(address, self.inserter.function);
647                }
648
649                // We want to set the load for every load even if the address has a known value
650                // and the previous load instruction was removed.
651                // We are safe to still remove a repeat load in this case as we are mapping from the current load's
652                // result to the previous load, which if it was removed should already have a mapping to the known value.
653                references.set_last_load(address, instruction);
654            }
655            Instruction::Store { address, value } => {
656                let address = *address;
657                let value = *value;
658
659                let address_aliases = references.get_aliases_for_value(address);
660                // If there was another store to this address without any (unremoved) loads or
661                // function calls in-between, we can remove the previous store.
662                // However, we must be conservative if there are loop carried aliases, as loads
663                // through those aliases may occur in future loop iterations.
664                let has_loop_aliases = self.has_loop_carried_aliases(address);
665
666                if !self.aliased_references.contains_key(&address)
667                    && address_aliases.single_alias().is_some()
668                    && !has_loop_aliases
669                {
670                    if let Some(last_store) = references.last_stores.get(&address) {
671                        self.store_instructions_to_remove
672                            .entry(address)
673                            .or_default()
674                            .push((block_id, *last_store));
675                    }
676                }
677
678                // Remember that we used the value in this instruction. If this instruction
679                // isn't removed at the end, we need to keep the stores to the value as well.
680                for alias in references.get_aliases_for_value(value).iter() {
681                    self.aliased_references.entry(alias).or_default().insert(instruction);
682                }
683
684                references.set_known_value(address, value);
685                // If we see a store to an address, the last load to that address needs to remain.
686                references.keep_last_load_for(address);
687                references.last_stores.insert(address, instruction);
688            }
689            Instruction::Allocate => {
690                // Register the new reference
691                let [result] = self.inserter.function.dfg.instruction_result(instruction);
692                references.expressions.insert(result, Expression::Other(result));
693                references.aliases.insert(Expression::Other(result), AliasSet::known(result));
694            }
695            Instruction::ArrayGet { array, .. } => {
696                let [result] = self.inserter.function.dfg.instruction_result(instruction);
697
698                if self.inserter.function.dfg.type_of_value(result).contains_reference() {
699                    let array = *array;
700                    self.instruction_input_references
701                        .extend(references.get_aliases_for_value(array).iter());
702                    references.mark_value_used(array, self.inserter.function);
703
704                    // An expression for the value might already exist, so try to fetch it first
705                    let expression = references.expressions.get(&array).copied();
706                    let expression = expression.unwrap_or(Expression::Other(array));
707                    if let Some(aliases) = references.aliases.get_mut(&expression) {
708                        aliases.insert(result);
709                    }
710
711                    // Any aliases of the array need to be updated to also include the result of the array get in their alias sets.
712                    for alias in (*references.get_aliases_for_value(array)).clone().iter() {
713                        // An expression for the alias might already exist, so try to fetch it first
714                        let expression = references.expressions.get(&alias).copied();
715                        let expression = expression.unwrap_or(Expression::Other(alias));
716                        if let Some(aliases) = references.aliases.get_mut(&expression) {
717                            aliases.insert(result);
718                        }
719                    }
720
721                    // In this SSA:
722                    //
723                    // v2 = array_get v0, index v1 -> Field
724                    //
725                    // make v2 point to v0 so they share the same alias set
726                    references.expressions.insert(result, expression);
727                }
728            }
729            Instruction::ArraySet { array, value, .. } => {
730                references.mark_value_used(*array, self.inserter.function);
731                let element_type = self.inserter.function.dfg.type_of_value(*value);
732
733                if element_type.contains_reference() {
734                    let [result] = self.inserter.function.dfg.instruction_result(instruction);
735                    let array = *array;
736
737                    let expression = references.expressions.get(&array).copied();
738                    let expression = expression.unwrap_or(Expression::Other(array));
739
740                    let mut aliases = if let Some(aliases) = references.aliases.get_mut(&expression)
741                    {
742                        aliases.clone()
743                    } else if let Some((elements, _)) =
744                        self.inserter.function.dfg.get_array_constant(array)
745                    {
746                        references.collect_all_aliases(elements)
747                    } else {
748                        AliasSet::unknown()
749                    };
750                    aliases.unify(&references.get_aliases_for_value(*value));
751
752                    references.expressions.insert(result, expression);
753                    references.aliases.insert(expression, aliases.clone());
754
755                    // Similar to how we remember that we used a value in a `Store` instruction,
756                    // take note that it was used in the `ArraySet`. If this instruction is not
757                    // going to be removed at the end, we shall keep the stores to this value as well.
758                    //
759                    // We want to make sure to mark aliased references before we update the value's alias list to match the array itself.
760                    // This ordering is necessary because if the unified alias list becomes unknown we will not end up
761                    // inserting the value as a possible aliased reference across blocks.
762                    // This could also be done by checking whether the new unified aliases are unknown and marking the `value` explicitly as an alias.
763                    for alias in references.get_aliases_for_value(*value).iter() {
764                        self.aliased_references.entry(alias).or_default().insert(instruction);
765                    }
766
767                    // The value being stored in the array also needs its aliases updated to match the array itself
768                    let value_expression = references.expressions.get(value).copied();
769                    let value_expression = value_expression.unwrap_or(Expression::Other(*value));
770                    references.aliases.insert(value_expression, aliases);
771                }
772            }
773            Instruction::Call { arguments, .. } => {
774                // We need to appropriately mark each alias of a reference as being used as a call argument.
775                // This prevents us potentially removing a last store from a preceding block or is altered within another function.
776                for arg in arguments {
777                    self.instruction_input_references
778                        .extend(references.get_aliases_for_value(*arg).iter());
779                }
780                self.mark_all_unknown(arguments, references);
781                // Call results might be aliases of their arguments, if they are references
782                let results = self.inserter.function.dfg.instruction_results(instruction);
783                let results_contains_references = results.iter().any(|result| {
784                    self.inserter.function.dfg.type_of_value(*result).contains_reference()
785                });
786
787                // Instead of aliasing results to arguments, because values might be nested references
788                // we'll just consider all arguments and references as now having unknown aliases.
789                if results_contains_references {
790                    for value in arguments.iter().chain(results) {
791                        self.clear_aliases(references, *value);
792                    }
793                }
794            }
795            Instruction::MakeArray { elements, typ } => {
796                // If `array` is an array constant that contains reference types, then insert each element
797                // as a potential alias to the array itself.
798                if typ.contains_reference() {
799                    let [array] = self.inserter.function.dfg.instruction_result(instruction);
800
801                    let expr = Expression::ArrayElement(array);
802                    references.expressions.insert(array, expr);
803
804                    let new_aliases = self.collect_array_aliases(elements, references);
805                    let aliases = references.aliases.entry(expr).or_insert(AliasSet::known_empty());
806                    aliases.unify(&new_aliases);
807                }
808            }
809            Instruction::IfElse { then_value, else_value, .. } => {
810                let [result] = self.inserter.function.dfg.instruction_result(instruction);
811                let result_type = self.inserter.function.dfg.type_of_value(result);
812
813                if result_type.contains_reference() {
814                    let expr = Expression::Other(result);
815                    references.expressions.insert(result, expr);
816
817                    // Collect all aliases from both branches.
818                    // For the case: `if cond { array1 } else { array2 }` where
819                    // both arrays contain references - we need to track them.
820                    let then_aliases = references.get_aliases_for_value(*then_value).into_owned();
821                    let else_aliases = references.get_aliases_for_value(*else_value).into_owned();
822                    let mut all_aliases =
823                        AliasSet::known_multiple(vec![*then_value, *else_value].into());
824                    all_aliases.unify(&then_aliases);
825                    all_aliases.unify(&else_aliases);
826
827                    references.aliases.insert(expr, all_aliases.clone());
828
829                    // Mark references in both branches as being used by this IfElse instruction.
830                    // This ensures that stores to those references are kept even in loops where
831                    // alias information may not propagate correctly through the back edge.
832                    for alias in then_aliases.iter() {
833                        self.aliased_references.entry(alias).or_default().insert(instruction);
834                    }
835                    for alias in else_aliases.iter() {
836                        self.aliased_references.entry(alias).or_default().insert(instruction);
837                    }
838
839                    // `then_value` and `else_value` are now aliased by `result`
840                    if let Some(then_expr) = references.expressions.get_mut(then_value) {
841                        if let Some(then_aliases) = references.aliases.get_mut(then_expr) {
842                            then_aliases.insert(result);
843                        }
844                    }
845
846                    if let Some(else_expr) = references.expressions.get_mut(else_value) {
847                        if let Some(else_aliases) = references.aliases.get_mut(else_expr) {
848                            else_aliases.insert(result);
849                        }
850                    }
851                }
852            }
853            _ => (),
854        }
855    }

857    /// In order to handle nested arrays we need to recursively search for whether there
858    /// are any aliases contained within an array's elements.
859    fn collect_array_aliases(
860        &self,
861        elements: &im::Vector<ValueId>,
862        references: &Block,
863    ) -> AliasSet {
864        let mut aliases = AliasSet::known_empty();
865        for &element in elements {
866            if let Some((elements, _)) = self.inserter.function.dfg.get_array_constant(element) {
867                aliases.unify(&self.collect_array_aliases(&elements, references));
868            } else if self.inserter.function.dfg.type_of_value(element).contains_reference() {
869                // Handles both direct references and non-constant arrays (e.g., array_set results)
870                aliases.unify(&references.get_aliases_for_value(element));
871            }
872        }
873        aliases
874    }

876    fn set_aliases(&self, references: &mut Block, address: ValueId, new_aliases: AliasSet) {
877        let expression =
878            references.expressions.entry(address).or_insert(Expression::Other(address));
879        let aliases = references.aliases.entry(*expression).or_default();
880        *aliases = new_aliases;
881    }
882
883    fn clear_aliases(&self, references: &mut Block, value: ValueId) {
884        if !self.inserter.function.dfg.type_of_value(value).contains_reference() {
885            return;
886        }
887
888        if let Some(expression) = references.expressions.get(&value) {
889            references.aliases.remove(expression);
890        }
891
892        if let Some((values, _)) = self.inserter.function.dfg.get_array_constant(value) {
893            for value in values {
894                self.clear_aliases(references, value);
895            }
896        }
897    }
898
899    fn mark_all_unknown(&self, values: &[ValueId], references: &mut Block) {
900        for value in values {
901            let typ = self.inserter.function.dfg.type_of_value(*value);
902            // We must recursively check composite types for a reference (e.g., array of references),
903            // as we still need to mark those internal references as unknown.
904            if typ.contains_reference() {
905                let value = *value;
906                // If we have a nested reference (e.g., &mut &mut Field), recurse to invalidate what it points to.
907                // This is necessary because for example, a callee could load this reference and mutate through the inner reference.
908                if let Type::Reference(element) = &typ {
909                    if element.contains_reference() {
910                        if let Some(inner_ref) = references.get_known_value(value) {
911                            self.mark_all_unknown(&[inner_ref], references);
912                        }
913                    }
914                }
915
916                references.set_unknown(value);
917                references.mark_value_used(value, self.inserter.function);
918
919                // If a reference is an argument to a call, the last load to that address and its aliases needs to remain.
920                references.keep_last_load_for(value);
921
922                // Clear aliases AFTER keep_last_load_for so it can still find the aliases to preserve
923                self.clear_aliases(references, value);
924            }
925        }
926    }

928    /// Remove any instructions in `self.instructions_to_remove` from the current function.
929    /// This is expected to contain any loads which were replaced and any stores which are
930    /// no longer needed.
931    fn remove_instructions(&mut self) {
932        // The order we iterate blocks in is not important
933        for block in self.post_order.as_slice() {
934            self.inserter.function.dfg[*block]
935                .instructions_mut()
936                .retain(|instruction| !self.instructions_to_remove.contains(instruction));
937        }
938    }

944    fn handle_terminator(&mut self, block: BasicBlockId, references: &mut Block) {
945        self.inserter.map_terminator_in_place(block);
946
947        match self.inserter.function.dfg[block].unwrap_terminator() {
948            TerminatorInstruction::JmpIf { .. } | TerminatorInstruction::Unreachable { .. } => (), // Nothing to do
949            TerminatorInstruction::Jmp { destination, arguments, .. } => {
950                let destination_parameters = self.inserter.function.dfg[*destination].parameters();
951                assert_eq!(destination_parameters.len(), arguments.len());
952
953                // If we have multiple parameters that alias that same argument value,
954                // then those parameters also alias each other.
955                // We save parameters with repeat arguments to later mark those
956                // parameters as aliasing one another.
957                let mut arg_set: HashMap<ValueId, VecSet<[ValueId; 1]>> = HashMap::default();
958
959                // Add an alias for each reference parameter
960                for (parameter, argument) in destination_parameters.iter().zip(arguments) {
961                    match self.inserter.function.dfg.type_of_value(*parameter) {
962                        // If the type indirectly contains a reference we have to assume all references
963                        // are unknown since we don't have any ValueIds to use.
964                        Type::Reference(element) if element.contains_reference() => {
965                            self.mark_all_unknown(destination_parameters, references);
966                            return;
967                        }
968                        Type::Reference(_) => {
969                            if let Some(expression) = references.expressions.get(argument) {
970                                if let Some(aliases) = references.aliases.get_mut(expression) {
971                                    // If the argument has unknown aliases, we must be conservative
972                                    // and mark all destination parameters as unknown. Otherwise,
973                                    // inserting into an unknown alias set is a no-op and destination parameters
974                                    // would incorrectly end up in separate alias sets.
975                                    if aliases.is_unknown() {
976                                        self.mark_all_unknown(destination_parameters, references);
977                                        return;
978                                    }
979
980                                    let argument = *argument;
981
982                                    // The argument reference is possibly aliased by this block parameter
983                                    aliases.insert(*parameter);
984
985                                    // Check if we have seen the same argument
986                                    let seen_parameters = arg_set
987                                        .entry(argument)
988                                        .or_insert_with(|| VecSet::single(argument));
989                                    // Add the current parameter to the parameters we have seen for this argument.
990                                    // The previous parameters and the current one alias one another.
991                                    seen_parameters.insert(*parameter);
992                                    // Also add all of the argument aliases
993                                    seen_parameters.extend(aliases.iter());
994                                }
995                            }
996                        }
997                        typ if typ.contains_reference() => {
998                            self.mark_all_unknown(destination_parameters, references);
999                            return;
1000                        }
1001                        _ => continue,
1002                    }
1003                }
1004
1005                // Set the aliases of the parameters
1006                for (_, aliased_params) in arg_set {
1007                    for param in aliased_params.iter() {
1008                        self.set_aliases(
1009                            references,
1010                            *param,
1011                            AliasSet::known_multiple(aliased_params.clone()),
1012                        );
1013                    }
1014                }
1015            }
1016            TerminatorInstruction::Return { return_values, .. } => {
1017                // Removing all `last_stores` for each returned reference is more important here
1018                // than setting them all to unknown since no other block should
1019                // have a block with a Return terminator as a predecessor anyway.
1020                self.mark_all_unknown(return_values, references);
1021            }
1022        }
1023    }
