1//! Replaces `array_get` with known indices with values from previous instructions
2//! such as `array_set` or `make_array`.
3//!
4//! Given these two instructions:
5//!
6//! ```text
7//! v1 = array_set v0, index 0, value: 42
8//! v2 = array_get v1, index 0 -> Field
9//! ```
10//!
11//! because we get from `v1` at `index 0`, but `v1` is the result of setting the value "42"
12//! at `index 0`, we can conclude that `v2` will be "42", and so this SSA pass will do that.
13//! However, this is only safe to do if the `array_set` happened under the same side effects
14//! variable as the `array_get`. For example, in this case:
15//!
16//! ```text
17//! enable_side_effects v100
18//! v1 = array_set v0, index 0, value: 42
19//! enable_side_effects v200
20//! v2 = array_get v1, index 0 -> Field
21//! ```
22//!
23//! it would be wrong to replace `v2` with "42" as the previous `array_set` might not have
24//! been executed.
25//!
26//! However, in this case:
27//!
28//! ```text
29//! enable_side_effects u1 1
30//! v1 = array_set v0, index 0, value: 42
31//! enable_side_effects v200
32//! v2 = array_get v1, index 0 -> Field
33//! ```
34//!
35//! the optimization can be applied because the `array_set` is unconditional.
36//!
37//! In this case:
38//!
39//! ```text
40//! v1 = array_set v0, index 0, value: 42
41//! v2 = array_set v1, index 1, value: 15
42//! v3 = array_get v2, index 0 -> Field
43//! ```
44//!
45//! for `v3` the optimization will try to find a previous `array_set` with the same index (`index 0`).
46//! It will first find `v2`. Because it's an `array_set` of a different **known** index, it will
47//! then find `v1` and apply the same optimization as before. Note that it's safe to skip `v2` and
48//! look at `v1` even if `v2` was under a different side effects var because it doesn't affect
49//! the index used in `v3`.
50//!
51//! In this case:
52//!
53//! ```text
54//! v1 = array_set v0, index 0, value: 42
55//! v2 = array_set v1, index v88, value: 15
56//! v3 = array_get v2, index 0 -> Field
57//! ```
58//!
59//! for `v3` the optimization will find `v2`. Because the set index is unknown, and it might be zero,
60//! the optimization can't deduce anything so it won't do anything.
61//!
62//! Another case where the optimization applies is when it finds a `make_array`:
63//!
64//! ```text
65//! v1 = make_array [Field 10, Field 20] : [Field; 2]
66//! v2 = array_get v1, index 0 -> Field
67//! ```
68//!
69//! In this case `v2` will be replaced with `Field 10`. A `make_array` could also be reached
70//! after passing through other `array_set` with a different index, as previously shown.
71//!
72//! Finally, the optimization might also reach to params:
73//!
74//! ```text
75//! b0(v1: [Field; 2]):
76//!   v2 = array_set v1, index 1, value: 42
77//!   v3 = array_get v2, index 0 -> Field
78//! ```
79//!
80//! In this case `v3` will be replaced with `array_get v1, index 0`, directly getting from `v1`
81//! instead of from `v2`, because `v2` is the same as `v1` except for what's in index 1, but
82//! `v3` is getting from index 0.
83//!
84//! The [pass][Function::array_get_optimization] applies all of the above by scanning the function
85//! and caching the known contents of the array values it writes (an [`ArrayView`]). Resolving an
86//! `array_get` at a constant index is then a lookup into that view rather than a walk back over
87//! previous instructions, so a long chain of `array_set`s no longer makes each read more expensive.
88//! Each cached element records the side-effects predicate it was written under, and a read only uses
89//! it when that predicate is unconditional or equal to the read's own predicate. The defining
90//! `array_set` dominates the read (the array reaches it by data flow) and a false predicate leaves
91//! the element unchanged, so that comparison is sound whether the write and the read are in the same
92//! block or not; the cache is therefore kept for the whole function. See
93//! [the pass][Function::array_get_optimization] for the full argument.
94//!
95//! This module also provides a [`try_optimize_array_get_from_previous_instructions`] function
96//! that is used in other SSA-related optimizations. For example, whenever an `array_get` is inserted
97//! into a [`DFG`][crate::ssa::ir::dfg::DataFlowGraph]: in this case a previous `array_set` with the
98//! same index as the `array_get` cannot be used because we don't know under which side effects var it
99//! happens. However, `array_set` with a different known index can be skipped through to eventually
100//! reach a `make_array` or param. That helper has no cache to consult, so it does a small bounded
101//! walk instead.

121impl Ssa {
122    /// Replaces `array_get` instructions with known indices with known values from
123    /// previous instructions. See the [`array_get`][self] module for more information.
124    #[tracing::instrument(level = "trace", skip(self))]
125    pub(crate) fn array_get_optimization(mut self) -> Self {
126        for func in self.functions.values_mut() {
127            func.array_get_optimization();
128        }
129        self
130    }
131}

133impl Function {
134    fn array_get_optimization(&mut self) {
135        #[cfg(debug_assertions)]
136        array_get_optimization_pre_check(self);
137
138        // Caches the known contents of each array value as the function is scanned, so resolving an
139        // `array_get` at a constant index is a lookup rather than a walk back through previous
140        // instructions.
141        //
142        // Each cached element records the side-effects predicate of the `array_set` that wrote it,
143        // and `resolve` only uses it when the write was unconditional or carried the same predicate
144        // value the read sees. That condition is sound on its own: the array value is reached by
145        // data flow, so its defining `array_set` dominates the read, and an `array_set` whose
146        // predicate is false leaves the element unchanged — so a matching (or unconditional)
147        // predicate means the written value is really there. `simple_optimization` resets the
148        // predicate to `1` at the start of every block, so a non-trivial predicate is only ever
149        // seen within a single block; a read in a later block sees predicate `1` and folds only
150        // unconditional writes. Either way the comparison holds across blocks, so the cache is kept
151        // for the whole function rather than reset per block.
152        let mut views: HashMap<ValueId, ArrayView> = HashMap::new();
153
154        self.simple_optimization(|context| {
155            let instruction_id = context.instruction_id;
156
157            match context.instruction() {
158                Instruction::ArraySet { array, index, value, .. } => {
159                    let array = *array;
160                    let value = *value;
161                    let Some(index) = constant_index(context.dfg, *index) else {
162                        // A dynamic index may write to any element, so nothing is known about the
163                        // result. Leaving it uncached means reads of it resolve as `Unknown`.
164                        return;
165                    };
166                    let predicate = context.enable_side_effects;
167                    let [result] = context.dfg.instruction_result(instruction_id);
168
169                    let view = array_view(&views, context.dfg, array)
170                        .with_element(index, value, predicate);
171                    views.insert(result, view);
172                }
173                Instruction::ArrayGet { array, index } => {
174                    let array = *array;
175                    let index = *index;
176                    let Some(target_index) = constant_index(context.dfg, index) else {
177                        return;
178                    };
179                    let predicate = context.enable_side_effects;
180
181                    let view = array_view(&views, context.dfg, array);
182                    let Some(resolution) =
183                        view.resolve(array, target_index, predicate, context.dfg)
184                    else {
185                        return;
186                    };
187
188                    context.remove_current_instruction();
189                    let [result] = context.dfg.instruction_result(instruction_id);
190                    match resolution {
191                        Resolution::Value(value) => {
192                            context.replace_value(result, value);
193                        }
194                        Resolution::ReadFrom(source) => {
195                            debug_assert_ne!(
196                                source, array,
197                                "a read rewritten to the same array would re-trigger forever"
198                            );
199                            let array_get = Instruction::ArrayGet { array: source, index };
200                            let result_typ = context.dfg.type_of_value(result).into_owned();
201                            let new_result = context
202                                .insert_instruction(array_get, Some(vec![result_typ]))
203                                .first();
204                            context.replace_value(result, new_result);
205                        }
206                    }
207                }
208                _ => {}
209            }
210        });
211    }
212}

214/// Returns the cached view of `array`, seeding a fresh one with [`ArrayView::for_value`] for arrays
215/// not written by an `array_set` earlier in the current block.
216fn array_view(
217    views: &HashMap<ValueId, ArrayView>,
218    dfg: &DataFlowGraph,
219    array: ValueId,
220) -> ArrayView {
221    views.get(&array).cloned().unwrap_or_else(|| ArrayView::for_value(dfg, array))
222}

224fn constant_index(dfg: &DataFlowGraph, index: ValueId) -> Option<u32> {
225    dfg.get_numeric_constant(index)?.try_to_u32()
226}

228/// Asserts the invariant the cross-block `array_get` cache relies on: a non-trivial side-effects
229/// predicate is confined to a single block.
230///
231/// This holds exactly when a multi-block function contains no `enable_side_effects` instruction
232/// (those appear only after flattening, which collapses the function to a single block). SSA
233/// validation enforces this at construction but not between passes, so guard it here too: a breach
234/// means the cross-block cache can no longer be trusted, and asserting it once up front catches a
235/// future pipeline change that breaks the invariant rather than letting it silently emit an unsound
236/// fold.
237#[cfg(debug_assertions)]
238fn array_get_optimization_pre_check(func: &Function) {
239    // The pass runs both before flattening (several blocks, but no `enable_side_effects` yet) and
240    // after it (a single block that may carry `enable_side_effects`). A single-block function
241    // trivially confines any predicate to that block, so only a multi-block function can break the
242    // invariant — and a well-formed one predates flattening and has no `enable_side_effects` at all.
243    if func.reachable_blocks().len() > 1 {
244        super::checks::for_each_instruction(func, |instruction, _| {
245            super::checks::assert_not_enable_side_effects(instruction);
246        });
247    }
248}

250/// The result of the `array_get` optimization.
251pub(crate) enum ArrayGetOptimizationResult {
252    /// The `array_get` can be replaced with the given value.
253    Value(ValueId),
254    /// The `array_get` can be replaced by fetching from the given array at the same index as
255    /// the `array_get`'s index.
256    ArrayGet(ValueId),
257}
258
259/// Side effects information to be able to optimize `array_get` more efficiently.
260pub(crate) struct ArrayGetOptimizationSideEffects<'a> {
261    /// The current value of the side effects var.
262    pub(crate) side_effects_var: ValueId,
263    /// The side effects var applied to each known `array_set` instruction.
264    pub(crate) array_set_predicates: &'a HashMap<InstructionId, ValueId>,
265}

267/// Tries to replace an `array_get` instructions with values from previous instructions.
268/// See the [`array_get`][self] module for more information.
269pub(crate) fn try_optimize_array_get_from_previous_instructions(
270    mut array_id: ValueId,
271    target_index: FieldElement,
272    dfg: &DataFlowGraph,
273    side_effects: Option<&ArrayGetOptimizationSideEffects>,
274) -> Option<ArrayGetOptimizationResult> {
275    let original_array_id = array_id;
276    let target_index_u32 = target_index.try_to_u32()?;
277
278    // Arbitrary number of maximum tries just to prevent this optimization from taking too long.
279    let max_tries = 5;
280    for _ in 0..max_tries {
281        if let Some((instruction, other_instruction_id)) =
282            dfg.get_local_or_global_instruction_with_id(array_id)
283        {
284            match instruction {
285                Instruction::ArraySet { array, index, value, .. } => {
286                    if let Some(constant) = dfg.get_numeric_constant(*index) {
287                        if constant == target_index {
288                            match side_effects {
289                                None => {
290                                    // If it's an array_set with the same index as the array_get, we don't
291                                    // use the value at that index. The reason is that the array_set might
292                                    // be under a different predicate than the array_get, so the set value
293                                    // might not be the correct one in the end.
294                                    return None;
295                                }
296                                Some(ArrayGetOptimizationSideEffects {
297                                    side_effects_var,
298                                    array_set_predicates,
299                                }) => {
300                                    // If there's an array_set with the same index as the array_get, we
301                                    // can only apply this optimization if they are under the same predicate.
302                                    let array_set_predicate = array_set_predicates
303                                        .get(&other_instruction_id)
304                                        .expect("Expected to know the predicate of every array_set preceding this array_get");
305
306                                    let array_set_predicate_is_one = dfg
307                                        .get_numeric_constant(*array_set_predicate)
308                                        .is_some_and(|var| var.is_one());
309                                    let can_optimize = array_set_predicate_is_one
310                                        || array_set_predicate == side_effects_var;
311                                    if !can_optimize {
312                                        return None;
313                                    }
314                                }
315                            }
316
317                            return Some(ArrayGetOptimizationResult::Value(*value));
318                        }
319
320                        // If it's for a different known index, we can safely recur, because
321                        // regardless of whether the array_set ends up being executed or not, it
322                        // won't modify the value at the array_get index.
323                        array_id = *array;
324                        continue;
325                    }
326                }
327                Instruction::MakeArray { elements: array, typ: _ } => {
328                    // `make_array` stores its elements in logical order, but after
329                    // `brillig_array_get_and_set` a constant index is shifted past the
330                    // in-memory header (RC for arrays, RC/Size/Capacity for vectors).
331                    // Map the index back to logical space before reading an element.
332                    let offset = make_array_index_offset(dfg, array_id);
333                    if let Some(index) = target_index_u32.checked_sub(offset) {
334                        let index = index as usize;
335                        if index < array.len() {
336                            return Some(ArrayGetOptimizationResult::Value(array[index]));
337                        }
338                    }
339                }
340                _ => (),
341            }
342        } else if let Value::Param { typ: Type::Array(_, length), .. } = &dfg[array_id]
343            && target_index_u32 < length.0
344        {
345            // There's no optimization if we end up getting from the original array
346            if array_id == original_array_id {
347                return None;
348            }
349
350            return Some(ArrayGetOptimizationResult::ArrayGet(array_id));
351        }
352
353        break;
354    }
355
356    None
357}

359/// The Brillig in-memory header offset baked into a constant `array_get`/`array_set` index by
360/// [`brillig_array_get_and_set`][crate::ssa::opt::brillig_array_get_and_set], or `0` when indices
361/// have not been shifted (ACIR, or before that pass has run).
362fn make_array_index_offset(dfg: &DataFlowGraph, array: ValueId) -> u32 {
363    if !dfg.brillig_arrays_offset {
364        return 0;
365    }
366    match *dfg.type_of_value(array) {
367        Type::Array(..) => ArrayOffset::Array.to_u32(),
368        Type::Vector(..) => ArrayOffset::Vector.to_u32(),
369        _ => 0,
370    }
371}

379    #[test]
380    fn resolves_array_get_per_branch_against_its_own_array() {
381        // `v0` selects between two branches. `b1` writes index 0 and reads its own result; `b2`
382        // reads the original `make_array` at index 0. Each read resolves against the array value it
383        // actually names — the conditional write in `b1` is a distinct value, so it does not leak
384        // into `b2`'s read. The branch where the write happened folds to `Field 99`; the branch
385        // where it did not folds to the original `Field 10`.
386        let src = "
387        acir(inline) fn main f0 {
388          b0(v0: u1):
389            v1 = make_array [Field 10, Field 20] : [Field; 2]
390            jmpif v0 then: b1(), else: b2()
391          b1():
392            v2 = array_set v1, index u32 0, value Field 99
393            v3 = array_get v2, index u32 0 -> Field
394            jmp b3(v3)
395          b2():
396            v4 = array_get v1, index u32 0 -> Field
397            jmp b3(v4)
398          b3(v5: Field):
399            return v5
400        }
401        ";
402        let ssa = Ssa::from_str(src).unwrap();
403
404        let ssa = ssa.array_get_optimization();
405        assert_ssa_snapshot!(ssa, @r"
406        acir(inline) fn main f0 {
407          b0(v0: u1):
408            v4 = make_array [Field 10, Field 20] : [Field; 2]
409            jmpif v0 then: b1(), else: b2()
410          b1():
411            v7 = array_set v4, index u32 0, value Field 99
412            jmp b3(Field 99)
413          b2():
414            jmp b3(Field 10)

648    #[test]
649    #[cfg(debug_assertions)]
650    #[should_panic(expected = "enable_side_effects instruction")]
651    fn rejects_enable_side_effects_in_multi_block_function() {
652        // The cross-block cache assumes a non-trivial side-effects predicate is confined to a
653        // single block, which holds exactly when a multi-block function contains no
654        // `enable_side_effects` instruction (those appear only after flattening, which yields a
655        // single block). A multi-block function carrying one breaks that invariant, so the pass
656        // must reject it.
657        //
658        // This malformed shape is normally rejected by SSA validation, which does not run between
659        // every pass, so parse it without validation to exercise the pass's own guard directly.
660        let src = "
661        acir(inline) fn main f0 {
662          b0(v0: u1, v1: [Field; 3]):
663            enable_side_effects v0
664            v2 = array_set v1, index u32 0, value Field 1
665            jmp b1()
666          b1():
667            return
668        }
669        ";
670        let ssa = Ssa::from_str_no_validation(src).unwrap();
671        let _ = ssa.array_get_optimization();
672    }
