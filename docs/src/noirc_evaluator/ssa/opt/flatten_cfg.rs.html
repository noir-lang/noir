1//! This file contains the SSA flattening pass - a required pass for ACIR to remove any remaining
2//! control-flow in the singular program-function, resulting in a single block containing the
3//! program logic.
4//!
5//! ACIR/Brillig differences within this pass:
6//!   - This pass is strictly ACIR-only and never mutates brillig functions.
7//!
8//! Conditions:
9//!   - Precondition: Inlining has been performed which should result in there being no remaining
10//!     `call` instructions to acir/constrained functions (unless they are `InlineType::Fold`).
11//!     This also means the only acir functions in the program should be `main` (if main is
12//!     constrained), or any constrained `InlineType::Fold` functions.
13//!   - Precondition: Each constrained function should have no loops (unrolling has been performed).
14//!   - Precondition: "Equal" constraints have not been turned into "NotEqual".
15//!   - Postcondition: Each constrained function should now consist of only one block where the
16//!     terminator instruction is always a return.
17//!
18//! Relevance to other passes:
19//!   - Flattening effectively eliminates control-flow entirely which can make it easier for
20//!     subsequent passes to optimize code. Mem2reg for example should be able to remove all
21//!     references in constrained (ACIR) code.
22//!   - Flattening inserts `Instruction::IfElse` to merge the values from an if-expression's "then"
23//!     and "else" branches. These are immediately simplified out for numeric values, but for
24//!     arrays and vectors we require the `remove_if_else` SSA pass to later be run to remove the
25//!     remaining `Instruction::IfElse` instructions.
26//!
27//! Implementation details & examples:
28//!
29//! The flatten cfg optimization pass "flattens" the entire control flow graph into a single block.
30//! This includes branches in the CFG with non-constant conditions. Flattening these requires
31//! special handling for operations with side-effects and can lead to a loss of information since
32//! the jmpif will no longer be in the program. As a result, this pass should usually be towards or
33//! at the end of the optimization passes.
34//!
35//! Furthermore, this pass assumes that no loops are present in the program and will assume
36//! that a jmpif is a branch point and will attempt to merge both blocks. No actual looping will occur.
37//!
38//! This pass is also known to produce some extra instructions which may go unused (usually 'Not')
39//! while merging branches. These extra instructions can be cleaned up by a later dead instruction
40//! elimination (DIE) pass.
41//!
42//! Though CFG information is lost during this pass, some key information is retained in the form
43//! of `EnableSideEffectsIf` instructions. Each time the flattening pass enters and exits a branch of
44//! a jmpif, an instruction is inserted to capture a condition that is analogous to the activeness
45//! of the program point. For example:
46//!
47//! ```text
48//! b0(v0: u1):
49//!   jmpif v0, then: b1, else: b2
50//! b1():
51//!   v1 = call f0
52//!   jmp b3(v1)
53//! ... blocks b2 & b3 ...
54//! ```
55//!
56//! Would brace the call instruction as such:
57//! ```text
58//!   enable_side_effects v0
59//!   v1 = call f0
60//!   enable_side_effects u1 1
61//! ```
62//!
63//! (Note: we restore to "true" to indicate that this program point is not nested within any
64//! other branches. Each `enable_side_effects` overrides the previous, they do not implicitly stack.)
65//!
66//! When we are flattening a block that was reached via a jmpif with a non-constant condition `c`,
67//! the following transformations of certain instructions within the block are expected:
68//!
69//! 1. A constraint is multiplied by the condition and changes the constraint to
70//!    an equality with `c`:
71//! ```text
72//! constrain v0
73//! ---- becomes ----
74//! v1 = mul v0, c
75//! v2 = eq v1, c
76//! constrain v2
77//! ```
78//!
79//! 2. If we reach the end block of the branch created by the jmpif instruction, its block parameters
80//!    will be merged. To merge the jmp arguments of the then and else branches, the formula
81//!    `c * then_arg + !c * else_arg` is used for each argument. Note that this is represented by
82//!    `Instruction::IfElse` which is often simplified to the above when inserted, but in the case
83//!    of complex values (arrays and vectors) this simplification is delayed until the
84//!    `remove_if_else` SSA pass.
85//!
86//! ```text
87//! b0(v0: u1, v1: Field, v2: Field):
88//!   jmpif v0, then: b1, else: b2
89//! b1():
90//!   jmp b3(v1)
91//! b2():
92//!   jmp b3(v2)
93//! b3(v3: Field):
94//!   ... b3 instructions ...
95//! --------- becomes --------
96//! b0(v0: u1, v1: Field, v2: Field):
97//!   v3 = mul v0, v1
98//!   v4 = not v0
99//!   v5 = mul v4, v2
100//!   v6 = add v3, v5
101//!   ... b3 instructions ...
102//! ```
103//!
104//! 3. Each `store v0 in v1` is replaced with a store of a new value
105//!    `v4 = if v3 then v0 else v2` where `v3` is the current condition
106//!    given by `enable_side_effects v3` and `v2` is the result of
107//!    a newly-given `v2 = load v0` inserted before the store.
108//!
109//! ```text
110//! b0(v0: u1):
111//!   v1 = allocate -> &mut Field
112//!   store Field 3 at v1
113//!   jmpif v0, then: b1, else: b2
114//! b1():
115//!   store Field 5 at v1
116//!   ... b1 instructions ...
117//!   jmp b3
118//! b2():
119//!   store Field 7 at v1
120//!   ... b2 instructions ...
121//!   jmp b3
122//! b3():
123//!   ... b3 instructions ...
124//! --------- becomes --------
125//! b0():
126//!   v1 = allocate -> &mut Field
127//!   store Field 3 at v1     // no prior value so we do not load & merge
128//!   enable_side_effects v0  // former block b1
129//!   v2 = load v1 -> Field
130//!   v3 = not v0
131//!   v4 = if v0 then Field 5 else (if v3) v2
132//!   store v4 at v1
133//!   ... b1 instructions ...
134//!   enable_side_effects v3  // former block b2
135//!   v5 = load v1 -> Field
136//!   v6 = if v3 then Field 7 else (if v0) v5
137//!   store v6 at v1
138//!   ... b2 instructions ...
139//!   enable_side_effects u1 1
140//!   ... b3 instructions ...
141//! ```

168    /// Flattens the control flow graph of main such that the function is left with a
169    /// single block containing all instructions and no more control-flow.
170    ///
171    /// This pass will modify any instructions with side effects in particular, often multiplying
172    /// them by jump conditions to maintain correctness even when all branches of a jmpif are inlined.
173    /// For more information, see the module-level comment at the top of this file.
174    #[tracing::instrument(level = "trace", skip(self))]
175    pub(crate) fn flatten_cfg(mut self) -> Ssa {
176        // Retrieve the 'no_predicates' attribute of the functions in a map, to avoid problems with borrowing
177        let no_predicates: HashMap<_, _> =
178            self.functions.values().map(|f| (f.id(), f.is_no_predicates())).collect();
179
180        for function in self.functions.values_mut() {
181            // This pass may run forever on a brillig function - we check if block predecessors have
182            // been processed and push the block to the back of the queue. This loops forever if
183            // there are still any loops present in the program.
184            if matches!(function.runtime(), RuntimeType::Brillig(_)) {
185                continue;
186            }
187
188            #[cfg(debug_assertions)]
189            flatten_cfg_pre_check(function);
190
191            flatten_function_cfg(function, &no_predicates);
192
193            #[cfg(debug_assertions)]
194            flatten_cfg_post_check(function);
195        }
196        self
197    }
198}

200/// Pre-check condition for [Ssa::flatten_cfg].
201///
202/// Panics if:
203///   - Any ACIR function has at least 1 loop
204///   - Any ACIR function has a `ConstrainNotEqual` instruction
205#[cfg(debug_assertions)]
206fn flatten_cfg_pre_check(function: &Function) {
207    if function.runtime().is_acir() {
208        super::checks::assert_no_loops(function);
209        super::checks::for_each_instruction(function, |instruction, _dfg| {
210            super::checks::assert_not_constrain_not_equal(instruction);
211        });
212    }

226/// Mutable context threaded through the CFG flattening pass.
227///
228/// Holds the function inserter, the pre-modification CFG, branch-end map, and all
229/// bookkeeping needed to merge stores and conditions as branches are inlined one by
230/// one into `target_block`.
231pub(crate) struct Context<'f> {
232    pub(crate) inserter: FunctionInserter<'f>,
233
234    /// This `ControlFlowGraph` is the graph from before the function was modified by this flattening pass.
235    cfg: ControlFlowGraph,
236
237    /// Target block of the flattening.
238    pub(crate) target_block: BasicBlockId,
239
240    /// Maps start of branch -> end of branch.
241    branch_ends: HashMap<BasicBlockId, BasicBlockId>,
242
243    /// A stack of each jmpif condition that was taken to reach a particular point in the program.
244    /// When two branches are merged back into one, this constitutes a join point, and is analogous
245    /// to the rest of the program after an if statement. When such a join point / end block is
246    /// found, the top of this conditions stack is popped since we are no longer under that
247    /// condition. If we are under multiple conditions (a nested if), the topmost condition is
248    /// the most recent condition combined with all previous conditions via `And` instructions.
249    condition_stack: Vec<ConditionalContext>,
250
251    /// Arguments prepared by the last inlined block for the next block we are about to process.
252    next_arguments: Option<Vec<ValueId>>,
253
254    /// Stores all allocations local to the current branch.
255    ///
256    /// Since these are local to the current branch (i.e. only defined within one branch of
257    /// an if expression), they should not be merged with their previous value or stored value in
258    /// the other branch since there is no such value.
259    ///
260    /// The `ValueId` here is that which is returned by the allocate instruction.
261    local_allocations: HashSet<ValueId>,
262
263    /// A map from `cond` to `Not(cond)`.
264    ///
265    /// `Not` instructions are inserted constantly by this pass and this map helps keep
266    /// us from unnecessarily inserting extra instructions, and keeps IDs unique which
267    /// helps simplifications.
268    not_instructions: HashMap<ValueId, ValueId>,
269
270    /// Maps merge result ValueId to the provenance of the IfElse that produced it.
271    /// Used to detect and collapse redundant nested merges in `inline_branch_end`.
272    /// See [`Context::try_collapse_merge`] for the four patterns this enables.
273    merge_provenance: HashMap<ValueId, MergeProvenance>,
274
275    /// Flag to tell the context to not issue 'enable_side_effect' instructions during flattening.
276    ///
277    /// It is set with an attribute when defining a function that cannot fail whatsoever to avoid
278    /// the overhead of handling side effects.
279    ///
280    /// It can also be set to true when no instruction is known to fail.
281    pub(crate) no_predicate: bool,
282}

284/// Tracks the origin of a merge result to collapse redundant nested merges.
285///
286/// When nested `jmpif` blocks thread the same value through their else-arguments,
287/// the outer merge is redundant. For example:
288///   `IfElse(c1, IfElse(c2, x, _, y), _, y)` collapses to `IfElse(c2, x, NOT(c2), y)`
289/// because both the inner and outer merges default to the same value `y`.
290///
291/// See [`Context::try_collapse_merge`] for all four supported patterns.
292struct MergeProvenance {
293    then_condition: ValueId,
294    then_value: ValueId,
295    else_condition: ValueId,
296    else_value: ValueId,
297}

303    ///
304    /// It starts out empty, then gets filled in when we finish the branch.

310/// All bookkeeping for a single `jmpif` that is currently being flattened.
311///
312/// Pushed onto `Context::condition_stack` when a `jmpif` is entered and popped when
313/// its join point is reached.
314struct ConditionalContext {
315    /// Condition from the conditional statement
316    condition: ValueId,
317    /// Block containing the conditional statement
318    entry_block: BasicBlockId,
319    /// First block of the then branch
320    then_branch: ConditionalBranch,
321    /// First block of the else branch
322    else_branch: Option<ConditionalBranch>,
323    /// Call stack where the final location is that of the entire `if` expression
324    call_stack: CallStackId,
325    /// List of values which have been replaced with a predicated variant,
326    /// mapping them to their original value.
327    ///
328    /// For example if we have `v1 = v2` predicated upon `v0`, then `v1` becomes `v0 * v2`,
329    /// and this mapping will contain `v1 -> v2`.
330    ///
331    /// We use this information to reset the values to their originals when we exit from branches.
332    predicated_values: HashMap<ValueId, ValueId>,
333    /// The allocations accumulated before processing the branch.
334    local_allocations: HashSet<ValueId>,
335    /// When JmpIf's else_destination is the exit/merge block (no separate else block),
336    /// stores the resolved else_arguments so `inline_branch_end` can use them.
337    jmpif_else_arguments: Option<Vec<ValueId>>,
338}

340/// Flattens the control flow graph of the function such that it is left with a
341/// single block containing all instructions and no more control-flow.
342fn flatten_function_cfg(function: &mut Function, no_predicates: &HashMap<FunctionId, bool>) {
343    // Creates a context that will perform the flattening
344    // We give it the map of the conditional branches in the CFG
345    // and the target block where the flattened instructions should be added.
346    let cfg = ControlFlowGraph::with_function(function);
347    let branch_ends = branch_analysis::find_branch_ends(function, &cfg);
348    let target_block = function.entry_block();
349
350    let mut context = Context::new(function, cfg, branch_ends, target_block);
351
352    context.flatten(no_predicates);
353}

355/// Blocks enqueued for processing.
356///
357/// It contains a block at most once.
358pub(crate) type WorkList = IndexSet<BasicBlockId>;

360impl<'f> Context<'f> {
361    /// Creates a new flattening context.
362    ///
363    /// `cfg` must be computed from `function` before any modifications are made.
364    /// `branch_ends` maps each branch-start block to its join/exit block.
365    /// `target_block` is the single block into which all instructions will be inlined.
366    pub(crate) fn new(
367        function: &'f mut Function,
368        cfg: ControlFlowGraph,
369        branch_ends: HashMap<BasicBlockId, BasicBlockId>,
370        target_block: BasicBlockId,
371    ) -> Self {
372        Context {
373            inserter: FunctionInserter::new(function),
374            cfg,
375            branch_ends,
376            condition_stack: Vec::new(),
377            next_arguments: None,
378            local_allocations: HashSet::default(),
379            not_instructions: HashMap::default(),
380            merge_provenance: HashMap::default(),
381            target_block,
382            no_predicate: false,
383        }
384    }
385
386    /// Flatten the CFG by inlining all instructions from the queued blocks
387    /// until all blocks have been flattened.
388    ///
389    /// We follow the terminator of each block to determine which blocks to process next:
390    /// * If the terminator is a 'JumpIf', we assume we are entering a conditional statement and
391    ///   add the start blocks of the 'then_branch', 'else_branch' and the 'exit' block to the queue.
392    /// * Other blocks will have only one successor, so we will process them iteratively,
393    ///   until we reach one block already in the queue, added when entering a conditional statement,
394    ///   i.e. the 'else_branch' or the 'exit'. In that case we switch to the next block in the queue,
395    ///   instead of the successor.
396    ///
397    /// This process ensures that the blocks are always processed in this order:
398    /// * if_entry -> then_branch -> else_branch -> exit
399    ///
400    /// In case of nested if statements, for instance in the 'then_branch', it will be:
401    /// * if_entry -> then_branch -> if_entry_2 -> then_branch_2 -> exit_2 -> else_branch -> exit
402    ///
403    /// Information about the nested if statements is stored in the 'condition_stack' which
404    /// is popped/pushed when entering/leaving a conditional statement.
405    pub(crate) fn flatten(&mut self, no_predicates: &HashMap<FunctionId, bool>) {
406        let mut work_list = WorkList::new();
407        work_list.insert(self.target_block);
408        while let Some(block) = work_list.pop() {
409            self.inline_block(block, no_predicates);
410            let to_process = self.handle_terminator(block, &work_list);
411            work_list.extend(to_process);
412        }
413        assert!(self.next_arguments.is_none(), "no leftover arguments");
414        self.inserter.map_data_bus_in_place();
415    }

417    /// Returns the updated condition so that
418    /// it is 'AND-ed' with the previous condition (if any)
419    fn link_condition(&mut self, condition: ValueId) -> ValueId {
420        // Retrieve the previous condition
421        if let Some(last_condition) = self.get_last_condition() {
422            let and = Instruction::binary(BinaryOp::And, last_condition, condition);
423            let call_stack = self.inserter.function.dfg.get_value_call_stack_id(condition);
424            self.insert_instruction(and, call_stack)
425        } else {
426            condition
427        }
428    }

430    /// Returns the current condition
431    ///
432    /// The conditions are in a stack, they are added as conditional branches are encountered
433    /// so the last one is the current condition.
434    /// When processing a conditional branch, we first follow the 'then' branch and only after we
435    /// process the 'else' branch. At that point, the `ConditionalContext` has the 'else_branch'
436    fn get_last_condition(&self) -> Option<ValueId> {
437        self.condition_stack
438            .last()
439            .map(|context| context.else_branch.as_ref().unwrap_or(&context.then_branch))
440            .map(|branch| branch.condition)
441    }

443    /// Use the provided map to say if the instruction is a call to a `no_predicates` function
444    fn is_call_to_no_predicate_function(
445        &self,
446        no_predicates: &HashMap<FunctionId, bool>,
447        instruction: &InstructionId,
448    ) -> bool {
449        if let Instruction::Call { func, .. } = self.inserter.function.dfg[*instruction]
450            && let Value::Function(fid) = self.inserter.function.dfg[func]
451        {
452            return no_predicates.get(&fid).copied().unwrap_or_default();
453        }
454        false
455    }

457    /// Prepare the arguments for the next block to consume.
458    ///
459    /// Panics if we already have something prepared.
460    pub(crate) fn prepare_args(&mut self, args: Vec<ValueId>) {
461        assert!(self.next_arguments.is_none(), "already prepared the arguments");
462        if !args.is_empty() {
463            self.next_arguments = Some(args);
464        }
465    }

467    /// Consume the arguments prepared by the previous block.
468    ///
469    /// Panics if there was nothing prepared.
470    fn consume_args(&mut self) -> Vec<ValueId> {
471        self.next_arguments.take().expect("there are no arguments prepared")
472    }

474    /// Inline all instructions from the given block into the target block, and track vector capacities.
475    /// This is done by processing every instruction in the block and using the flattening context
476    /// to push them in the target block.
477    ///
478    /// - `no_predicates` indicates which functions have no predicates and for which we disable the handling of side effects.
479    pub(crate) fn inline_block(
480        &mut self,
481        block: BasicBlockId,
482        no_predicates: &HashMap<FunctionId, bool>,
483    ) {
484        // We do not inline the target block into itself.
485        // This is the case in the beginning for the entry block.
486        if self.target_block == block {
487            return;
488        }
489
490        // If the block has parameters, they should have been prepared by the last block.
491        if !self.inserter.function.dfg.block_parameters(block).is_empty() {
492            let arguments = self.consume_args();
493            self.inserter.remember_block_params(block, &arguments);
494        }
495
496        // If this is not a separate variable, clippy gets confused and says the to_vec is
497        // unnecessary, when removing it actually causes an aliasing/mutability error.
498        let instructions = self.inserter.function.dfg[block].instructions().to_vec();
499        for instruction in instructions {
500            if self.is_call_to_no_predicate_function(no_predicates, &instruction) {
501                // disable side effect for no_predicate functions
502                let bool_type = NumericType::bool();
503                let one = self.inserter.function.dfg.make_constant(FieldElement::one(), bool_type);
504                self.insert_instruction_with_typevars(
505                    Instruction::EnableSideEffectsIf { condition: one },
506                    None,
507                    CallStackId::root(),
508                );
509                self.push_instruction(instruction);
510                self.insert_current_side_effects_enabled();
511            } else {
512                self.push_instruction(instruction);
513            }
514        }
515    }

517    /// Returns the list of blocks that need to be processed after the given block,
518    /// and prepare any arguments for the next-to-be-inlined block to consume.
519    ///
520    /// For a normal block, it would be its successor.
521    ///
522    /// For blocks related to a conditional statement, we ensure to process
523    /// the 'then_branch', then the 'else_branch' (if it exists), and finally the exit block.
524    ///
525    /// The update of the context is done by the functions `if_start`, `then_stop` and `else_stop`
526    /// which perform the business logic when entering a conditional statement, finishing the 'then_branch'
527    /// and the 'else_branch', respectively.
528    ///
529    /// We know if a block is related to the conditional statement if is referenced by the `work_list`.
530    /// Indeed, the start blocks of the 'then_branch' and 'else_branch' are added to the `work_list` when
531    /// starting to process a conditional statement.
532    pub(crate) fn handle_terminator(
533        &mut self,
534        block: BasicBlockId,
535        work_list: &WorkList,
536    ) -> Vec<BasicBlockId> {
537        let terminator = self.inserter.function.dfg[block].unwrap_terminator().clone();
538        match &terminator {
539            TerminatorInstruction::JmpIf {
540                condition,
541                then_destination,
542                then_arguments,
543                else_destination,
544                else_arguments,
545                call_stack,
546            } => {
547                // The `then` branch is next and we can prepare its args now, but the `else`
548                // branch's args need to be prepared only when the branch is later started.
549                let resolved = vecmap(then_arguments, |v| self.inserter.resolve(*v));
550                self.prepare_args(resolved);
551                self.if_start(
552                    condition,
553                    then_destination,
554                    else_destination,
555                    else_arguments,
556                    &block,
557                    *call_stack,
558                )
559            }
560            TerminatorInstruction::Jmp { destination, arguments, call_stack: _ } => {
561                // If the destination is already on the work list, it means it's an exit block in an if-then-else,
562                // and was put there by `if_start` as the last to be processed out of [then, else, exit].
563                if work_list.contains(destination) {
564                    // Since we enqueued [then, else, exit] after each other, if the next block on the work list
565                    // is the exit block, then this must be the else.
566                    if work_list.last().unwrap() == destination {
567                        // The arguments for the exit block will be prepared here.
568                        self.else_stop(&block);
569                    } else {
570                        // No need to prepare arguments: the eventual `else_stop` will look them up directly.
571                        self.then_stop(&block);
572                    }
573                    // The destination was in the queue, no need to return anything.
574                    vec![]
575                } else {
576                    // The destination is a normal block, not an exit block, so there is no argument merging involved,
577                    // we can prepare any arguments for direct consumption.
578                    if !arguments.is_empty() {
579                        let arguments = vecmap(arguments, |value| self.inserter.resolve(*value));
580                        self.prepare_args(arguments);
581                    }
582                    vec![*destination]
583                }
584            }
585            TerminatorInstruction::Return { return_values, call_stack } => {
586                let call_stack = *call_stack;
587                let return_values =
588                    vecmap(return_values.clone(), |value| self.inserter.resolve(value));
589                let new_return = TerminatorInstruction::Return { return_values, call_stack };
590                let target = self.target_block;
591
592                self.inserter.function.dfg.set_block_terminator(target, new_return);
593                vec![]
594            }
595            TerminatorInstruction::Unreachable { .. } => {
596                // The pass which introduces unreachable terminators must come after flattening, as it destroys the CFG.
597                unreachable!("unexpected unreachable terminator in flattening")
598            }
599        }
600    }

602    /// Process a conditional statement by creating a `ConditionalContext`
603    /// with information about the branch, and storing it in the dedicated stack.
604    /// Local allocations are moved to the 'then_branch' of the `ConditionalContext`.
605    /// Returns the blocks corresponding to the 'then_branch', 'else_branch',
606    /// and exit block of the conditional statement, so that they will be processed in this order.
607    pub(crate) fn if_start(
608        &mut self,
609        condition: &ValueId,
610        then_destination: &BasicBlockId,
611        else_destination: &BasicBlockId,
612        else_arguments: &[ValueId],
613        if_entry: &BasicBlockId,
614        call_stack: CallStackId,
615    ) -> Vec<BasicBlockId> {
616        let then_condition = self.inserter.resolve(*condition);
617
618        // Take the current allocations: everything for the new branch is non-local.
619        let branch = ConditionalBranch {
620            condition: self.link_condition(then_condition),
621            // To be filled in by `then_stop`.
622            last_block: None,
623        };
624        let local_allocations = std::mem::take(&mut self.local_allocations);
625        let jmpif_else_arguments = if *else_destination == self.branch_ends[if_entry] {
626            Some(vecmap(else_arguments, |v| self.inserter.resolve(*v)))
627        } else {
628            None
629        };
630        let cond_context = ConditionalContext {
631            condition: then_condition,
632            entry_block: *if_entry,
633            then_branch: branch,
634            // To be filled in by `then_stop`.
635            else_branch: None,
636            call_stack,
637            predicated_values: HashMap::default(),
638            local_allocations,
639            jmpif_else_arguments,
640        };
641        // Clear merge provenance from previous conditionals at this nesting level.
642        // Provenance from a previous conditional's merges must not be re-used by
643        // the current conditional's merge, as the conditions would be from an
644        // unrelated context. Provenance from INNER merges (created by deeper
645        // inline_branch_end calls within the current conditional's branches) will
646        // be freshly recorded and available when this conditional's merge runs.
647        self.merge_provenance.clear();
648        self.condition_stack.push(cond_context);
649        self.insert_current_side_effects_enabled();
650
651        // We disallow this case as it results in the `else_destination` block
652        // being inlined before the `then_destination` block due to block deduplication in the work queue.
653        //
654        // The `else_destination` block then gets treated as if it were the `then_destination` block
655        // and has the incorrect condition applied to it.
656        assert_ne!(
657            self.branch_ends[if_entry], *then_destination,
658            "ICE: branches merge inside of `then` branch"
659        );
660        vec![self.branch_ends[if_entry], *else_destination, *then_destination]
661    }

663    /// Switch context to the 'else_branch':
664    /// - Negates the condition for the 'else_branch' and set it in the `ConditionalContext`
665    /// - Move the local allocations to the 'else_branch'
666    /// - Reset the predicated values to their old mapping in the inserter
667    /// - Issues the 'enable_side_effect' instruction
668    fn then_stop(&mut self, block: &BasicBlockId) {
669        assert_eq!(self.cfg.successors(*block).len(), 1);
670
671        let mut cond_context = self.condition_stack.pop().unwrap();
672        cond_context.then_branch.last_block = Some(*block);
673
674        let condition_call_stack =
675            self.inserter.function.dfg.get_value_call_stack_id(cond_context.condition);
676
677        let else_condition = self.not_instruction(cond_context.condition, condition_call_stack);
678        let else_condition = self.link_condition(else_condition);
679
680        // Pass on the local allocations that came before the 'then_branch' to the 'else_branch'.
681        let else_branch = ConditionalBranch { condition: else_condition, last_block: None };
682        // All local allocations on the stopped 'then_branch' go out of scope.
683        self.local_allocations.clear();
684        cond_context.else_branch = Some(else_branch);
685        self.reset_predicated_values(&mut cond_context);
686        self.condition_stack.push(cond_context);
687
688        self.insert_current_side_effects_enabled();
689    }

692    fn not_instruction(&mut self, condition: ValueId, call_stack: CallStackId) -> ValueId {

702    /// Attempt to collapse a redundant nested merge where one value is shared.
703    ///
704    /// When nested `jmpif` blocks thread the same value through their arguments,
705    /// the outer merge is redundant. This detects four patterns:
706    ///
707    /// **Group 1 — shared else_value (y):**
708    /// - `IfElse(c1, IfElse(c2, x, _, y), _, y)` → `IfElse(c2, x, NOT(c2), y)`
709    /// - `IfElse(c1, y, _, IfElse(c2, x, _, y))` → `IfElse(c2, x, NOT(c2), y)`
710    ///
711    /// **Group 2 — shared then_value (x):**
712    /// - `IfElse(c1, x, _, IfElse(_, x, c2e, z))` → `IfElse(c2e, z, NOT(c2e), x)`
713    /// - `IfElse(c1, IfElse(_, x, c2e, z), _, x)` → `IfElse(c2e, z, NOT(c2e), x)`
714    ///
715    /// Returns `(new_then_condition, new_then_value, new_else_value)` if collapsible.
716    fn try_collapse_merge(
717        &mut self,
718        then_arg: ValueId,
719        else_arg: ValueId,
720    ) -> Option<(ValueId, ValueId, ValueId)> {
721        // Check then_arg provenance
722        if let Some(prov) = self.merge_provenance.get(&then_arg) {
723            let prov_else = self.inserter.resolve(prov.else_value);
724            let prov_then = self.inserter.resolve(prov.then_value);
725            if prov_else == else_arg {
726                // Group 1: IfElse(c1, IfElse(c2, x, _, y), _, y) → IfElse(c2, x, _, y)
727                let result = (prov.then_condition, prov_then, else_arg);
728                self.merge_provenance.remove(&then_arg);
729                return Some(result);
730            }
731            if prov_then == else_arg {
732                // Group 2: IfElse(c1, IfElse(_, x, c2e, z), _, x) → IfElse(c2e, z, _, x)
733                let result = (prov.else_condition, prov_else, else_arg);
734                self.merge_provenance.remove(&then_arg);
735                return Some(result);
736            }
737        }
738        // Check else_arg provenance (safe because merge_provenance is cleared at
739        // each jmpif entry, so only provenance from the current conditional survives).
740        if let Some(prov) = self.merge_provenance.get(&else_arg) {
741            let prov_else = self.inserter.resolve(prov.else_value);
742            let prov_then = self.inserter.resolve(prov.then_value);
743            if prov_else == then_arg {
744                // Group 1: IfElse(c1, y, _, IfElse(c2, x, _, y)) → IfElse(c2, x, _, y)
745                let result = (prov.then_condition, prov_then, then_arg);
746                self.merge_provenance.remove(&else_arg);
747                return Some(result);
748            }
749            if prov_then == then_arg {
750                // Group 2: IfElse(c1, x, _, IfElse(_, x, c2e, z)) → IfElse(c2e, z, _, x)
751                let result = (prov.else_condition, prov_else, then_arg);
752                self.merge_provenance.remove(&else_arg);
753                return Some(result);
754            }
755        }
756        None
757    }

759    /// Switch context the 'exit' block of a conditional statement:
760    /// - Retrieves the local allocations from the Conditional Context
761    /// - Reset the predicated values to their old mapping in the inserter
762    /// - Issues the 'enable_side_effect' instruction
763    /// - Joins the arguments from both branches
764    fn else_stop(&mut self, block: &BasicBlockId) {
765        assert_eq!(self.cfg.successors(*block).len(), 1);
766
767        let mut cond_context = self.condition_stack.pop().unwrap();
768        if cond_context.else_branch.is_none() {
769            // `then_stop` has not been called, this means that the conditional statement has no else branch
770            // so we simply do the `then_stop` now, sandwiched between pushing the context back on the stack,
771            // then popping it again after `then_stop` is done popping and pushing.
772            self.condition_stack.push(cond_context);
773            self.then_stop(block);
774            cond_context = self.condition_stack.pop().unwrap();
775        }
776
777        let mut else_branch = cond_context.else_branch.unwrap();
778        self.local_allocations = std::mem::take(&mut cond_context.local_allocations);
779        else_branch.last_block = Some(*block);
780        cond_context.else_branch = Some(else_branch);
781
782        self.reset_predicated_values(&mut cond_context);
783
784        // We must remember to reset whether side effects are enabled when both branches
785        // end, in addition to resetting the value of old_condition since it is set to
786        // known to be true/false within the then/else branch respectively.
787        self.insert_current_side_effects_enabled();
788
789        // While there is a condition on the stack we don't compile outside the condition
790        // until it is popped. This ensures we inline the full then and else branches
791        // before continuing from the end of the conditional here where they can be merged properly.
792        let end = self.branch_ends[&cond_context.entry_block];
793
794        // Merge arguments and stores from the else/end branches
795        self.inline_branch_end(end, cond_context);
796    }

798    /// Inline the ending block of a branch, the point where all blocks from a jmpif instruction
799    /// join back together. In particular this function must handle merging block arguments from
800    /// all of the join point's predecessors, and it must handle any differing side effects from
801    /// each branch.
802    ///
803    /// The merge of arguments is done by inserting an 'IfElse' instructions which returns
804    /// the argument from the 'then_branch' or the 'else_branch' depending the the condition.
805    ///
806    /// The arguments are prepared for the destination to consume in the next immediate inlining.
807    fn inline_branch_end(&mut self, destination: BasicBlockId, cond_context: ConditionalContext) {
808        assert_eq!(self.cfg.predecessors(destination).len(), 2);
809
810        // Look up and resolve the 'else' and 'then' arguments directly in their terminators,
811        // rather than rely on argument passing in the context.
812        // When JmpIf's else_destination is the exit block, the else_arguments were stored
813        // in jmpif_else_arguments since there is no separate else block to read them from.
814        let else_args = if let Some(args) = &cond_context.jmpif_else_arguments {
815            args.clone()
816        } else if cond_context.else_branch.is_some() {
817            let last_else = cond_context.else_branch.as_ref().unwrap().last_block.unwrap();
818            self.inserter.function.dfg[last_else].terminator_arguments().to_vec()
819        } else {
820            Vec::new()
821        };
822
823        let last_then = cond_context.then_branch.last_block.unwrap();
824        let then_args = self.inserter.function.dfg[last_then].terminator_arguments().to_vec();
825
826        let params = self.inserter.function.dfg.block_parameters(destination);
827        assert_eq!(params.len(), then_args.len());
828        assert_eq!(params.len(), else_args.len());
829
830        if params.is_empty() {
831            return;
832        }
833
834        let args = vecmap(then_args.iter().zip_eq(else_args), |(then_arg, else_arg)| {
835            (self.inserter.resolve(*then_arg), self.inserter.resolve(else_arg))
836        });
837        let Some(else_branch) = cond_context.else_branch else {
838            unreachable!("malformed branch");
839        };
840        let block = self.target_block;
841
842        // Cannot include this in the previous vecmap since it requires exclusive access to self
843        let args = vecmap(args, |(then_arg, else_arg)| {
844            let call_stack = cond_context.call_stack;
845
846            // Try to collapse a redundant nested merge. When the inner merge's
847            // else_value (or then_value) matches the outer merge's corresponding
848            // argument, the two merges can be combined into one.
849            let collapsed = self.try_collapse_merge(then_arg, else_arg);
850            let (then_condition, then_value, else_condition, else_value) =
851                if let Some((inner_then_cond, inner_then_val, shared_val)) = collapsed {
852                    // For the collapsed merge, the then_condition is the inner's
853                    // condition which already incorporates all outer conditions.
854                    // The correct else_condition is NOT(then_condition).
855                    let inner_else_cond = self.not_instruction(
856                        inner_then_cond,
857                        self.inserter.function.dfg.get_value_call_stack_id(inner_then_cond),
858                    );
859                    (inner_then_cond, inner_then_val, inner_else_cond, shared_val)
860                } else {
861                    (cond_context.then_branch.condition, then_arg, else_branch.condition, else_arg)
862                };
863
864            let instruction =
865                Instruction::IfElse { then_condition, then_value, else_condition, else_value };
866            let result = self
867                .inserter
868                .function
869                .dfg
870                .insert_instruction_and_results(instruction, block, None, call_stack)
871                .first();
872
873            // Record provenance only for non-collapsed merges — collapsed results
874            // carry conditions from an inner nesting level that may not be "under"
875            // the next outer condition. Provenance is also consumed (removed) on
876            // use by `try_collapse_merge`, preventing stale entries from matching
877            // at unrelated merge points. Skip when the IfElse simplified to an
878            // existing value (e.g. then_value == else_value, or one of the conditions).
879            // If we don't skip conditions, an inner merge that simplifies to its
880            // own condition (e.g. IfElse(v0, 1, _, 0) -> v0) would attach provenance
881            // to v0, causing false collapses at outer merges that use v0 as an argument.
882            if collapsed.is_none()
883                && result != then_condition
884                && result != then_value
885                && result != else_condition
886                && result != else_value
887            {
888                self.merge_provenance.insert(
889                    result,
890                    MergeProvenance { then_condition, then_value, else_condition, else_value },
891                );
892            }
893
894            result
895        });
896
897        self.prepare_args(args);
898    }

971    /// Push the given instruction to the end of the target block of the current function.
972    ///
973    /// Note that each ValueId of the instruction will be mapped via `self.inserter.resolve`.
974    /// As a result, the instruction that will be pushed will actually be a new instruction
975    /// with a different InstructionId from the original. The results of the given instruction
976    /// will also be mapped to the results of the new instruction.
977    fn push_instruction(&mut self, id: InstructionId) {
978        let (instruction, call_stack) = self.inserter.map_instruction(id);
979        let instruction = self.handle_instruction_side_effects(instruction, call_stack);
980
981        let instruction_is_allocate = matches!(&instruction, Instruction::Allocate);
982        let results = self.inserter.push_instruction_value(
983            instruction,
984            id,
985            self.target_block,
986            call_stack,
987            true,
988        );
989
990        // Remember an allocate was created local to this branch so that we do not try to merge store
991        // values across branches for it later.
992        if instruction_is_allocate {
993            self.local_allocations.insert(results.first());
994        }
995    }

997    /// If we are currently in a branch, we need to modify instructions that have side effects
998    /// (e.g. constraints, stores, range checks) to ensure that the side effect is only applied
999    /// if their branch is taken.
1000    ///
1001    /// For instance we multiply constrain instructions by the branch's condition (see optimization #1 in the module comment).
1002    fn handle_instruction_side_effects(
1003        &mut self,
1004        instruction: Instruction,
1005        call_stack: CallStackId,
1006    ) -> Instruction {
1007        let Some(condition) = self.get_last_condition() else { return instruction };
1008
1009        match instruction {
1010            Instruction::Constrain(lhs, rhs, message) => {
1011                // Replace constraint `lhs == rhs` with `condition * lhs == condition * rhs`.
1012                let lhs = self.mul_by_condition(lhs, condition, call_stack);
1013                let rhs = self.mul_by_condition(rhs, condition, call_stack);
1014                Instruction::Constrain(lhs, rhs, message)
1015            }
1016            Instruction::ConstrainNotEqual(_, _, _) => {
1017                unreachable!("flattening cannot handle ConstrainNotEqual");
1018            }
1019            Instruction::Store { address, value } => {
1020                // If this store is to a reference that was allocated on this branch,
1021                // then we don't have to merge with anything else, we can ignore the condition.
1022                if self.local_allocations.contains(&address) {
1023                    Instruction::Store { address, value }
1024                } else {
1025                    // If the reference was allocated before this condition took effect, then we must only
1026                    // overwrite it if the condition is true.
1027                    // Instead of storing `value`, we store: `if condition { value } else { previous_value }`
1028                    let typ = self.inserter.function.dfg.type_of_value(value).into_owned();
1029                    let load = Instruction::Load { address };
1030                    let previous_value = self
1031                        .insert_instruction_with_typevars(load, Some(vec![typ]), call_stack)
1032                        .first();
1033
1034                    let else_condition = self.not_instruction(condition, call_stack);
1035
1036                    let instruction = Instruction::IfElse {
1037                        then_condition: condition,
1038                        then_value: value,
1039                        else_condition,
1040                        else_value: previous_value,
1041                    };
1042
1043                    let updated_value = self.insert_instruction(instruction, call_stack);
1044                    Instruction::Store { address, value: updated_value }
1045                }
1046            }
1047            Instruction::RangeCheck { value, max_bit_size, assert_message } => {
1048                // Replace value with `value * predicate` to zero out value when predicate is inactive.
1049
1050                // Condition needs to be cast to argument type in order to multiply them together.
1051                let casted_condition =
1052                    self.cast_condition_to_value_type(condition, value, call_stack);
1053                let predicate_value = self.mul_by_condition(value, casted_condition, call_stack);
1054                // Issue #8617: update the value to be the predicated value.
1055                // This ensures that the value has the correct bit size in all cases.
1056                self.predicate_value(value, predicate_value);
1057                Instruction::RangeCheck { value: predicate_value, max_bit_size, assert_message }
1058            }
1059            Instruction::Call { func, arguments } => {
1060                let arguments =
1061                    self.handle_call_side_effects(condition, func, arguments, call_stack);
1062                Instruction::Call { func, arguments }
1063            }
1064            // The following instructions don't need their arguments nullified:
1065            Instruction::Binary(_)
1066            | Instruction::Cast(_, _)
1067            | Instruction::Not(_)
1068            | Instruction::Truncate { .. }
1069            | Instruction::Allocate
1070            | Instruction::Load { .. }
1071            | Instruction::EnableSideEffectsIf { .. }
1072            | Instruction::ArrayGet { .. }
1073            | Instruction::ArraySet { .. }
1074            | Instruction::IncrementRc { .. }
1075            | Instruction::DecrementRc { .. }
1076            | Instruction::IfElse { .. }
1077            | Instruction::MakeArray { .. }
1078            | Instruction::Noop => instruction,
1079        }
1080    }

1102    /// Weave the `condition` into intrinsic call arguments, returning the modified arguments.
1103    fn handle_intrinsic_side_effects(
1104        &mut self,
1105        condition: ValueId,
1106        intrinsic: Intrinsic,
1107        mut arguments: Vec<ValueId>,
1108        call_stack: CallStackId,
1109    ) -> Vec<ValueId> {
1110        match intrinsic {
1111            Intrinsic::ToBits(_) | Intrinsic::ToRadix(_) => {
1112                let field = arguments[0];
1113                let casted_condition =
1114                    self.cast_condition_to_value_type(condition, field, call_stack);
1115                let field = self.mul_by_condition(field, casted_condition, call_stack);
1116
1117                arguments[0] = field;
1118
1119                arguments
1120            }
1121            Intrinsic::BlackBox(blackbox) => {
1122                self.handle_blackbox_side_effects(condition, blackbox, arguments, call_stack)
1123            }
1124            // The following intrinsics may have side effects, but we don't deal with them by
1125            // multiplying their arguments with the condition.
1126            Intrinsic::ArrayLen
1127            | Intrinsic::ArrayAsStrUnchecked
1128            | Intrinsic::AsVector
1129            | Intrinsic::AssertConstant
1130            | Intrinsic::StaticAssert
1131            | Intrinsic::VectorPushBack
1132            | Intrinsic::VectorPushFront
1133            | Intrinsic::VectorPopBack
1134            | Intrinsic::VectorPopFront
1135            | Intrinsic::VectorInsert
1136            | Intrinsic::VectorRemove

1149    /// Weave the `condition` into blackbox call arguments, returning the modified arguments.
1150    fn handle_blackbox_side_effects(
1151        &mut self,
1152        condition: ValueId,
1153        blackbox: BlackBoxFunc,
1154        mut arguments: Vec<ValueId>,
1155        call_stack: CallStackId,
1156    ) -> Vec<ValueId> {
1157        match blackbox {
1158            BlackBoxFunc::EmbeddedCurveAdd => {
1159                arguments[6] = self.mul_by_condition(arguments[6], condition, call_stack);
1160                arguments
1161            }
1162
1163            BlackBoxFunc::MultiScalarMul => {
1164                arguments[2] = self.mul_by_condition(arguments[2], condition, call_stack);
1165                arguments
1166            }
1167
1168            BlackBoxFunc::EcdsaSecp256k1 | BlackBoxFunc::EcdsaSecp256r1 => {
1169                arguments[4] = self.mul_by_condition(arguments[4], condition, call_stack);
1170                arguments
1171            }
1172
1173            // The predicate is injected in ACIRgen so no modification is needed here.
1174            BlackBoxFunc::RecursiveAggregation => arguments,
1175
1176            // These functions will always be satisfiable no matter the input so no modification is needed.
1177            BlackBoxFunc::AND
1178            | BlackBoxFunc::XOR
1179            | BlackBoxFunc::AES128Encrypt
1180            | BlackBoxFunc::Blake2s
1181            | BlackBoxFunc::Blake3
1182            | BlackBoxFunc::Keccakf1600
1183            | BlackBoxFunc::Poseidon2Permutation
1184            | BlackBoxFunc::Sha256Compression => arguments,
1185
1186            BlackBoxFunc::RANGE => {
1187                unreachable!("RANGE should have been converted into `Instruction::RangeCheck`")
1188            }
1189        }
1190    }

1192    /// 'Cast' the 'condition' to 'value' type
1193    ///
1194    /// This is needed because we need to multiply the condition with several values
1195    /// in order to 'nullify' side-effects when the 'condition' is false (in 'handle_instruction_side_effects' function).
1196    ///
1197    /// Since the condition is a boolean, it can be safely casted to any other type.
1198    fn cast_condition_to_value_type(
1199        &mut self,
1200        condition: ValueId,
1201        value: ValueId,
1202        call_stack: CallStackId,
1203    ) -> ValueId {
1204        let argument_type = self.inserter.function.dfg.type_of_value(value);
1205        let cast = Instruction::Cast(condition, argument_type.unwrap_numeric());
1206        self.insert_instruction(cast, call_stack)
1207    }

1209    /// Insert a multiplication between 'condition' and 'value'
1210    fn mul_by_condition(
1211        &mut self,
1212        value: ValueId,
1213        condition: ValueId,
1214        call_stack: CallStackId,
1215    ) -> ValueId {
1216        // Unchecked mul because the condition is always 0 or 1
1217        let cast_condition = self.cast_condition_to_value_type(condition, value, call_stack);
1218        self.insert_instruction(
1219            Instruction::binary(BinaryOp::Mul { unchecked: true }, value, cast_condition),
1220            call_stack,
1221        )
1222    }
