Noir's reference material is produced by several different tools and lives on a few different
surfaces — some hand-written in this site, some generated from the source code at build time, and
some published separately as rustdoc. This page is the single map: use it to find which reference is
authoritative for the question you have.

## Which reference should I use?

- **"How do I run / build / prove with Noir from the command line?"** → [Nargo CLI](./nargo_commands.md).
- **"What does this language feature mean / how do I write it?"** → [The Noir Language](https://noir-lang.org/docs/noir/concepts/data_types).
- **"What's in the standard library?"** → start with the [narrative stdlib docs](https://noir-lang.org/docs/noir/standard_library/cryptographic_primitives) for explanations, then the [generated stdlib API](https://noir-lang.github.io/noir/stdlib/) for the complete, always-current signatures.
- **"How do I drive Noir from JavaScript / the browser?"** → [NoirJS](./NoirJS/noir_js/index.md) (run programs) and [Noir WASM](./NoirJS/noir_wasm/index.md) (compile programs). For generating TypeScript bindings, see [Noir Codegen](./noir_codegen.md).
- **"What is the low-level circuit format?"** → the [ACIR reference](https://noir-lang.github.io/noir/docs/acir/circuit/index.html).
- **"How does the compiler work internally?"** → the [compiler architecture overview](https://github.com/noir-lang/noir/blob/master/docs/compiler/architecture.md) for the big picture, and the [compiler Rust API](https://noir-lang.github.io/noir/docs/) for crate-level detail.

:::note

Hand-written reference pages live in this Docusaurus site and are versioned with each Noir release.
Generated pages (the stdlib API, ACIR, and the compiler Rust API) are rustdoc/`nargo doc` output
published to GitHub Pages and always track `master`, so they may describe features ahead of the
latest stable release.

:::
