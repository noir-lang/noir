Noir reference material is split across authored pages, generated API pages, and external backend or IR references. Use this map to choose the right source.

## Noir Commands

Use the generated Nargo command reference for exact CLI syntax, options, defaults, and subcommands. This page is generated during the docs build from the `nargo` binary, so it should be treated as the command-line source of truth.

For workflow-oriented examples, start with [Getting Started](../getting_started/index.md) or [Tooling](../tooling/index.md) instead.

## Language and Code Generation

Use language concept pages under [The Noir Language](../noir/index.md) for syntax and semantics.

Use [Noir Codegen for TypeScript](./noir_codegen.md) when generating TypeScript bindings from exported Noir functions.

## NoirJS and Noir WASM

NoirJS and Noir WASM API pages are generated by TypeDoc during the Docusaurus build. Use them for TypeScript signatures, class methods, return types, and package-level API details.

Use [Building a web app with Noir and Barretenberg](../tutorials/noirjs_app.md) when you need an end-to-end browser example rather than exact API signatures.

## Standard Library

The Docusaurus [Standard Library](../noir/standard_library/index.md) pages are narrative docs. They explain common modules and include examples for containers, cryptographic primitives, traits, recursion, and metaprogramming.

The generated standard library API docs are produced by `nargo doc` and published separately. Use those pages for exact function, trait, struct, and module signatures.

## ACIR

Use the [ACIR reference](https://noir-lang.github.io/noir/docs/acir/circuit/index.html) when you need details about Noir's backend-agnostic circuit representation. Most Noir users do not need ACIR details for everyday development.

## Compiler Architecture

Use the [compiler architecture notes](https://github.com/noir-lang/noir/blob/master/docs/compiler/architecture.md) when working on Noir internals, compiler passes, SSA, Brillig, ACIR generation, or tooling internals.

## Backend Docs

Noir is backend-agnostic, but many proving workflows use Barretenberg. Backend-specific proving, verification, Solidity verifier, browser proving, and recursion workflows may live in the [Barretenberg docs](https://barretenberg.aztec.network/docs/getting_started).
