Expand description

Checks that the type of a function in a trait impl matches the type of the corresponding function declaration in the trait itself.

To do this, given a trait such as: trait Foo<A> { fn foo<B>(...); }

And an impl such as: impl<C> Foo<D> for Bar<E> { fn foo<F>(...); }

We have to substitute:

  • Self for Bar<E>
  • A for D
  • B for F

Before we can type check. Finally, we must also check that the unification result does not introduce any new bindings. This can happen if the impl functionâ€™s type is more general than that of the trait function. E.g. fn baz<A, B>(a: A, b: B) when the impl required fn baz<A>(a: A, b: A).

This does not type check the body of the impl function.
