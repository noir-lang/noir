Oracle functions cannot return more than 1 slice in their output.

This is currently a limitation with the AVM: to return multiple slices of unknown length, it would need to support allocating memory for them in the call handler, and return their final address. Currently only the Brillig codegen knows about the Free Memory Pointer, and the VM writes to whatever address is in the destination, so we can only safely deal with one vector.
