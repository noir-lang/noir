Expand description

Represents something that can be generic over type variables such as a trait, struct type, or type alias.

Used primarily by Elaborator::resolve_type_args so that we can have one function to do this for struct types, type aliases, traits, etc.

fn accepts_named_type_args(&self) -> bool

True if this item kind can ever accept named type arguments. Currently, this is only true for traits. Structs & aliases can never have named args.
