pub struct ModuleData {
    pub parent: Option<LocalModuleId>,
    pub children: HashMap<Ident, LocalModuleId>,
    pub child_declaration_order: Vec<LocalModuleId>,
    scope: ItemScope,
    definitions: ItemScope,
    traits_in_scope: HashMap<TraitId, Ident>,
    pub location: Location,
    pub is_contract: bool,
    pub is_type: bool,
    pub attributes: Vec<SecondaryAttribute>,
}
Expand description

Contains the actual contents of a module: its parent (if one exists), children, and scope with all definitions defined within the scope.

Fields§

§parent: Option<LocalModuleId>§children: HashMap<Ident, LocalModuleId>§child_declaration_order: Vec<LocalModuleId>

Each child in the order they were declared in the parent module. E.g. for a module containing mod foo; mod bar; mod baz this would be vec![foo, bar, baz].

§scope: ItemScope

Contains all definitions visible to the current module. This includes all definitions in self.definitions as well as all imported definitions.

§definitions: ItemScope

Contains only the definitions directly defined in the current module

§traits_in_scope: HashMap<TraitId, Ident>

All traits in scope, either from use imports or trait declarations. The Ident value is the trait name or the use alias, if any. This is stored separately from scope to quickly check if a trait is in scope.

§location: Location§is_contract: bool

True if this module is a contract Foo { ... } module containing contract functions

§is_type: bool

True if this module is actually a type

§attributes: Vec<SecondaryAttribute>
