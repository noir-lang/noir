    source_variables: HashMap<SourceVarId, String>,

    source_to_debug_vars: HashMap<SourceVarId, DebugVarId>,
    variables: HashMap<DebugVarId, (SourceVarId, DebugTypeId)>,
    functions: HashMap<DebugFnId, DebugFunction>,
    types: HashMap<DebugTypeId, PrintableType>,
    types_reverse: HashMap<PrintableType, DebugTypeId>,
    next_var_id: u32,
    next_type_id: u32,
}
Expand description

We keep a collection of the debug variables and their types in this structure. The source_var_id refers to the ID given by the debug instrumenter. This variable does not have a type yet and hence it can be instantiated for multiple types if itโs in the context of a generic variable. The var_id refers to the ID of the instantiated variable which will have a valid type.
