pub(super) struct FunctionContext<'a> {
    ctx: &'a mut Context,
    id: FuncId,
    next_local_id: u32,
    next_ident_id: u32,
    budget: usize,
    globals: Scope<GlobalId>,
    locals: ScopeStack<LocalId>,
    dynamics: BTreeSet<LocalId>,
    in_loop: bool,
    in_no_dynamic: bool,
    call_targets: BTreeMap<CallableId, HashSet<Type>>,
    has_call: bool,
}
Expand description

Context used during the generation of a function body.

Fields§

§ctx: &'a mut Context

Top level context, to access global variables and other functions.

§id: FuncId

Self ID.

§next_local_id: u32

Every variable created in the function will have an increasing ID, which does not reset when variables go out of scope.

§next_ident_id: u32

Every identifier created in the function will have an increasing ID, which does not reset when variables go out of scope.

§budget: usize

Number of statements remaining to be generated in the function.

§globals: Scope<GlobalId>

Global variables.

§locals: ScopeStack<LocalId>

Variables accumulated during the generation of the function body, initially consisting of the function parameters, then extended by locally defined variables. Block scopes add and remove layers.

§dynamics: BTreeSet<LocalId>

Indicate which local variables are derived from function inputs.

§in_loop: bool

Indicator of being in a loop (and hence able to generate break and continue statements)

§in_no_dynamic: bool

Indicator of computing an expression that should not contain dynamic input.

§call_targets: BTreeMap<CallableId, HashSet<Type>>

All the functions callable from this one, with the types we can produce from their return value.

§has_call: bool

Indicate that we have generated a Call.

fn gen_block( &mut self, u: &mut Unstructured<'_>, typ: &Type, ) -> Result<(Expression, bool)>

Generate a block of statements, finally returning a target type.

This should always succeed, as we can always create a literal in the end.

We might use dynamic input in the block and not return a dynamic result from it. For example:

ⓘ
fn main(a: u32) -> u32 {
    let b: u32 = {
        let c = a + 1;
        2
    };
    b
}

Generate a statement, which is an expression that doesn’t return anything, for example loops, variable declarations, etc.

Because the statement doesn’t return anything (return unit) we don’t track whether it used dynamic inputs; there is nothing to propagate.
