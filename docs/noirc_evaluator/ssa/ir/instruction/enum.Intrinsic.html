Expand description

These are similar to built-ins in other languages. These can be classified under two categories:

  • Opcodes which the IR knows the target machine has special support for. (LowLevel)
  • Opcodes which have no function definition in the source code and must be processed by the IR.

pub(crate) fn has_side_effects(&self) -> bool

Returns whether the Intrinsic has side effects.

If there are no side effects then the Intrinsic can be removed if the result is unused.

An example of a side effect is increasing the reference count of an array, but functions which can fail due to implicit constraints are also considered to have a side effect.
