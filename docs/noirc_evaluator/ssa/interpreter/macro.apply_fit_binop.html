Expand description

Applies a fallible integer binary operation on Fitted values, promoting values to Field in case there is an overflow, thus turning the operation infallible.

If the result is an overflow, it promotes the values to Field and performs the operation there. If the operation is applied on Unfit values, and the result fits in the original numeric type, it is converted back to a Fit value.

For example we would normally have an infallible wrapped_add, but we want to match ACIR by not wrapping around but extending into larger bit sizes.
