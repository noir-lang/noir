should_insert_oob_check 🔒
Array get/set has explicit out of bounds (OOB) checks laid down in the Brillig runtime. These checks are not laid down in the ACIR runtime as that runtime maps the SSA to a memory model where OOB accesses will be prevented. Essentially all array ops in ACIR will have a side effect where they check for the index being OOB. However, in order to maintain parity between the Brillig and ACIR runtimes, if we have an unused array operation we need insert an OOB check so that the side effects ordering remains correct.
