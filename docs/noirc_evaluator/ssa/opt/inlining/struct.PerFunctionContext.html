struct PerFunctionContext<'function> {
    entry_function: &'function Function,
    source_function: &'function Function,
    context: &'function mut InlineContext,
    values: HashMap<ValueId, ValueId>,
    blocks: HashMap<BasicBlockId, BasicBlockId>,
    inlining_entry: bool,
    globals: &'function GlobalsGraph,
}
Expand description

The per-function inlining context contains information that is only valid for one function. For example, each function has its own DataFlowGraph, and thus each function needs a translation layer to translate between BlockId to BlockId for the current function and the function to inline into. The same goes for ValueIds, InstructionIds, and for storing other data like parameter to argument mappings.

ยงblocks: HashMap<BasicBlockId, BasicBlockId>

Maps blocks in the source function to blocks in the function being inlined into, where each mapping is from the start of a source block to an inlined block in which the analogous program point occurs.

Note that the starts of multiple source blocks can map into a single inlined block. Conversely the whole of a source block is not guaranteed to map into a single inlined block.

fn translate_value(&mut self, id: ValueId) -> ValueId

Translates a ValueId from the function being inlined to a ValueId of the function being inlined into. Note that this expects value ids for all Value::Instruction and Value::Param values are already handled as a result of previous inlining of instructions and blocks respectively. If these assertions trigger it means a value is being used before the instruction or block that defines the value is inserted.
