) -> bool
Expand description

True if array[i] requires an explicit SSA out-of-bounds check rather than relying on ACIR’s implicit OOB check on the underlying memory op.

ACIR derives the implicit check from the memory op emitted for array_get / array_set. When the array’s element flattens to zero ACIR cells (e.g. [(); N]), no memory op is laid down — for array_get the access can be optimized away, and for array_set the value to store has zero cells to write — so the implicit check is missing and an explicit one must be emitted. Brillig has no implicit check at all and always needs the explicit one. array_type must be a Type::Array.
